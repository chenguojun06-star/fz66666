## 🏗️ 系统架构规范

### 后端架构（Orchestrator模式）

```
Controller → Orchestrator → Service → Mapper
```

**层次职责**：
- **Controller**: 接收请求，参数验证（@PreAuthorize权限控制）
- **Orchestrator**: 跨服务业务编排，复杂事务管理（26个编排器）
- **Service**: 单表/单领域CRUD操作
- **Mapper**: MyBatis Plus数据访问

**核心原则**：
- ✅ 复杂业务逻辑必须使用Orchestrator编排
- ✅ 跨Service调用必须通过Orchestrator
- ❌ Controller不允许直接调用多个Service
- ❌ Service不允许互相调用

### 前端架构（模块化路由）

```
frontend/src/
  ├── modules/              # 业务模块（按功能分组）
  │   ├── basic/            # 基础数据（款式、订单、模板、数据中心）
  │   ├── production/       # 生产管理（订单、裁剪、质检、样板等）
  │   ├── finance/          # 财务管理（对账、工资、成本）
  │   ├── system/           # 系统管理（用户、角色、字典、日志）
  │   └── dashboard/        # 数据看板
  ├── components/           # 通用组件（26个标准组件）
  ├── services/             # API服务层（统一封装）
  ├── utils/                # 工具函数
  └── styles/               # 全局样式
```

**模块划分原则**：
- ✅ 按业务功能分组（不按技术分层）
- ✅ 每个模块独立入口（index.tsx懒加载）
- ✅ 模块内高内聚（相关页面放一起）
- ❌ 禁止跨模块直接引用组件（使用通用组件）

### 小程序架构（工具函数模式）

```
miniprogram/
  ├── pages/                # 页面
  ├── components/           # 组件
  ├── utils/                # 工具函数（核心）
  │   ├── request.js        # 网络请求
  │   ├── dataValidator.js  # 数据验证
  │   ├── errorHandler.js   # 错误处理
  │   └── syncManager.js    # 数据同步
  └── styles/               # 设计系统（Design Token）
```

## 🔤 字体系统（Typography）

### 最佳字体栈（推荐）

```css
/* 全局默认字体（已优化） */
font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', 
             'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei',
             'Roboto', 'Helvetica Neue', 'Arial', sans-serif;
```

**字体优先级**：
1. **系统字体优先**（-apple-system, BlinkMacSystemFont）
2. **中文字体**（PingFang SC > Hiragino Sans GB > Microsoft YaHei）
3. **英文后备**（Roboto > Helvetica Neue > Arial）

**特性**：
- ✅ **跨平台一致性**（macOS, Windows, Linux）
- ✅ **中文渲染优秀**（PingFang SC 是最佳选择）
- ✅ **性能优化**（系统内置，无需加载）

### 字体大小规范

```css
/* 全局规范（14px基准） */
--font-size-xs: 12px;      /* 辅助文字 */
--font-size-sm: 13px;      /* 次要文字 */
--font-size-base: 14px;    /* 正文（基准） */
--font-size-md: 15px;      /* 小标题 */
--font-size-lg: 16px;      /* 标题 */
--font-size-xl: 18px;      /* 大标题 */
--font-size-2xl: 20px;     /* 页面标题 */
--font-size-3xl: 24px;     /* 超大标题 */
```

**使用场景**：
- **12px**: Tag标签、时间戳、辅助信息
- **13px**: Modal字段标签、次要说明
- **14px**: 正文、表格内容、按钮文字（标准）
- **15px**: Modal段落标题、重要字段标签
- **16px**: 页面小标题、Card标题
- **18px**: Modal重要字段值（如订单号）
- **20px+**: 页面主标题

### 字体权重规范

```css
--font-weight-normal: 400;    /* 正常 */
--font-weight-medium: 500;    /* 中等 */
--font-weight-semibold: 600;  /* 半粗 */
--font-weight-bold: 700;      /* 粗体 */
```

**使用原则**：
- **400**: 正文内容
- **500**: 次要标题、导航文字
- **600**: 字段标签、重要文字（推荐）
- **700**: 页面标题、强调文字

### 问题1：硬编码颜色（610处）
```tsx
❌ 错误：
```

**影响**：
- ❌ 主题切换困难
- ❌ 颜色不统一
- ❌ 维护成本高

**修复方案**：
```bash
# 1. 批量查找硬编码颜色
grep -r "#[0-9A-Fa-f]\{6\}" frontend/src --include="*.tsx"

# 2. 替换为CSS变量
# #2D7FF9 → var(--primary-color)
# #f5f5f5 → var(--neutral-light)
```

### 问题2：Modal尺寸不统一（39处）

```tsx
❌ 错误：固定像素宽度

<ResizableModal
```

**影响**：
- ❌ 大屏显示过小
- ❌ 小屏显示过大
- ❌ 响应式差

```tsx
// ✅ 永远使用CSS变量
```

```tsx
import ResizableModal from '@/components/common/ResizableModal';

  {/* 内容 */}
```

```tsx
import SortableColumn from '@/components/common/SortableColumn';

<Table dataSource={data}>

  />
```

```tsx
import QRCodeBox from '@/components/common/QRCodeBox';
```

### 常见错误与解决

| 错误示例 | 问题 | 正确做法 |
|---------|------|---------|
| `<Button size="small">` | 非表格场景用small | `<Button size="middle">` |
| `<Modal width={600}>` | 固定像素宽度 | `<ResizableModal defaultWidth="40vw">` |
| `style={{ color: '#2D7FF9' }}` | 硬编码颜色 | `style={{ color: 'var(--primary-color)' }}` |
| `padding: '10px'` | 非标准间距 | `padding: 'var(--spacing-sm)'` |
| `background: linear-gradient(...)` | 使用渐变 | `background: var(--primary-color)` 纯色 |

# 工序节点数据联动实现说明

## 📋 需求概述

实现订单工序明细中**裁剪、车缝、尾部、入库**的完整数据联动：
1. **采购节点**：已完成（显示完成状态、操作人、时间）
2. **裁剪节点**：显示完成数量、剩余数量、操作人、时间
3. **车缝节点**：每个子工序都显示完成数量、剩余数量
4. **尾部节点**：每个子工序都显示完成数量、剩余数量
5. **入库节点**：显示完成数量、剩余数量、操作人、时间

## 🎯 前端实现要点

### 1. 状态管理

在订单列表页面添加状态：

```typescript
// 工序状态（包含所有节点）
const [processStatus, setProcessStatus] = useState<any>(null);
```

### 2. 数据获取

在打开工序明细时获取所有工序状态：

```typescript
const openProcessDetail = async (record: ProductionOrder, type: 'all' | 'procurement' | 'cutting' | 'sewing' | 'finishing' | 'warehousing') => {
  setProcessDetailRecord(record);
  setProcessDetailType(type);
  setProcessDetailVisible(true);
  
  // 获取所有工序状态
  try {
    const res = await api.get(`/production/order/process-status/${record.id}`);
    setProcessStatus(res.data.data);
  } catch (err) {
    console.error('[工序状态] 获取失败:', err);
  }
  
  // 仍然保留采购单独获取（向后兼容）
  if (type === 'procurement' || type === 'all') {
    try {
      const res = await api.get(`/production/order/procurement-status/${record.id}`);
      setProcurementStatus(res.data.data);
    } catch (err) {
      console.error('[采购状态] 获取失败:', err);
    }
  }
};
```

## 📊 数据流转

```
1. 用户点击"工序明细"按钮
   ↓
2. 调用 openProcessDetail(record, type)
   ↓
3. 同时发起两个API请求：
   - GET /process-status/{orderId}  （所有工序状态）
   - GET /procurement-status/{orderId} （采购状态-向后兼容）
   ↓
4. 后端从 ProductionOrder 表读取字段：
   - cuttingQuantity, cuttingCompletionRate, cuttingOperatorName, cuttingEndTime
   - sewingCompletionRate, sewingOperatorName, sewingEndTime
   - qualityCompletionRate, qualityOperatorName, qualityEndTime
   - warehousingQualifiedQuantity, warehousingCompletionRate, warehousingOperatorName
   ↓
5. 计算剩余数量：
   - 裁剪剩余 = orderQuantity - cuttingQuantity
   - 车缝剩余 = cuttingQuantity - warehousingQualifiedQuantity
   - 尾部剩余 = cuttingQuantity - warehousingQualifiedQuantity
   - 入库剩余 = cuttingQuantity - warehousingQualifiedQuantity
   ↓
6. 前端更新状态：setProcessStatus(data)
   ↓
7. UI根据 processStatus 显示各节点的完成数量、剩余数量、操作人
```

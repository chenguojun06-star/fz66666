# 小程序生成菲号功能使用指南

## 📱 功能说明

在小程序端添加了**一键生成菲号**功能，允许用户直接在手机上生成裁剪菲号，无需登录PC端操作。

## ✨ 功能特点

1. **快速生成** - 在工作页面直接点击按钮生成
2. **统一弹窗** - 使用与扫码确认相同的弹窗样式
3. **多行录入** - 支持添加多组颜色/尺码/数量
4. **实时同步** - 生成后立即刷新订单列表

## 🎯 使用流程

### 1. 进入工作页面
- 打开小程序 → 底部导航栏 → **生产**

### 2. 找到需要生成菲号的订单
- 订单卡片上会显示当前进度
- **裁剪**或**采购**阶段的订单会显示"📋 生成菲号"按钮

### 3. 点击生成菲号
- 点击订单卡片上的"📋 生成菲号"按钮
- 弹出生成菲号的表单

### 4. 填写菲号信息
表单中需要填写：
- **颜色**：如 红色、蓝色、黑色
- **尺码**：如 S、M、L、XL、XXL
- **数量**：每扎的数量，如 10、20、50

### 5. 添加多行（可选）
- 点击"+ 添加一行"可以添加更多组合
- 点击"✕ 删除"可以删除某一行

### 6. 确认生成
- 点击"确认生成"按钮
- 系统会自动生成菲号二维码
- 生成成功后显示提示

## 📋 示例

### 示例1：单一规格
```
订单：PO20260122001
款号：ST20260122
颜色：红色
尺码：L
数量：20
```
→ 生成1个菲号：`PO20260122001-ST20260122-红色-L-20-001`

### 示例2：多规格
```
第1行：颜色=红色, 尺码=M, 数量=15
第2行：颜色=红色, 尺码=L, 数量=20
第3行：颜色=蓝色, 尺码=L, 数量=25
```
→ 生成3个菲号：
- `PO20260122001-ST20260122-红色-M-15-001`
- `PO20260122001-ST20260122-红色-L-20-002`
- `PO20260122001-ST20260122-蓝色-L-25-003`

## ⚠️ 注意事项

1. **物料到齐检查**
   - 必须物料100%到齐才能生成菲号
   - 如果提示"物料未到齐"，请先完成采购

2. **任务状态检查**
   - 裁剪任务必须先领取
   - 如果提示"请先领取任务"，请在PC端领取

3. **重复生成检查**
   - 每个订单只能生成一次菲号
   - 如果已生成，会提示"该订单已生成裁剪单"

4. **数据验证**
   - 颜色和尺码不能为空
   - 数量必须大于0
   - 至少要填写一行有效数据

## 🔄 生成后的操作

### 扫码操作
1. 在扫码页面选择"裁剪"类型
2. 扫描生成的菲号二维码
3. 数量会自动从菲号识别
4. 确认提交扫码记录

### 查看菲号
- 生成的菲号会保存在系统中
- 可以在PC端"裁剪管理"页面查看和打印

## 🔧 技术实现

### 前端文件
- `miniprogram/pages/work/index.wxml` - 页面结构
- `miniprogram/pages/work/index.js` - 逻辑处理
- `miniprogram/pages/work/index.wxss` - 样式设计
- `miniprogram/utils/api.js` - API接口

### 后端接口
- **POST** `/api/production/cutting/generate`
- 参数：
  ```json
  {
    "orderId": "订单ID",
    "bundles": [
      { "color": "红色", "size": "L", "quantity": 20 },
      { "color": "蓝色", "size": "M", "quantity": 15 }
    ]
  }
  ```

### 菲号格式
```
{订单号}-{款号}-{颜色}-{尺码}-{数量}-{序号}
例如：PO20260122001-ST20260122-红色-L-20-001
```

## 📊 状态流转

```
待生产 → 采购 → [生成菲号] → 裁剪 → 车缝 → 质检 → 包装 → 入库 → 完成
                    ↑
                小程序一键生成
```

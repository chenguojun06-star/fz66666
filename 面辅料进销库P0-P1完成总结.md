### 核心功能：BOM库存检查

#### 1. 数据库增强
**修改表**：`t_style_bom`
```sql

ALTER TABLE t_style_bom ADD:

- available_stock INT            -- 可用库存数量
- required_purchase INT          -- 需采购数量
```

#### 2. 后端增强
**StyleBom实体**：新增3个字段映射

**StyleBomService新增方法**：
```java
// 1. 保存BOM并自动检查库存
List<StyleBom> saveBomWithStockCheck(List<StyleBom> bomList, Integer productionQty);

// 2. 获取BOM库存汇总
Map<String, Object> getBomStockSummary(Long styleId, Integer productionQty);
```

**核心算法**：
```
需求量 = 单件用量 × 生产数量 × (1 + 损耗率)
可用库存 = 总库存 - 锁定库存

if 可用库存 >= 需求量:
    状态 = "sufficient"（充足）
    需采购 = 0
elif 可用库存 > 0:
    状态 = "insufficient"（不足）
    需采购 = 需求量 - 可用库存
else:
    状态 = "none"（无库存）
    需采购 = 需求量
```

**库存查找规则**：
- 精确匹配：物料编码 + 颜色 + 尺码
- 多条记录时选择库存最多的
- 无记录时返回null（无库存）

#### 3. 测试验证
```bash
./test-bom-stock-check.sh
```

**测试场景**：
- BOM1: 需求800件，库存1000件 → ✅ sufficient
- BOM2: 需求500件，库存300件 → ⚠️ insufficient（需采购200件）
- BOM3: 需求200件，无库存 → ❌ none（需采购200件）

**操作Hook** (useOrderActions / useProgressActions):
```typescript
// 职责：业务操作、错误处理、刷新触发
export const useOrderActions = (refreshCallback) => {
  const [saving, setSaving] = useState(false);
  
  const quickEdit = useCallback(async (record, updates) => {
    setSaving(true);
    try {
      const res = await api.update(record.id, updates);
      if (res.code === 200) {
        message.success('保存成功');
        refreshCallback?.();
        return true;
      }
    } catch (error) {
      message.error(error.message);
      return false;
    } finally {
      setSaving(false);
    }
  }, [refreshCallback]);

  return { saving, quickEdit };
};
```

## 🎯 最佳实践总结

### ✅ DO（推荐做法）

1. **单文件行数 <500行**
   - 主文件控制在 300行左右
   - 组件控制在 150行左右
   - Hook控制在 200行左右

2. **Hooks分离**
   - 数据Hook：获取、缓存、状态
   - 操作Hook：增删改、错误处理、刷新

3. **组件职责单一**
   - 筛选面板：只管筛选
   - 数据表格：只管展示
   - 弹窗：只管详情/编辑

4. **Props明确**
   - 每个组件Props定义清晰的interface
   - 回调函数命名规范（onXxx）

5. **TypeScript严格**
   - 100%类型覆盖
   - 禁止any类型滥用

### ❌ DON'T（避免做法）

1. **超大文件**
   - ❌ 单文件 >1000行
   - ❌ 所有逻辑混在一起

2. **状态膨胀**
   - ❌ 30+个useState
   - ❌ 状态管理混乱

3. **逻辑耦合**
   - ❌ 数据获取和UI渲染混在一起
   - ❌ Service层逻辑直接写在组件中

4. **重复代码**
   - ❌ 相同逻辑在多个页面复制粘贴
   - ❌ 表格列定义重复

5. **命名不规范**
   - ❌ 变量名a、b、c
   - ❌ 函数名handleClick1、handleClick2

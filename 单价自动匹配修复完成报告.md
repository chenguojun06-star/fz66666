## 🔴 问题描述

### 核心问题
**工序单价匹配失败导致工资结算错误**

### 影响范围
- 所有扫码工序的工资计算
- 工人工资对账
- 财务结算数据准确性

### 问题表现
```sql
-- 扫码记录中单价为0
SELECT process_name, unit_price, quantity, total_amount
FROM t_scan_record
WHERE order_no = 'PO20260130003';

-- 结果：
-- process_name | unit_price | quantity | total_amount
-- 裁剪          | 0.00       | 5        | 0.00
```

**期望**: 裁剪工序单价应为 ¥18.00，总金额 = ¥18.00 × 5 = ¥90.00

---

## 🔍 根因分析

### 1. 数据存储问题

#### 订单工序配置（数据库charset问题）
```json
// t_production_order.progress_workflow_json
{
  "nodes": [
    {"id": "01", "name": "??", "unitPrice": 0},
    {"id": "02", "name": "??", "unitPrice": 18},  // 应为"裁剪"
    {"id": "03", "name": "??", "unitPrice": 1}
  ]
}
```

**问题**: 中文工序名因字符集问题存储为 `"??"`

#### 扫码记录（数据正确）
```sql
-- t_scan_record.process_name
SELECT HEX(process_name) FROM t_scan_record WHERE order_no = 'PO20260130003';
-- 结果: E8A381E589AA (裁剪的UTF-8编码) ✅ 正确存储
```

### 2. 匹配逻辑缺陷

**原匹配逻辑**（单一匹配）:
```java
// SKUServiceImpl.java (修复前)
for (Map<String, Object> priceInfo : prices) {
    String name = priceInfo.get("name");  // "??" 从JSON读取
    if (name.equalsIgnoreCase(processName)) {  // "??" != "裁剪"
        // 匹配成功 - 但因为name乱码，永远不会执行
    }
}
// 结果：匹配失败 → unitPrice = 0
```

### 3. 数据流转链路

```
订单创建（前端）
  ↓ (progressWorkflowJson)
数据库存储
  ↓ (charset问题)
工序名变"??"
  ↓
扫码（小程序）
  ↓ (process_name = "裁剪" ✅)
后端匹配
  ↓
"裁剪" vs "??" ❌ 不匹配
  ↓
unitPrice = 0 ❌ 错误结果
```

---

## ✅ 解决方案

### 三层保护机制

修改 `SKUServiceImpl.getUnitPriceByProcess()` 方法，实现三层匹配：

```java
// 【优先级1】通过工序名精确匹配（正常情况）
if (name.equalsIgnoreCase(processName)) {
    result.put("unitPrice", unitPrice);
    result.put("matchBy", "name");
    return result;
}

// 【优先级2】通过工序ID匹配（charset乱码时的fallback）
if (StringUtils.hasText(id) && id.equalsIgnoreCase(processName)) {
    result.put("unitPrice", unitPrice);
    result.put("matchBy", "id");
    log.warn("⚠️ 通过ID匹配成功（名称可能乱码）");
    return result;
}

// 【优先级3】模糊匹配常见工序（最后的兜底）
if (processName.contains("裁剪") && (name.contains("裁") || name.equals("??"))) {
    result.put("unitPrice", unitPrice);
    result.put("matchBy", "fuzzy");
    log.warn("⚠️ 通过模糊匹配成功");
    return result;
}
```

### 增强工序配置解析

修改 `SKUServiceImpl.getProcessUnitPrices()` 确保返回工序ID：

```java
for (Map<String, Object> node : nodes) {
    String processId = node.get("id");   // ✅ 新增：获取工序ID
    String processName = node.get("name");
    
    Map<String, Object> priceInfo = new HashMap<>();
    priceInfo.put("id", processId);      // ✅ 新增：返回ID
    priceInfo.put("name", processName);
    priceInfo.put("unitPrice", unitPrice);
    
    // 即使名称为"??"，只要有ID和单价就添加
    if (StringUtils.hasText(processId) || StringUtils.hasText(processName)) {
        result.add(priceInfo);
    }
}
```

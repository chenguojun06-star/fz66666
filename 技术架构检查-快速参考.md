## 🔴 紧急修复（P0 级）- 立即处理

### 1️⃣ Redis 配置缺失 ❗

---

### 2️⃣ 异步线程池未配置 ❗
**影响**: 高并发时系统崩溃（OOM）  
**风险**: 每个异步任务创建新线程，无复用

**现状**:
```java
❌ SimpleAsyncTaskExecutor (默认，危险)
   └─> 每次创建新线程 → 高并发 → OOM
```

**修复后**:
```java
✅ ThreadPoolTaskExecutor (安全)
   ├─> 核心线程: 10
   ├─> 最大线程: 20
   └─> 队列容量: 500
```

---

### 3️⃣ 分布式锁缺失 ❗
**影响**: 分布式部署时数据重复  
**场景**: 生成订单号、扣减库存

**问题代码**:
```java
// MaterialInboundServiceImpl.java
public synchronized String generateInboundNo() {
    // ❌ 单机锁，多实例部署时会重复
}
```

**修复方案**: 使用 Redis 分布式锁（已提供 `DistributedLockService`）

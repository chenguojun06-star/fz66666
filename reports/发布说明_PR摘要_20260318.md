# 2026-03-18 发布说明 / PR 摘要

## 标题

`fix: 修复智能驾驶舱 what-if、节点舱、进度跳转与裁剪生产方语义`

## 一句话概述

本次更新把智能驾驶舱的 What-If 推演、六阶段节点舱、生产进度分组跳转和裁剪单“内部自产 / 外发加工”语义一起拉回真实业务数据，同时清理了一批历史兼容代码和失效测试，确保发布前结构、编译和关键路径冒烟均通过。

## 本次主要变更

### 1. 智能驾驶舱 What-If 推演恢复真实数据绑定

- 前端不再错误按固定状态抓订单，改为从活跃生产订单中聚合出真实可推演款号
- 前后端统一 What-If 请求结构，改为标准 `type/value/factoryId` 场景参数
- 推演结果新增“推演依据”和“目标工厂名”，不再是纯黑盒分数
- 转厂场景接入真实工厂数据，按剩余件数、平均进度、当前工厂负载、目标工厂产能快照估算收益

### 2. 智能驾驶舱新增六阶段节点舱

- 新增采购、裁剪、二次工艺、车缝、尾部、入库六个节点舱视图
- 节点舱会按订单当前真实阶段自动归舱，并展示订单数、件数、主订单、进度
- 点击节点舱可带 `orderNos` 批量参数跳转到生产进度页，直接聚焦同舱订单集合

### 3. 驾驶舱补齐今日业务统计

- 日报接口新增今日下单、今日入库、今日出库字段
- 驾驶舱首页增加 3 个今日统计卡片，和现有在制、逾期、高风险视图形成联动
- 接口返回值按 `int` 出口，规避长整型序列化造成的前端统计精度问题

### 4. 裁剪单创建语义修正

- 裁剪单创建弹窗新增“内部自产 / 外发加工”切换
- 内部场景要求选择组织部门，外发场景要求选择工厂
- 后端建单时会分别写入 `factoryType`、`factoryId`、`orgUnitId`、组织路径、联系人等字段，避免以前全部落成空工厂名的伪语义

### 5. 生产进度页分组聚焦能力补齐

- 生产进度页支持从 URL 读取 `orderNos` 批量聚焦参数
- 当从驾驶舱或其他联动入口跳转时，可只展示指定订单集合并保持顺序
- 为后续更多“按集合跳转”场景打通基础链路

### 6. 历史兼容代码与失效测试清理

- 删除洗水唛旧兼容序列化方法和小程序扫码内部废弃代理方法
- 清理多处无效 `eslint-disable`、未使用 import / 变量、Ant Design 新版 `Alert` 不兼容写法
- 修复多处后端测试，包括日报、执行引擎、裁剪建单、入库统计、质量 AI 建议、尺码编排等

**文件**: `frontend/src/modules/basic/pages/StyleInfo/components/StyleColorSizeTable.tsx`  
**状态**: ❌ 经分析决定跳过迁移

#### 不迁移原因
1. **架构不兼容**: 
   - 横向固定矩阵结构（3行×6列：码数/颜色/数量行，每行5个SKU）

   - 当前使用50+个独立props（size1-5, color1-5, qty1-5及其setters）
   - Table需要array数据源，改造需要重构整个数据模型

3. **独特功能深度集成**:

   - 内联添加新码数/颜色
   - 自动计算总数量
   - 锁定状态管理

4. **迁移会增加复杂度**:
   - 当前292行代码已经很简洁
   - 强行迁移需要自定义Table组件，反而增加代码量

#### 建议
保持现状，仅考虑轻量优化（提取重复样式为常量）。

## 🎓 经验总结

### 1. 表格改造决策原则

#### ✅ 适合迁移的场景
- 纵向动态行数据（动态添加/删除行）
- 标准列配置（title + dataIndex + render）
- 需要排序、筛选、分页等Table功能
- 大量重复的行内样式

#### ❌ 不适合迁移的场景
- 打印模板表格（需要纯HTML字符串）
- 横向固定矩阵结构（如颜色码数表）
- 非标准数据模型（非array数据源）
- 当前代码已经很简洁

### 2. 改造技巧

#### 可编辑表格实现
```tsx
{
  title: '工序单价',
  dataIndex: 'unitPrice',
  render: (value: number, record: ProcessStep) => (
    <InputNumber
      value={value}
      onChange={(val) => handleChange(record.processCode, 'unitPrice', val)}
    />
  )
}
```

#### 防止数组排序导致的UI问题
```tsx
// ❌ 错误：添加后立即排序
const newItem = { processCode: '', ... };
setItems([...items, newItem]);
normalizeItems(); // 会导致新行跳到第一位

// ✅ 正确：计算下一个序号
const maxCode = Math.max(...items.map(i => Number(i.processCode) || 0), 0);
const newItem = { processCode: String(maxCode + 1).padStart(2, '0'), ... };
setItems([...items, newItem]); // 不调用normalize
```

### 3. 打印模板特殊处理

打印模板表格必须使用纯HTML：
```tsx
// 生成打印内容
const printContent = `
  <table style="width: 100%; border-collapse: collapse;">
    <thead>
      <tr>
        <th style="border: 1px solid #000;">序号</th>
        <th style="border: 1px solid #000;">内容</th>
      </tr>
    </thead>
    <tbody>
      ${data.map((item, i) => `
        <tr>
          <td style="border: 1px solid #000;">${i + 1}</td>
          <td style="border: 1px solid #000;">${item.content}</td>
        </tr>
      `).join('')}
    </tbody>
  </table>
`;

// 打开新窗口打印
const win = window.open('', '_blank');
win.document.write(printContent);
win.print();
```

# 用户审批功能使用指南

## 功能概述
用户审批功能允许管理员审核新注册的用户，确保只有经过批准的用户才能登录系统。

## 功能特点
- ✅ **双端支持**：PC端和小程序端都可以审批用户
- ✅ **实时提醒**：小程序个人中心显示待审批用户数量角标
- ✅ **批准/拒绝**：可以批准或拒绝新用户，拒绝时可填写原因
- ✅ **登录限制**：待审批和已拒绝的用户无法登录系统
- ✅ **权限控制**：仅管理员和主管可以审批用户

---

## PC端使用方法

### 1. 访问用户审批页面
1. 使用管理员账号登录PC端（http://localhost:5174）
2. 点击左侧菜单 **系统设置** → **用户审批**
3. 进入用户审批页面

### 2. 查看待审批用户
- 页面会显示所有待审批用户的列表
- 包含用户名、姓名、角色、手机号、邮箱、注册时间等信息
- 顶部显示当前待审批用户总数

### 3. 批准用户
1. 找到要批准的用户
2. 点击操作列的 **批准** 按钮
3. 在确认对话框中点击 **批准**
4. 批准成功后，该用户可以正常登录系统

### 4. 拒绝用户
1. 找到要拒绝的用户
2. 点击操作列的 **拒绝** 按钮
3. 在弹出的对话框中输入拒绝原因（必填）
4. 点击 **确定拒绝**
5. 拒绝后，该用户无法登录，登录时会看到拒绝原因

### 5. 刷新列表
点击页面右上角的 **刷新** 按钮可以重新加载待审批用户列表

## 小程序端使用方法

### 1. 进入用户审批页面
1. 使用管理员账号登录微信小程序
2. 点击底部导航栏的 **个人** 标签
3. 在"管理功能"区域点击 **用户审批** 卡片
   - 如果有待审批用户，卡片右侧会显示红色数字角标
   - 如果不是管理员，不会看到这个卡片

### 2. 查看待审批用户
- 页面显示所有待审批用户的卡片列表
- 每个用户卡片包含：
  - 用户姓名/用户名
  - 角色、手机号、邮箱
  - 注册时间
- 顶部显示当前待审批用户总数

### 3. 批准用户
1. 找到要批准的用户卡片
2. 点击 **批准** 按钮（绿色）
3. 在确认对话框中点击 **批准**
4. 批准成功后，该用户可以正常登录

### 4. 拒绝用户
1. 找到要拒绝的用户卡片
2. 点击 **拒绝** 按钮（红色边框）
3. 在弹出的对话框中输入拒绝原因
4. 点击 **确定拒绝**
5. 拒绝后，该用户无法登录

### 5. 下拉刷新
在用户审批页面下拉可以刷新列表

### 6. 加载更多
滚动到页面底部自动加载更多待审批用户

## 用户状态说明

### 审批状态
- **pending（待审批）**：新注册的用户，需要管理员审核
- **approved（已批准）**：管理员已批准，可以正常登录
- **rejected（已拒绝）**：管理员已拒绝，无法登录

### 账号状态
- **active（启用）**：账号正常可用
- **inactive（停用）**：账号被停用

### 状态关系
- 批准用户时：`approval_status` → approved，`status` → active
- 拒绝用户时：`approval_status` → rejected，`status` → inactive

## 用户登录提示

### 待审批用户登录
当待审批用户尝试登录时，会看到提示：
```
您的账号正在审批中，请耐心等待管理员审核
```

### 已拒绝用户登录
当已拒绝用户尝试登录时，会看到提示：
```
您的账号已被拒绝，原因：[管理员填写的拒绝原因]
```

## 注意事项

### 1. 权限要求
- ✅ 只有**管理员**和**主管**可以审批用户
- ❌ 普通员工无法访问用户审批功能

### 2. 新用户注册
- 现有用户的 `approval_status` 默认为 `approved`（已批准）
- 新注册的用户可以通过PC端"人员管理"中添加
- 未来可以添加用户自助注册功能，新注册用户自动设为 `pending`

### 3. 批准后的用户
- 批准后，用户可以立即登录系统
- 用户的角色权限由其分配的角色决定
- 建议在批准用户前，先在PC端为用户分配合适的角色

### 4. 拒绝原因
- 拒绝用户时建议填写明确的原因
- 拒绝原因会显示给用户，帮助用户了解被拒绝的理由
- 常见拒绝原因示例：
  - "信息不完整，请补充完整信息后重新申请"
  - "不符合员工注册要求"
  - "重复注册，请使用原账号登录"

## 常见问题

### Q1: 如何查看已批准/已拒绝的用户？
A: 在PC端"人员管理"页面可以查看所有用户及其审批状态。可以通过筛选条件查看不同状态的用户。

### Q2: 可以撤销审批决定吗？
A: 可以。在PC端"人员管理"页面：
- 已拒绝用户：可以重新编辑用户，将状态改为"启用"，审批状态改为"已批准"
- 已批准用户：可以通过"停用"按钮停用用户

### Q3: 小程序为什么看不到用户审批入口？
A: 用户审批功能仅对管理员和主管开放。请确认：
1. 您的账号角色是否为"管理员"或"主管"
2. 在小程序个人中心是否显示了"管理功能"区域
3. 如果看不到，请联系其他管理员为您分配管理员角色

### Q4: 批准用户后，用户还是无法登录？
A: 请检查：
1. 用户的"状态"是否为"启用"（应该是绿色的"启用"标签）
2. 用户是否分配了角色
3. 用户的密码是否正确

### Q5: 待审批用户数量不更新？
A: 小程序端的待审批数量在以下情况会刷新：
- 进入个人中心页面时
- 下拉刷新个人中心时
- 审批完成后返回个人中心时

PC端的待审批数量需要手动点击刷新按钮更新。

## 技术说明

### 数据库字段
```sql
-- t_user 表新增字段
approval_status VARCHAR(20) NULL DEFAULT 'approved' COMMENT '审批状态: pending, approved, rejected'
approval_time DATETIME NULL COMMENT '审批时间'
approval_remark VARCHAR(500) NULL COMMENT '审批备注'
```

### API接口
```
GET  /api/system/user/pending          # 获取待审批用户列表
POST /api/system/user/{id}/approve     # 批准用户
POST /api/system/user/{id}/reject      # 拒绝用户
```

### 权限要求
所有审批相关接口都需要管理员权限（ROLE_admin, ROLE_ADMIN, ROLE_1）

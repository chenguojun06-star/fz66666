## 🏗️ 架构模式总结

### 1. **Hooks模式** (数据 + 操作分离)

```typescript
// 数据Hook - 专注数据获取和状态管理
const { loading, data, fetchData } = useXXXData();

// 操作Hook - 专注业务逻辑和副作用
const { handleSave, handleDelete } = useXXXActions(refreshCallback);
```

**优势**:
- ✅ 职责清晰：数据与操作分离
- ✅ 复用性强：可在多个组件中使用
- ✅ 测试友好：独立测试数据逻辑和操作逻辑

### 2. **组件模式** (UI + 交互)

```typescript
// 单一职责组件 - 清晰的Props接口
interface ComponentProps {
  data: DataType[];
  onAction: (id: string) => void;
  loading?: boolean;
}

const Component: React.FC<ComponentProps> = ({ data, onAction, loading }) => {
  // 仅包含UI渲染和事件处理
  return <UI />;
};
```

**优势**:
- ✅ 可复用：独立组件可跨页面使用
- ✅ 可维护：每个组件职责单一
- ✅ 可测试：Props驱动，易于单元测试

### 3. **主文件模式** (组合 + 编排)

```typescript
const MainPage = () => {
  // 1. Hooks (数据 + 操作)
  const { data, loading } = useXXXData();
  const { handleAction } = useXXXActions(fetchData);
  
  // 2. 本地状态 (UI状态)
  const [modalOpen, setModalOpen] = useState(false);
  
  // 3. 事件处理 (简单回调)
  const handleClick = () => { setModalOpen(true); };
  
  // 4. 渲染 (组件组合)
  return (
    <Layout>
      <FilterPanel onSearch={...} />
      <DataTable data={data} onAction={handleAction} />
      <Modal open={modalOpen} />
    </Layout>
  );
};
```

**特点**:
- ✅ 主文件 <500行
- ✅ 无复杂逻辑（逻辑在Hooks）
- ✅ 无大段UI（UI在组件）
- ✅ 仅负责组合和编排

## 🔢 虚拟数据说明

### 为什么使用虚拟数据？
当前所有3个组件都使用虚拟数据进行UI测试，后端API已实现但暂未连接。

### 虚拟数据规则

**订单裁剪图**：
```typescript
dates: 最近30天（MM-DD格式）
orderQuantities: [3000-5000随机] × 30
cuttingQuantities: [2500-4300随机] × 30
```

**扫菲对比图**：
```typescript
dates: 最近30天（MM-DD格式）
scanCounts: [80-130随机] × 30
scanQuantities: [5000-8000随机] × 30
```

**延期订单表**：
```typescript
共25条记录
orderNo: PO2026{月}{序号}  // 如 PO202601015
styleNo: ST001-ST100随机
quantity: 200-1000随机
deliveryDate: 最近30天随机日期
overdueDays: 1-20随机
```

### 如何切换到真实API？
在每个组件的 `loadData()` 函数中：
1. 删除虚拟数据生成代码
2. 取消注释API调用代码
3. 重启前端服务

```typescript
// 删除这部分
const mockDates = Array.from({ length: 30 }, ...);

// 取消注释这部分
const result = await api.get('/api/dashboard/...');
```

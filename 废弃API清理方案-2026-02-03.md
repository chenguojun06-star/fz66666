### 步骤 2：删除后端废弃端点（建议 2026-05-01 执行）

创建清理脚本 `scripts/remove-deprecated-apis.sh`：

```bash
#!/bin/bash
# 删除标记为 @Deprecated 的方法

echo "⚠️  警告：此脚本将永久删除 58 个废弃 API 端点"
echo "请确保："
echo "1. 前端 legacyApiAdapter.ts 已验证正常工作"
echo "2. 所有客户端（PC/小程序）已更新到新版本"
echo "3. 已完成数据备份"
echo ""
read -p "确认继续？(输入 YES 继续): " confirm

if [ "$confirm" != "YES" ]; then
    echo "❌ 取消操作"
    exit 1
fi

# 备份代码
echo "📦 备份代码到 backup-before-api-cleanup-$(date +%Y%m%d).tar.gz"
tar -czf backup-before-api-cleanup-$(date +%Y%m%d).tar.gz backend/

# 列出所有废弃方法
echo "🔍 查找废弃方法..."
find backend/src/main/java -name "*Controller.java" -exec grep -l "@Deprecated" {} \; > deprecated-files.txt

echo "✅ 找到 $(wc -l < deprecated-files.txt) 个包含废弃方法的文件"
echo "📋 文件列表已保存到 deprecated-files.txt"
echo ""
echo "⚠️  请手动审查并删除标记为 @Deprecated 的方法"
echo "💡 提示：保留方法的注释文档，说明迁移路径"
```

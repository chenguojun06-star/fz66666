## 🟡 中优先级：Modal 组件统一性问题

### 发现：35+ 文件使用 Modal，但有 **5 种不同模式**

#### 模式1：自定义 Modal 布局组件（仅2个文件使用）

```tsx
// 📍 Cutting/index.tsx (9次使用)
// 📍 MaterialPurchaseDetail/index.tsx (3次使用)

<Modal ...>
  <ModalHeaderCard isMobile={isMobile}>
    <ModalSideLayout>
      <ModalVerticalStack gap={12}>
        <ModalPrimaryField label="款号" value="ST001" />
        <ModalFieldRow gap={24}>
          <ModalField label="颜色" value="黑色" />
          <ModalField label="尺码" value="M" />
        </ModalFieldRow>
      </ModalVerticalStack>
    </ModalSideLayout>
  </ModalHeaderCard>
</Modal>
```

**组件清单**：
- `ModalHeaderCard` - Modal头部卡片容器
- `ModalSideLayout` - 左右布局
- `ModalVerticalStack` - 垂直堆叠容器
- `ModalPrimaryField` - 主要字段（大号文字）
- `ModalField` - 普通字段
- `ModalFieldRow` - 横向字段组
- `ModalFieldGrid` - 网格布局

**问题**：
- ✅ 组件设计优秀（响应式、语义化）
- ⚠️ 但仅在2个文件中使用（利用率低）
- ⚠️ 无文档说明
- ⚠️ 其他33个文件不知道这些组件存在

---

#### 模式2：useModal 自定义Hook（10+ 文件）

```tsx
// 使用文件：MaterialPurchase, SampleInventory, UserList, 
//         FactoryList, RoleList, DictManage等

const quickEditModal = useModal<ProductionOrder>();
const logModal = useModal();

// 使用
quickEditModal.open(record);
quickEditModal.close();
```

**优点**：
- ✅ 类型安全（支持泛型）
- ✅ API简洁
- ✅ 多个文件使用

**问题**：
- ⚠️ 但仍有25个文件不使用此Hook
- ⚠️ 导致状态管理不统一

---

#### 模式3：useState 管理 Modal 可见性（15+ 文件）

```tsx
const [printModalVisible, setPrintModalVisible] = useState(false);
const [rejectModalVisible, setRejectModalVisible] = useState(false);

<Modal
  open={printModalVisible}
  onCancel={() => setPrintModalVisible(false)}
/>
```

**问题**：
- ⚠️ 与 useModal 模式并存
- ⚠️ 命名不统一（xxxModalVisible, xxxModalOpen, modalVisible等）
- ⚠️ 每个Modal需要单独管理状态

---

#### 模式4：useViewport Hook 响应式宽度（8+ 文件）

```tsx
const { modalWidth, isMobile } = useViewport();

<Modal width={modalWidth} ...>
```

**优点**：
- ✅ 响应式设计

**问题**：
- ⚠️ 但仍有27个文件硬编码宽度或不使用此Hook

---

#### 模式5：Modal.useModal() API（1个文件）

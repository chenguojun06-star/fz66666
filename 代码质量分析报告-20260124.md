### 关键问题分析

#### 1. `any` 类型泛滥（2300个）

**影响**:
- 失去 TypeScript 类型保护
- 潜在运行时错误风险
- IDE 无法提供智能提示

**修复策略**:
1. **P0 - API 调用和数据处理**（优先）
   - 为所有 API 响应定义类型接口
   - 为表单数据定义类型
   - 为组件 props 定义类型

2. **P1 - 事件处理和回调**（次要）
   - 使用 React 内置类型（如 `React.MouseEvent`）
   - 定义回调函数的参数类型

3. **P2 - 工具函数**（可选）
   - 为工具函数添加泛型支持
   - 使用 `unknown` 替代 `any`

**示例**:
```typescript
// ❌ 错误
const handleSubmit = (values: any) => {
  // ...
}

// ✅ 正确
interface FormValues {
  orderNo: string;
  quantity: number;
  // ...
}

const handleSubmit = (values: FormValues) => {
  // ...
}
```

#### 2. useEffect 依赖问题（78个）

**影响**:
- 可能导致内存泄漏
- 组件未正确更新
- 性能问题

**修复策略**:
1. 添加缺失的依赖
2. 使用 `useCallback` 稳定函数引用
3. 必要时使用 `eslint-disable-next-line` 并添加注释说明

**示例**:
```typescript
// ❌ 错误
useEffect(() => {
  fetchData(id);
}, []); // 缺少 id 依赖

// ✅ 正确
useEffect(() => {
  fetchData(id);
}, [id, fetchData]);
```

```bash
# 4. 检查未使用导出（需要先安装 ts-prune）
npm install -D ts-prune
npm run check:unused
```

**预期输出**:
- ESLint 详细报告
- 未使用依赖列表
- 循环依赖列表
- 未使用导出列表

### 【第二阶段】优先级修复（预计 3-5 天）

#### 步骤 2.1: P0 问题修复（高优先级）
**目标**: 修复影响稳定性和性能的关键问题

**前端 P0**:
1. 修复 `any` 类型中的高风险部分（API 调用、数据处理）
2. 修复 `useEffect` 依赖警告（可能导致内存泄漏）
3. 移除未使用的依赖（减少打包体积）

**后端 P0**:
1. 优化复杂度 > 80 分的 Orchestrator
2. 修复 Checkstyle/SpotBugs 错误级别问题
3. 优化慢查询（> 1s）

**执行方式**:
- 每天修复 50-100 个警告
- 每修复一批，运行测试确保功能正常
- 提交时使用清晰的 commit message

---

#### 步骤 2.2: P1 问题修复（中优先级）
**目标**: 提升代码质量和可维护性

**前端 P1**:
1. 修复剩余的 `any` 类型（非关键路径）
2. 优化导入（移除未使用的导入）
3. 统一代码风格

**后端 P1**:
1. 重构中等复杂度的 Orchestrator（50-80分）
2. 优化数据库查询（N+1 问题）
3. 添加必要的单元测试

# 动态工序列实现方案

## 背景
用户需求：在"我的订单"页面支持显示多人车缝、多个流水工序，而不是固定的工序列。

## 当前实现
**固定列方式**（frontend/src/pages/Production/List.tsx Line 643-649）：
```typescript

...stageColumns('carSewing', { start: '车缝开始', end: '车缝完成', operator: '车缝员', rate: '车缝完成率' }),

...stageColumns('packaging', { start: '包装开始', end: '包装完成', operator: '包装员', rate: '包装完成率' }),
...stageColumns('quality', { start: '质检时间', end: '质检完成', operator: '质检员', rate: '质检完成率' }),
...stageColumns('warehousing', { start: '入库时间', end: '入库完成', operator: '入库员', rate: '入库完成率' }),
```

## 数据库支持
订单表有 `progress_workflow_json` 字段存储工序配置：
```json
{

    {"id": "purchase", "unitPrice": 0, "name": "采购"},
    {"id": "cutting", "unitPrice": 0, "name": "裁剪"},
```

## 解决方案

### 方案1：基于第一条订单的工序配置动态生成列（简单）

**实现步骤**：

1. 在 `fetchProductionList` 成功后，解析第一条订单的 `progressWorkflowJson`
2. 根据解析出的工序节点动态生成 stageColumns
3. 将动态列合并到固定列中

**代码示例**：
```typescript
// 在 List.tsx 中添加状态
const [dynamicStageColumns, setDynamicStageColumns] = useState<any[]>([]);

// 解析工序配置
const parseWorkflowNodes = (workflowJson: string | null | undefined) => {
  if (!workflowJson) return [];
  try {
    const workflow = JSON.parse(workflowJson);
    return workflow.nodes || [];
  } catch (e) {
    return [];
  }
};

// 在 fetchProductionList 中添加
const fetchProductionList = async () => {
  setLoading(true);
  try {
    const response = await api.get<PaginatedResponse<ProductionOrder>>(
      '/production/order/list',
      { params: queryParams }
    );
    if (isApiSuccess(response)) {
      const records = response.data.records || [];
      setProductionList(records);
      setTotal(response.data.total || 0);
      
      // 🔥 动态生成工序列
      if (records.length > 0) {
        const firstOrder = records[0];
        const nodes = parseWorkflowNodes(firstOrder.progressWorkflowJson);
        const dynamicCols = nodes.map((node: any) => {
          const fieldPrefix = node.id; // 例如 'sewing', 'cutting'
          const stageName = node.name; // 例如 '车缝', '裁剪'
          
          return stageColumns(
            fieldPrefix,
            {
              start: `${stageName}开始`,
              end: `${stageName}完成`,
              operator: `${stageName}员`,
              rate: `${stageName}完成率`
            }
          );
        }).flat();
        
        setDynamicStageColumns(dynamicCols);
      }
    }
  } catch (error) {
    message.error('获取生产订单列表失败');
  } finally {
    setLoading(false);
  }
};

// 修改 columns 定义
const columns = [
  // ... 前面的固定列
  {
    title: '下单时间',
    dataIndex: 'createTime',
    key: 'createTime',
    width: 170,
    render: renderStageTime,
  },
  // 🔥 使用动态工序列
  ...dynamicStageColumns,
  // ... 后面的固定列（裁剪数量、完成数量等）
];
```

**优点**：
- 简单易实现
- 根据实际订单的工序配置显示列
- 无需手动维护工序列表

**缺点**：
- 不同订单如果工序不同，可能会导致列不匹配
- 依赖第一条订单的配置

### 方案2：基于所有订单的工序并集动态生成列（完整）

**实现思路**：
1. 遍历所有订单的 `progressWorkflowJson`
2. 收集所有出现过的工序节点（去重）
3. 按固定顺序排列工序
4. 生成动态列

**代码示例**：
```typescript
const collectAllWorkflowNodes = (orders: ProductionOrder[]) => {
  const nodeMap = new Map<string, { id: string; name: string; order: number }>();
  
  // 定义工序优先级
  const stageOrder: Record<string, number> = {
    'procurement': 1,
    'cutting': 2,
    'sewing': 3,
    'carSewing': 4,
    'ironing': 5,
    'pressing': 5,
    'quality': 6,
    'packaging': 7,
    'warehousing': 8,
  };
  
  orders.forEach(order => {
    const nodes = parseWorkflowNodes(order.progressWorkflowJson);
    nodes.forEach((node: any) => {
      if (!nodeMap.has(node.id)) {
        nodeMap.set(node.id, {
          id: node.id,
          name: node.name,
          order: stageOrder[node.id] || 999
        });
      }
    });
  });
  
  // 按优先级排序
  return Array.from(nodeMap.values()).sort((a, b) => a.order - b.order);
};

const fetchProductionList = async () => {
  // ... 获取数据
  
  if (records.length > 0) {
    const allNodes = collectAllWorkflowNodes(records);
    const dynamicCols = allNodes.map(node => {
      return stageColumns(
        node.id,
        {
          start: `${node.name}开始`,
          end: `${node.name}完成`,
          operator: `${node.name}员`,
          rate: `${node.name}完成率`
        }
      );
    }).flat();
    
    setDynamicStageColumns(dynamicCols);
  }
};
```

### 方案3：支持多人车缝工序（最灵活）

如果一个订单有多个车缝工序（例如 `carSewing1`, `carSewing2`, `carSewing3`），需要：

**数据库扩展**：
```json
{
  "nodes": [
    {"id": "cutting", "name": "裁剪"},
    {"id": "carSewing1", "name": "车缝-做领", "unitPrice": 5},
    {"id": "carSewing2", "name": "车缝-上领", "unitPrice": 3},
    {"id": "carSewing3", "name": "车缝-袖子", "unitPrice": 4},
    {"id": "carSewing4", "name": "车缝-前片", "unitPrice": 3},
    {"id": "ironing", "name": "大烫"}
  ]
}
```

**后端支持**：
需要在 `ProductionOrder` 实体添加对应的字段（或使用 JSON 字段动态存储）：
```java
// 方式1：预定义多个车缝字段
private LocalDateTime carSewing1StartTime;
private LocalDateTime carSewing1EndTime;
private String carSewing1OperatorName;
private Integer carSewing1CompletionRate;

private LocalDateTime carSewing2StartTime;
// ... 等等

// 方式2：使用JSON存储（推荐）
@TableField("progress_data_json")
private String progressDataJson; // 存储所有工序的实际数据
```

**JSON 存储示例**：
```json
{
  "cutting": {
    "startTime": "2026-01-24 10:00:00",
    "endTime": "2026-01-24 12:00:00",
    "operatorName": "张三",
    "completionRate": 100
  },
  "carSewing1": {
    "startTime": "2026-01-24 13:00:00",
    "endTime": "2026-01-24 14:00:00",
    "operatorName": "李四",
    "completionRate": 100
  },
  "carSewing2": {
    "startTime": "2026-01-24 14:00:00",
    "endTime": null,
    "operatorName": "王五",
    "completionRate": 50
  }
}
```

**前端动态渲染**：
```typescript
const columns = [
  // ... 固定列
  ...dynamicStageColumns, // 根据 progressWorkflowJson 动态生成
  // ... 固定列
];

// stageColumns 函数需要支持从 progressDataJson 读取数据
const stageColumns = (nodeId: string, titles: {...}) => {
  return [
    {
      title: titles.start,
      dataIndex: `progressData.${nodeId}.startTime`, // 读取 JSON 字段
      key: `${nodeId}StartTime`,
      width: 170,
      render: (value: unknown, record: ProductionOrder) => {
        const progressData = JSON.parse(record.progressDataJson || '{}');
        const nodeData = progressData[nodeId];
        return nodeData?.startTime ? formatDateTime(nodeData.startTime) : '-';
      },
    },
    // ... 其他列类似处理
  ];
};
```

## 推荐实现顺序

1. **先实现方案1**：基于第一条订单动态生成列（快速验证）
2. **如果需要更完整**：实现方案2（收集所有工序并集）
3. **如果需要多人流水线**：
   - 后端添加 `progress_data_json` 字段
   - 修改扫码提交工序数据时，更新 JSON
   - 前端从 JSON 动态读取和展示

## 实施建议

- **短期方案**：使用方案1，快速支持基本的动态工序显示
- **长期方案**：采用 JSON 存储方式，支持任意数量的流水工序，配合小程序扫码系统动态记录每个工序的进度

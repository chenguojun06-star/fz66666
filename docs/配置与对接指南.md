## 一、AI 接口对接

### 1.1 OpenAI 兼容 API

任何支持 OpenAI API 的客户端都可以直接对接，包括 ChatBox、Open WebUI、Cursor、Cherry Studio 等。

**端点**

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/v1/health` | 健康检查（无需认证） |
| GET | `/v1/models` | 模型列表 |
| POST | `/v1/chat/completions` | 对话（支持流式/非流式） |

**环境变量**

| 变量名 | 必填 | 示例值 | 说明 |
|--------|:----:|--------|------|
| `AI_OPENAI_COMPAT_ENABLED` | 是 | `true` | 开关 |
| `AI_OPENAI_COMPAT_API_KEY` | 是 | `sk-webyszl-2026-prod-xxxxx` | 客户端鉴权密钥，自己编一个强字符串 |
| `AI_OPENAI_COMPAT_TENANT_ID` | 是 | `106` | 默认租户 ID（查 `t_tenant` 表） |
| `AI_OPENAI_COMPAT_MODEL_NAME` | 否 | `xiaoyun-agent` | 返回给客户端的模型名 |
| `AI_OPENAI_COMPAT_API_KEY_MAP` | 否 | `key-a=101,key-b=102` | 多租户多 Key 映射 |

**客户端配置示例**

```
API Base URL:  https://api.webyszl.cn/v1
API Key:       sk-webyszl-2026-prod-xxxxx
Model:         xiaoyun-agent
```

**请求示例**

```bash
# 非流式
curl -X POST https://api.webyszl.cn/v1/chat/completions \
  -H "Authorization: Bearer sk-webyszl-2026-prod-xxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "xiaoyun-agent",
    "messages": [{"role": "user", "content": "你好"}],
    "stream": false
  }'

# 流式 SSE
curl -N https://api.webyszl.cn/v1/chat/completions \
  -H "Authorization: Bearer sk-webyszl-2026-prod-xxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "xiaoyun-agent",
    "messages": [{"role": "user", "content": "你好"}],
    "stream": true
  }'
```

**ChatBox 配置截图指引**

1. 设置 → AI 服务商 → 选「OpenAI API 兼容」
2. API 域名填 `https://api.webyszl.cn`
3. API Key 填你设置的密钥
4. 模型名填 `xiaoyun-agent`

### 1.2 MCP 协议（Cursor / Claude Desktop）

通过 MCP（Model Context Protocol）让 AI 编程工具直接调用供应链 68+ 工具。

**端点**

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/intelligence/mcp/token` | 用 appKey + HMAC 签名换取 MCP Token |
| GET | `/api/intelligence/mcp/sse` | SSE 连接（需 MCP Token） |
| POST | `/api/intelligence/mcp/messages` | JSON-RPC 消息（需 MCP Token） |

**前置条件**

在系统后台「系统 → 开放API → 我的应用」中创建应用，获取 `appKey` 和 `appSecret`。

**换取 Token**

```bash
APP_KEY="你的appKey"
APP_SECRET="你的appSecret"
TIMESTAMP=$(python3 -c "import time; print(str(int(time.time()*1000)))")
SIGNATURE=$(echo -n "${TIMESTAMP}" | openssl dgst -sha256 -hmac "${APP_SECRET}" | awk '{print $NF}')

curl -X POST https://api.webyszl.cn/api/intelligence/mcp/token \
  -H "X-App-Key: ${APP_KEY}" \
  -H "X-Timestamp: ${TIMESTAMP}" \
  -H "X-Signature: ${SIGNATURE}"
```

返回：
```json
{
  "code": 200,
  "data": {
    "mcpToken": "eyJ...",
    "expiresIn": 7776000,
    "tokenType": "Bearer"
  }
}
```

**Cursor 配置**

在 Cursor 的 `.cursor/mcp.json` 中添加：

```json
{
  "mcpServers": {
    "小云供应链": {
      "url": "https://api.webyszl.cn/api/intelligence/mcp/sse?token=<mcpToken>"
    }
  }
}
```

**Claude Desktop 配置**

在 `claude_desktop_config.json` 中添加：

```json
{
  "mcpServers": {
    "xiaoyun": {
      "url": "https://api.webyszl.cn/api/intelligence/mcp/sse?token=<mcpToken>",
      "transport": "sse"
    }
  }
}
```

> ⚠️ MCP Token 有效期 90 天，过期需重新换取。

### 1.3 A2A 协议（Agent-to-Agent）

让其他 AI Agent 通过 A2A 协议委托任务给小云。

**端点**

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/.well-known/agent.json` | AgentCard 发现（无需认证） |
| POST | `/api/intelligence/a2a/token` | appKey + HMAC 换取 A2A Token |
| POST | `/api/intelligence/a2a/rpc` | JSON-RPC 2.0 任务委托 |

**换取 Token**（与 MCP 相同的签名算法）

```bash
APP_KEY="你的appKey"
APP_SECRET="你的appSecret"
TIMESTAMP=$(python3 -c "import time; print(str(int(time.time()*1000)))")
SIGNATURE=$(echo -n "${TIMESTAMP}" | openssl dgst -sha256 -hmac "${APP_SECRET}" | awk '{print $NF}')

curl -X POST https://api.webyszl.cn/api/intelligence/a2a/token \
  -H "X-App-Key: ${APP_KEY}" \
  -H "X-Timestamp: ${TIMESTAMP}" \
  -H "X-Signature: ${SIGNATURE}"
```

**JSON-RPC 调用示例**

```bash
# 列出可用工具
curl -X POST https://api.webyszl.cn/api/intelligence/a2a/rpc \
  -H "Authorization: Bearer <a2aToken>" \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/list"}'

# 发送任务
curl -X POST https://api.webyszl.cn/api/intelligence/a2a/rpc \
  -H "Authorization: Bearer <a2aToken>" \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tasks/send","params":{"skill":"order_query","arguments":{"orderNo":"PO20260501001"}}}'

# 查询任务状态
curl -X POST https://api.webyszl.cn/api/intelligence/a2a/rpc \
  -H "Authorization: Bearer <a2aToken>" \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":3,"method":"tasks/get","params":{"taskId":"task-xxx"}}'
```

## 二、IM 平台 AI 助手对接

### 2.1 微信公众号 AI 助手

用户在微信公众号中发消息，小云自动回复供应链相关建议。

**环境变量**

| 变量名 | 必填 | 说明 |
|--------|:----:|------|
| `AI_WECHAT_AI_ENABLED` | 是 | `true` 开启 |
| `AI_WECHAT_AI_VERIFY_TOKEN` | 是 | 微信后台配置的 Token |
| `AI_WECHAT_AI_APP_ID` | 是 | 微信公众号 AppID |
| `AI_WECHAT_AI_APP_SECRET` | 是 | 微信公众号 AppSecret |
| `AI_WECHAT_AI_AES_KEY` | 否 | 消息加解密密钥（安全模式必填） |
| `AI_WECHAT_AI_ASYNC_REPLY` | 否 | `true`（默认）先回复"思考中"，再通过客服消息推送 |

**微信公众号后台配置**

1. 登录 [微信公众平台](https://mp.weixin.qq.com)
2. 设置与开发 → 基本配置
3. 服务器配置：
   - URL：`https://api.webyszl.cn/api/intelligence/wechat-ai/callback`
   - Token：与 `AI_WECHAT_AI_VERIFY_TOKEN` 一致
   - EncodingAESKey：与 `AI_WECHAT_AI_AES_KEY` 一致（或随机生成后同步）
   - 消息加解密方式：安全模式（推荐）
4. 点击「启用」

**用户绑定**

用户需在系统中绑定微信 OpenID（`t_user.openid` 字段），否则小云无法识别身份。

### 2.2 飞书 AI 助手

**环境变量**

| 变量名 | 必填 | 说明 |
|--------|:----:|------|
| `AI_IM_AI_FEISHU_ENABLED` | 是 | `true` 开启 |
| `AI_IM_AI_FEISHU_VERIFICATION_TOKEN` | 是 | 飞书事件订阅验证 Token |
| `AI_IM_AI_FEISHU_ENCRYPT_KEY` | 否 | 飞书事件加密密钥 |

**飞书开放平台配置**

1. 登录 [飞书开放平台](https://open.feishu.cn)
2. 创建企业自建应用
3. 事件订阅：
   - 请求地址：`https://api.webyszl.cn/api/intelligence/im-ai/feishu/callback`
   - 验证 Token：与 `AI_IM_AI_FEISHU_VERIFICATION_TOKEN` 一致
   - Encrypt Key：与 `AI_IM_AI_FEISHU_ENCRYPT_KEY` 一致
4. 添加事件：`im.message.receive_v1`（接收消息）
5. 权限：`im:message`、`im:message:send_as_bot`
6. 发布版本

**用户绑定**

用户需在系统中绑定飞书 OpenID，格式为 `feishu:<open_id>`，存储在 `t_user.openid` 字段。

**钉钉开放平台配置**

## 五、安全与鉴权

| `/api/intelligence/wechat-ai/callback` | 微信 SHA-1 签名 |
| `/api/intelligence/im-ai/feishu/callback` | 飞书 HMAC-SHA256 签名 |
| `/api/intelligence/im-ai/dingtalk/callback` | 钉钉 HMAC-SHA256 签名 |
| `/api/intelligence/ai-advisor/chat/stream` | 自建鉴权 |
| `/openapi/**` | appKey + HMAC 签名 |
| `/api/webhook/**` | 各平台签名验证 |
| `/.well-known/agent.json` | 无需认证 |
| `/v1/health` | 无需认证 |
| `/actuator/health` | 无需认证 |

以下端点完全公开（登录/注册）：

| 路径 | 说明 |
|------|------|

### 5.2 HMAC 签名算法

MCP Token、A2A Token、开放 API 均使用相同的签名算法：

```
签名 = HMAC-SHA256(appSecret, timestamp字符串)
结果 = 小写十六进制
```

**Python 示例**

```python
import hmac, hashlib, time

app_secret = "你的appSecret"
timestamp = str(int(time.time() * 1000))
signature = hmac.new(
    app_secret.encode('utf-8'),
    timestamp.encode('utf-8'),
    hashlib.sha256
).hexdigest()
```

**请求头**

```
X-App-Key:    你的appKey
X-Timestamp:  1714700000000
X-Signature:  a1b2c3d4e5f6...
```

## 六、Nginx 代理规则

前端 Nginx 对不同路径有特殊处理：

| 路径 | 超时 | 特殊配置 |
|------|------|---------|
| `/v1/chat/completions` | 300s | `proxy_buffering off`（SSE 流式） |
| `/api/intelligence/ai-advisor/chat/stream` | 180s | `proxy_buffering off`（SSE 流式） |
| `/api/` | 60s | 常规反向代理 |
| `/ws/` | 3600s | WebSocket `Upgrade` |
| 静态资源 `.js/.css/...` | — | `Cache-Control: public, immutable, max-age=30d` |
| `/index.html` | — | `Cache-Control: no-store, no-cache, must-revalidate` |

> ⚠️ 静态资源缓存 30 天，发布新版本后 Vite 会自动生成新文件名（hash），无需手动清缓存。但 `index.html` 不缓存，确保总是加载最新入口。

# 单价维护问题修复总结

**修复时间**: 2026-02-02  
**问题类型**: 模板库工序单价字段缺失  
**影响范围**: 【基础资料】→【单价维护】页面数据不显示

---

## 🔍 问题原因

### 1. 数据结构不匹配

**前端期待的数据结构**：
```json
{
  "steps": [
    {"processCode": "01", "processName": "裁剪", "unitPrice": 5.0},
    {"processCode": "02", "processName": "缝制", "unitPrice": 8.0}
  ]
}
```

**数据库实际存储**：
```json
{
  "steps": [
    {"processCode": "01", "processName": "裁剪", "machineType": ""},
    {"processCode": "02", "processName": "缝制", "machineType": ""}
  ]
}
```

❌ **缺少 `unitPrice` 字段**，导致前端无法显示单价数据

### 2. 字段名称不统一

部分模板使用 `price` 字段而非标准的 `unitPrice` 字段。

## 📝 空模板处理指南

### 查询空模板列表
```sql
SELECT id, template_name, source_style_no
FROM t_template_library
WHERE template_type = 'process'
AND template_content = '{"steps":[]}';
```

### 处理建议

**方案1：删除无用模板**
```sql
-- 删除空模板（确认无用后）
DELETE FROM t_template_library
WHERE template_type = 'process'
AND template_content = '{"steps":[]}'
AND source_style_no = 'ST20260127002';  -- 指定款号
```

**方案2：在前端页面手动编辑**
1. 进入【基础资料】→【单价维护】
2. 找到空模板，点击"编辑"
3. 添加工序及单价
4. 保存

**方案3：从其他款号复制模板**
```sql
-- 复制模板内容
UPDATE t_template_library t1
INNER JOIN t_template_library t2 
  ON t2.source_style_no = 'HHY00322'  -- 源款号
  AND t2.template_type = 'process'
SET t1.template_content = t2.template_content
WHERE t1.id = 'xxx-空模板ID-xxx';
```

# 集成总览 vs 应用管理 功能对比

> **页面位置**：系统管理 → 客户应用管理
> **包含Tab**：6个（集成总览、应用管理、租户管理、角色模板、注册审批、使用教程）

---

## 🎯 核心区别（一句话说明）

| Tab | 本质 | 用途 |
|-----|------|------|
| **集成总览** | 📊 **仪表板**（只读） | 看整体对接情况，监控运行状态 |
| **应用管理** | ⚙️ **CRUD工具**（读写） | 创建/编辑/删除应用，管理密钥 |

**结论**：两个都需要！一个看，一个管。

---

## 📋 详细功能对比表

### 1️⃣ 集成总览 Tab (IntegrationOverviewTab)

#### 核心功能
- ✅ **监控面板**：4大模块对接状态（已对接/未对接）
- ✅ **统计卡片**：已配置应用数、运行中应用数、总API调用次数
- ✅ **模块卡片**：每个模块（下单、质检、物流、付款）的活跃应用数、累计调用数
- ✅ **API端点速查**：每个模块支持的API列表
- ✅ **最近调用日志**：最新10条API调用记录（时间轴展示）
- ✅ **数据入口跳转**：快速跳转到生产订单、质检记录、物流记录、财务对账页面

### 2️⃣ 应用管理 Tab (AppManagementTab)

#### 核心功能
- ✅ **创建应用**：填写应用名称、类型、Webhook地址，生成appKey和appSecret
- ✅ **编辑应用**：修改应用名称、Webhook地址、日调用上限
- ✅ **重置密钥**：重新生成appSecret（旧密钥立即失效）
- ✅ **停用/启用**：控制应用是否可以调用API
- ✅ **删除应用**：移除不再使用的应用
- ✅ **查看详情**：查看appKey、创建时间、到期时间、配置信息
- ✅ **查看日志**：查看该应用的所有API调用记录

#### 典型使用场景
1. **初次对接**：点击"创建应用"，输入应用名称和类型，获取appKey和appSecret
2. **配置Webhook**：编辑应用，填写客户系统的回调地址
3. **密钥轮换**：定期（如每6个月）重置密钥，提高安全性
4. **故障处理**：某个ERP系统出问题，暂时停用该应用
5. **多系统对接**：创建多个同类型应用（如2个ORDER_SYNC），对应不同ERP系统

#### CRUD操作示例
```typescript
// 创建应用
POST /api/tenant-app/create
{
  "appName": "国内ERP订单",
  "appType": "ORDER_SYNC",
  "webhookUrl": "http://erp-a.com/webhook",
  "dailyLimit": 1000
}

// 返回（appSecret仅此一次显示）
{
  "appKey": "os_1001_erp_a",
  "appSecret": "Kx9mPq2sT7vW..."  // ⚠️ 立即保存！
}

// 编辑应用
PUT /api/tenant-app/update/{id}
{
  "appName": "国内ERP订单（修改）",
  "webhookUrl": "http://new-erp.com/webhook",
  "dailyLimit": 2000
}

// 重置密钥
POST /api/tenant-app/reset-secret/{id}
// 返回新的appSecret

// 停用应用
PUT /api/tenant-app/update/{id}
{
  "status": "DISABLED"
}

// 删除应用
DELETE /api/tenant-app/delete/{id}
```

## 🔄 两者的协作关系

### 工作流程
```
1. 管理员在【应用管理】创建应用
         ↓
   获得 appKey 和 appSecret
         ↓
2. 配置到第三方ERP系统
         ↓
3. ERP系统开始调用API
         ↓
4. 在【集成总览】查看调用情况
         ↓
5. 发现问题 → 在【应用管理】查看详细日志
         ↓
6. 需要修改配置 → 在【应用管理】编辑应用
```

## 🎯 实际使用建议

### 对于管理员
1. **首次对接**：
   - 先在【应用管理】创建应用
   - 保存appKey和appSecret到安全位置
   - 配置到第三方系统
   
2. **日常监控**：
   - 每天查看【集成总览】，确保4个模块正常
   - 关注"最近API调用"，发现异常立即处理
   
3. **故障排查**：
   - 【集成总览】发现某模块调用失败
   - 切换到【应用管理】，找到对应应用
   - 点击"查看日志"，定位具体错误

### 对于开发者（第三方ERP开发）
1. **获取凭证**：联系管理员，从【应用管理】创建应用，获取appKey和appSecret
2. **查看文档**：切换到【使用教程】Tab，查看API对接指南
3. **测试调用**：使用appKey和appSecret调用API，观察是否成功
4. **查看日志**：让管理员在【应用管理】中查看该应用的调用日志

## ⚠️ 常见问题

### Q1: 为什么要分两个Tab，不能合并吗？
**A**: 设计思想不同：
- **集成总览** = 监控面板（给管理层看的）
- **应用管理** = 操作工具（给技术人员用的）
- 合并会导致界面混乱，操作和查看互相干扰

### Q2: 只用应用管理不行吗？
**A**: 不够直观：
- 应用管理是表格列表，看不出整体对接情况
- 集成总览是卡片视图，一眼看出4个模块是否正常
- 类比：应用管理是"文件夹详情"，集成总览是"文件资源管理器"

### Q3: 创建应用在哪个Tab？
**A**: 【应用管理】Tab

### Q4: 查看整体运行情况在哪个Tab？
**A**: 【集成总览】Tab

### Q5: 重置密钥在哪个Tab？
**A**: 【应用管理】Tab → 点击具体应用 → 重置密钥

### Q6: 查看某个模块的API端点在哪个Tab？
**A**: 
- 简单查看：【集成总览】Tab → API端点速查
- 详细教程：【使用教程】Tab

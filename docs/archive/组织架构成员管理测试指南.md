## 📝 测试场景

### 场景 1️⃣：创建部门并分配成员

**目标**：验证部门下的成员添加、查看、移出功能

1. **打开系统**
   ```
   网址：http://localhost:5173
   账号：admin
   密码：123456
   ```

2. **进入组织架构**
   - 左侧菜单 → `系统管理` → `组织架构`
   - 看到现有的部门树形结构

3. **新增测试部门**
   - 点击右上角 `新增部门` 按钮
   - 输入部门名称：`成员测试组_<当前时间>`
   - 上级部门：留空（创建顶级部门）
   - 内外标签：选`通用部门`
   - 点 `保存`
   - ✅ 验证：新部门出现在树形结构中

4. **新部门中添加成员**
   - 找到新创建的部门卡片
   - 点击右上角 `👥 添加成员` 按钮（绿色按钮）
   - 弹窗标题：`为「成员测试组_xxxxx」添加成员`
   - 在搜索框输入用户名（如 `admin`）
   - 点击用户行的 `分配` 按钮
   - ✅ 验证：
     - 弹窗关闭
     - 成员出现在部门下方（展开部门卡片查看）
     - 成员卡片显示头像、姓名、角色

5. **查看成员详情**
   - 验证成员卡片显示：
     - ✅ 用户头像（蓝色背景）
     - ✅ 用户姓名
     - ✅ 用户角色（或 `—` 未设置）

6. **移出成员**
   - 鼠标 hover 成员卡片
   - 出现右侧 `×` 按钮
   - 点击 `×`
   - 确认弹窗：点 `移出`
   - ✅ 验证：成员从部门列表消失

## 🐛 常见问题

### 问题：弹窗无法打开

**症状**：点 `👥 添加成员` 无反应

**排查**：
1. F12 → Console，查看 JS 错误
2. 检查 Network 标签， call `/api/system/organization/assignable-users` 是否 200
3. 如失败，检查后端日志：`tail -f backend/logs/fashion-supplychain.log | grep "assignable"`

### 问题：分配后成员不显示

**症状**：点分配后弹窗关闭，但成员不出现

**排查**：
1. 刷新页面（Cmd+R）
2. 检查 Network，验证 POST `/assign-member` 返回 200
3. 查看后端日志是否有错误

### 问题：生产页面工厂下拉为空

**症状**：生产订单页工厂筛选没有选项

**排查**：
1. 确保已创建至少一个工厂（组织架构 → 可看 🏭 工厂节点）
2. 刷新页面
3. 检查后端 `/api/production/factories/list` 是否返回数据

## 📊 API 端点验证（可选技术验证）

```bash
# 1. 登录获取 token
TOKEN=$(curl -s -X POST http://localhost:8088/api/system/user/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"123456"}' | grep -o '"token":"[^"]*"' | sed 's/"token":"\(.*\)"/\1/')

# 2. 获取可分配用户
curl -s -H "Authorization: Bearer $TOKEN" \
  -X GET http://localhost:8088/api/system/organization/assignable-users | jq

# 3. 分配成员
curl -s -H "Authorization: Bearer $TOKEN" \
  -X POST http://localhost:8088/api/system/organization/assign-member \
  -H "Content-Type: application/json" \
  -d '{"userId":"1","orgUnitId":"dept-001"}' | jq

# 4. 移出成员  
curl -s -H "Authorization: Bearer $TOKEN" \
  -X POST http://localhost:8088/api/system/organization/remove-member \
  -H "Content-Type: application/json" \
  -d '{"userId":"1"}' | jq
```

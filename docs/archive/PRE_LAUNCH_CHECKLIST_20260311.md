```markdown
# Redis 故障恢复手册

## 故障场景 1：Redis 容器宕机
- 现象：Token 认证延迟升高，但系统仍可用（熔断保护）
- 恢复步骤：
  1. 判断：`curl https://redis-status-api` 检查状态
  2. 手动重启：控制台 → my-redis → 重启
  3. 等待：~2分钟完全恢复
  4. 验证：Token 认证延迟 <5ms

## 故障场景 2：Redis 内存满溢
- 现象：新 Token 写入失败
- 恢复步骤：
  1. 清理策略自动触发（maxmemory-policy allkeys-lru）
  2. 或手动：`redis-cli FLUSHDB` 清理测试数据
  3. 监控内存不超过 80%

## 故障场景 3：Redis 和 Backend 网络断连
- 现象：认证转向 DB，性能下降 50%
- 恢复步骤：
  1. 核查网络策略（Security Group）
  2. 检查 Redis 服务健康
  3. 重启 Backend Pod
```

**优先级**：🟡 **建议** — 用于团队培训，不影响上线

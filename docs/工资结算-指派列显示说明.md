# 工资结算-指派列显示说明

## 新增列说明

在工资结算页面新增了两个列来显示工序指派信息：

### 1. **结算类型** 列

显示该工序的结算类型，使用不同颜色的标签：

| 类型 | 标签 | 说明 |
|------|------|------|
| 自己完成 | `灰色标签` | 未指派，员工自己完成的工序 |
| 内部指派 | `蓝色标签` | 指派给内部有账号的员工 |
| 外发工厂 | `橙色标签` | 指派给外部工厂（无系统账号） |

**悬停提示**：
- 如果实际操作员与结算对象不同，悬停会显示"由 XXX 代为操作"或"由 XXX 代为录入"

### 2. **指派对象** 列

显示被指派的人员或工厂名称：

| 情况 | 显示 | 样式 |
|------|------|------|
| 未指派 | `-` | 灰色 |
| 内部员工 | 员工姓名 | 蓝色加粗 |
| 外部工厂 | 工厂名称 | 橙色加粗 |

## 显示效果示例

### 示例1：自己完成
```
订单号: PO20260130001
人员: 张三
结算类型: [自己完成]
指派对象: -
工序名称: 裁剪
数量: 200
单价(元): 2.00
金额(元): 400.00
```

### 示例2：内部指派
```
订单号: PO20260130002
人员: 李四              ← 工资算给李四
结算类型: [内部指派] 🔵
指派对象: 李四           ← 蓝色显示
工序名称: 车缝
数量: 100
单价(元): 3.50
金额(元): 350.00
```

**如果悬停在 [内部指派] 标签上**：
- 如果李四自己扫的码：无提示
- 如果张三代为扫的码：显示"由 张三 代为操作"

### 示例3：外发工厂
```
订单号: PO20260130003
人员: 佳利工厂（外发）   ← 不计入内部工资
结算类型: [外发工厂] 🟠
指派对象: 佳利工厂       ← 橙色显示
工序名称: 刺绣
数量: 100
单价(元): 5.00
金额(元): 500.00          ← 记录外发成本，不发内部工资
```

**如果悬停在 [外发工厂] 标签上**：
- 显示"由 张三 代为录入"（如果是内部员工代为录入的）

## 数据来源

这些信息来自扫码记录的指派字段：

| 字段 | 说明 |
|------|------|
| `delegateTargetType` | `none`/`internal`/`external` |
| `delegateTargetName` | 被指派人或工厂名称 |
| `actualOperatorName` | 实际操作员（谁扫的码） |

## 筛选和统计

### 内部工资统计
- 只统计 `operator_id` 不为空的记录
- 排除外发工厂的记录（`delegateTargetType = 'external'`）

### 外发对账
- 单独统计 `delegateTargetType = 'external'` 的记录
- 需要在外发对账模块查看

## 使用场景

### 场景1：查看员工工资明细
1. 进入"工资结算"页面
2. 筛选人员：李四
3. 可以看到：
   - 李四自己完成的工序
   - 指派给李四的工序（由别人或自己完成）

### 场景2：查看外发成本
1. 进入"工资结算"页面
2. 结算类型列显示"外发工厂"的记录
3. 这些记录不计入内部工资
4. 到"外发对账"模块查看应付金额

### 场景3：审计追溯
1. 发现某笔工资有问题
2. 查看"结算类型"列
3. 如果是指派，悬停查看实际操作员
4. 可以追溯是谁录入的数据

## 列宽度说明

| 列名 | 宽度 | 说明 |
|------|------|------|
| 结算类型 | 130px | 显示标签+图标 |
| 指派对象 | 120px | 显示名称，可能较长 |

## 颜色规范

| 元素 | 颜色 | 色值 |
|------|------|------|
| 自己完成标签 | 灰色 | default |
| 内部指派标签 | 蓝色 | blue (#3b82f6) |
| 外发工厂标签 | 橙色 | orange (#f97316) |
| 未填充数据 | 浅灰 | #9ca3af |

## 注意事项

1. **历史数据**：迁移前的46条记录默认为"自己完成"
2. **权限控制**：不同角色看到的数据范围不同：
   - 财务：看全部
   - 主管：看团队
   - 员工：只看自己
3. **外发记录**：外发工厂的记录在"人员"列会显示工厂名+"（外发）"后缀

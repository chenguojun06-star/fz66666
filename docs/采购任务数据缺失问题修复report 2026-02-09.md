# 采购任务数据缺失问题修复报告

## 🔍 问题诊断

### 问题1：所有采购任务都没有 orderNo 字段
**现象**：
- floating-bell 加载任务时，`hasOrderNo: 0`（10个任务都没有 orderNo）  
- 前端无法通过orderNo查询采购单数据

**根本原因**：
- 数据库中 9 个采购任务的 `order_id` 和 `order_no` 都是 NULL
- 这些采购任务在创建时就没有关联订单（独立采购）

**SQL查询结果**：
```sql
SELECT COUNT(*) AS total, 
       SUM(CASE WHEN order_id IS NOT NULL THEN 1 ELSE 0 END) AS has_order_id
FROM t_material_purchase
WHERE status = 'received';

-- 结果：total=9, has_order_id=0
```

### 问题2：已完成的采购任务仍然显示
**现象**：
- 用户反馈"已完成的任务为什么还在手机端有提示"

**根本原因**：
- 后端过滤逻辑只检查 `status='received'`，没有排除 `arrived_quantity >= purchase_quantity` 的任务

---

## 🛠️ 已实施的修复

### 修复1：后端增加"已完成"任务过滤 ✅

**文件**：`MaterialPurchaseOrchestrator.java` (line 685-695)

**修改内容**：
```java
.filter(p -> {
    if (p.getArrivedQuantity() == null) return true;
    if (p.getPurchaseQuantity() == null) return true;
    return p.getArrivedQuantity() < p.getPurchaseQuantity();
})
```

**效果**：
- ✅ 排除已入库完成的采购任务（arrived_quantity >= purchase_quantity）
- ✅ 避免"已完成"任务继续显示在任务列表中

---

### 修复2：后端自动补充 orderNo 字段 ✅

**文件**：`MaterialPurchaseOrchestrator.java` (line 713-741)

**修改内容**：
```java
// 构建 orderId -> orderNo 映射
Map<String, String> orderIdToOrderNoMap = validOrders.stream()
    .collect(Collectors.toMap(
        ProductionOrder::getId,
        ProductionOrder::getOrderNo,
        (v1, v2) -> v1
    ));

// 返回时自动补充 orderNo
.peek(purchase -> {
    if (!StringUtils.hasText(purchase.getOrderNo())) {
        String orderId = purchase.getOrderId();
        if (StringUtils.hasText(orderId)) {
            String orderNo = orderIdToOrderNoMap.get(orderId);
            if (orderNo != null) {
                purchase.setOrderNo(orderNo);
            }
        }
    }
})
```

**效果**：
- ✅ 如果采购任务有 `orderId` 但没有 `orderNo`，自动从关联订单中获取
- ✅ 解决数据库中 order_no 字段为空但 order_id 有值的情况

---

### 修复3：允许无订单关联的独立采购任务 ✅

**文件**：`MaterialPurchaseOrchestrator.java` (line 730-741)

**修改内容**：
```java
.filter(purchase -> {
    String orderId = purchase.getOrderId();
    // 如果没有关联订单，保留（独立采购）
    if (!StringUtils.hasText(orderId)) {
        return true;
    }
    // 如果有关联订单，检查订单是否有效
    return validOrderIds.contains(orderId);
})
```

**效果**：
- ✅ 允许返回没有关联订单的采购任务（order_id 为 NULL）
- ✅ 解决当前 9 个任务都没有 order_id 的问题
- ✅ 前端可以正常显示独立采购任务

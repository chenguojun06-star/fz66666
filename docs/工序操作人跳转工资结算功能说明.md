# 工序操作人跳转到工资结算功能说明

**完成时间**: 2026-01-31  
**功能类型**: 快速导航链接

## 📋 功能概述

在生产订单的工序详情中，点击**操作人姓名**可以直接跳转到**财务模块-工资结算**页面，查看该操作人在该订单该工序下的工资明细。

## 🎯 功能位置

### 1. 生产订单列表 - 表格操作人列

在生产订单列表的表格中，以下列的操作人都可以点击跳转：

- **采购员** (`procurementOperatorName`)
- **裁剪员** (`cuttingOperatorName`)  
- **车缝员** (`carSewingOperatorName`)
- **尾部员** (`tailProcessOperatorName`)
- **入库员** (`warehousingOperatorName`)

**效果**：
- 操作人显示为蓝色链接 (`color: #1890ff`)
- 鼠标悬停显示手型光标
- 点击跳转到工资结算页面，自动筛选对应的订单和工序

## 🔗 跳转逻辑

### 跳转URL格式

```
/finance/payroll-operator-summary?orderNo={订单号}&processName={工序名称}
```

**示例**：
```
/finance/payroll-operator-summary?orderNo=PO20260122001&processName=裁剪
```

### 参数说明

- `orderNo`: 订单号（如：`PO20260122001`）
- `processName`: 工序名称（如：`采购`、`裁剪`、`车缝`、`尾部`、`入库`）

### 工序名称映射

| 节点Key | 显示名称 | processName参数 |
|---------|---------|----------------|
| procurement | 采购 | 采购 |
| cutting | 裁剪 | 裁剪 |
| carSewing | 车缝 | 车缝 |
| tailProcess | 尾部 | 尾部 |
| warehousing | 入库 | 入库 |

**工序委派Tab中**：
- 使用 `node.label` 作为工序名称（如：`node.label = "裁剪"`）

## 💻 技术实现

### 前端代码示例

#### 1. 表格列中的操作人链接（List/index.tsx 第830-852行）

```tsx
{
  title: titles.operator,
  dataIndex: `${prefix}OperatorName`,
  width: 120,
  render: (value: unknown, record: ProductionOrder) => {
    const operatorName = renderStageText(value);
    if (!operatorName || operatorName === '-') return '-';
    
    const orderNo = String(record?.orderNo || '').trim();
    const processName = titles.operator.replace('员', ''); // "裁剪员" → "裁剪"
    
    return (
      <a
        style={{ cursor: 'pointer', color: '#1890ff' }}
        onClick={() => {
          if (orderNo) {
            navigate(`/finance/payroll-operator-summary?orderNo=${orderNo}&processName=${processName}`);
          }
        }}
      >
        {operatorName}
      </a>
    );
  },
}
```

### 工资结算页面接收参数

在 `PayrollOperatorSummary/index.tsx` 中：

```tsx
const location = useLocation();
const navigate = useNavigate();

// 从URL参数获取订单号和工序名
useEffect(() => {
  const params = new URLSearchParams(location.search);
  const orderNoParam = params.get('orderNo');
  const processNameParam = params.get('processName');
  
  if (orderNoParam) {
    setOrderNo(orderNoParam);
  }
  if (processNameParam) {
    setProcessName(processNameParam);
  }
  
  // 自动触发查询
  if (orderNoParam || processNameParam) {
    handleSearch();
  }
}, [location.search]);
```

## 📊 使用场景

### 场景1：查看某个订单的工序工资分布

1. 在生产订单列表找到订单（如：`PO20260122001`）
2. 点击"工序详情"按钮
3. 查看裁剪节点，发现操作人是"李四"
4. **点击"李四"链接**
5. 自动跳转到工资结算页面，显示：
   - 订单号：`PO20260122001`
   - 工序：`裁剪`
   - 操作人：`李四`
   - 明细：扎号、颜色、尺码、数量、单价、金额...

### 场景2：追踪一个工序多人操作的情况

**问题**：一个工序（如车缝）可能由多个人共同完成，谁做了多少？

**解决方案**：
1. 打开工序详情 → 工序委派Tab
2. 看到"车缝"工序的操作人是"王五"（显示第一个操作人）
3. **点击"王五"链接**
4. 跳转到工资结算页面，显示：
   - 订单号：`PO20260122001`  
   - 工序：`车缝`
   - 此时会显示**所有参与车缝工序的人员**的工资明细
   - 可以看到：
     - 王五做了60件
     - 张三做了40件
     - 合计100件

### 场景3：快速查看某工序的成本

1. 在订单列表，点击车缝员列的操作人名字
2. 直接跳转到该订单车缝工序的工资结算页面
3. 查看总金额，即该工序的人工成本

## 🔍 常见问题

### Q1: 为什么只显示第一个操作人？

**A**: 在工序状态汇总（如工序委派Tab）中，`operatorName` 字段只记录了第一个完成操作的人。要查看所有参与人员，需要点击跳转到工资结算页面查看明细。

### Q2: 一个工序多人做，怎么查看每个人的工作量？

**A**: 点击操作人链接跳转到工资结算页面后，会显示该订单该工序的**所有扫码记录**，按操作人分组统计，可以看到每个人的工作量和工资。

### Q3: 如果没有操作人（显示"-"），可以点击吗？

**A**: 不可以。当操作人为空时，显示为普通文本"-"，不是链接，无法点击。

### Q4: 跳转后没有自动查询？

**A**: 检查工资结算页面是否正确接收URL参数，并在 `useEffect` 中监听 `location.search` 的变化。

## 🔄 业务流程详细说明

### 一、样衣开发阶段

```
┌─────────────────────────────────────────────────────────────────┐
│  样衣开发流程 (StyleInfo.progressNode 状态流转)                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ①新建款式 → ②纸样开发 → ③BOM配置 → ④尺码配置 → ⑤工序配置       │
│      ↓           ↓          ↓          ↓          ↓            │
│   pending    pattern      bom        size      process         │
│                 ↓                                               │
│            ⑥样衣生产(PatternProduction)                         │
│                 ↓                                               │
│       ┌────────┴────────┐                                       │
│       ↓                 ↓                                       │
│   样衣面料采购      样衣工序进度                                  │
│  (MaterialPurchase  (progressNodes)                             │
│   sourceType=        做领→上领→缝制                              │
│   'sample')                                                     │
│       ↓                 ↓                                       │
│       │                 │                                       │
│       ▼                 ▼                                       │
│  ┌──────────────────────────────┐                               │
│  │  面辅料采购页面 (统一管理)    │                               │
│  │  可筛选 sourceType='sample'  │                               │
│  └──────────────────────────────┘                               │
│       ↓                                                         │
│   ⑦样衣完成 → 可推送到下单管理                                   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

**关键字段**：
| 表 | 字段 | 说明 |
|---|---|---|
| t_style_info | progressNode | 当前开发进度节点 |
| t_style_info | patternStatus | 纸样状态 |
| t_style_info | patternAssignee | 纸样负责人 |
| t_pattern_production | progressNodes | 样衣工序进度JSON |
| t_material_purchase | sourceType='sample' | 样衣采购标识 |
| t_material_purchase | patternProductionId | 关联的样衣生产ID |

#### 采购单数据来源对照表

| 来源 | sourceType | 关联ID | 关联单号 | 典型场景 |
|-----|-----------|--------|---------|---------|
| 🟠 样衣开发 | `sample` | patternProductionId | styleNo | 新款打样、客户确认样 |
| 🔵 生产订单 | `order` | orderId | orderNo | 批量生产、工厂交货 |

## 🔑 关键状态流转

### 1. 订单状态 (ProductionOrder.status)
```
pending → production → completed
(待生产)  (生产中)     (已完成)
```

### 2. 采购状态 (MaterialPurchase.status)
```
pending → partial → received → completed
(待采购)  (部分到货) (已到货)   (已完成)
```

### 3. 裁剪状态 (CuttingTask.status)
```
pending → cutting → completed
(待裁剪)  (裁剪中)  (已完成)
```

### 4. 对账状态
```
pending → submitted → approved → settled
(待提交)  (已提交)    (已审批)   (已结算)
```

### 样衣采购 vs 订单采购 对比

| 对比项 | 🟠 样衣采购 (sample) | 🔵 订单采购 (order) |
|-------|---------------------|---------------------|
| **触发时机** | 样衣开发阶段 | 正式下单后 |
| **采购数量** | 小批量 (几米/几条) | 大批量 (几百/几千) |
| **关联单据** | PatternProduction | ProductionOrder |
| **关联字段** | patternProductionId | orderId |
| **显示单号** | styleNo (款号) | orderNo (订单号) |
| **用途** | 打版、试样、客户确认 | 批量生产、工厂交货 |
| **对账处理** | 计入样衣开发成本 | 计入生产成本 |

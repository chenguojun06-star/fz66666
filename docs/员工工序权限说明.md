# 员工工序模块权限说明

**创建时间**：2026-01-24  
**模块**：财务 → 员工工序

## ✅ 权限控制已打通

### 1. 后端数据权限（已实现）

#### API权限
- **权限码**：`MENU_PAYROLL_OPERATOR_SUMMARY`
- **Controller**：`PayrollSettlementController.getOperatorSummary()`
- **路径**：`POST /finance/payroll-settlement/operator-summary`

#### 数据范围控制（dataScope）

通过 `DataPermissionHelper.applyOperatorFilter()` 自动过滤：

| 角色类型 | dataScope 值 | 数据权限 | 说明 |
|---------|-------------|---------|------|
| **管理员** | `all` | 查看所有人员 | 不添加任何过滤条件 |
| **组长** | `team` | 查看团队数据 | 按 `team_id` 过滤 |
| **普通员工** | `own` | 只看自己 | 按 `operator_id` 或 `operator_name` 过滤 |

#### 实现位置
```java
// backend/src/main/java/com/fashion/supplychain/finance/orchestration/PayrollAggregationOrchestrator.java
// Line 67-68
// 应用数据权限过滤（根据角色：all=全部, team=团队, own=仅自己）
DataPermissionHelper.applyOperatorFilter(qw, "operator_id", "operator_name");
```

### 2. 前端权限（已实现）

#### 路由权限
- **路由配置**：`frontend/src/routeConfig.ts`
- **路径**：`/finance/payroll-operator-summary`
- **权限码**：`MENU_PAYROLL_OPERATOR_SUMMARY`
- **菜单显示**：财务 → 员工工序

#### Tab页面
1. **员工工序**：明细数据（订单号、款号、颜色、尺码、人员、工序等）
2. **工资汇总**：按人员聚合（总数量、总金额、扫码次数、订单数、备注、审核时间、付款时间）

### 3. 小程序端（暂无）

目前小程序端没有工资结算功能，员工只能：
- 扫码录入工序
- 查看自己的扫码记录

## 🔐 权限测试场景

### 场景1：管理员登录
- ✅ 能查看所有员工的工序数据
- ✅ 能进行批量审核、单条审核、退回操作
- ✅ 能看到所有订单的工资统计

### 场景2：组长登录
- ✅ 只能看到本团队员工的数据
- ✅ 可以审核团队成员的工资
- ❌ 看不到其他团队的数据

### 场景3：普通员工登录
- ✅ 只能看到自己的工序记录
- ✅ 只能看到自己的工资汇总
- ❌ 看不到其他员工的数据
- ❌ 没有审核权限

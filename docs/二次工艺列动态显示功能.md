# 二次工艺列常驻显示功能说明

## 📋 功能概述

二次工艺列采用**常驻显示 + 灰色占位**设计：
- ✅ **列永远显示**：不会因为无数据而隐藏整列
- 🎨 **有数据显示彩色**：显示彩色进度条，可点击查看详情/委派
- ⚪ **无数据显示灰色**：显示灰色占位符，点击无响应

## 🎯 设计理由

### 为什么不隐藏列？
1. **用户体验一致性**：表格列数固定，避免布局跳动
2. **数据完整性提示**：灰色占位提醒用户"这个订单没有二次工艺"
3. **与车缝等工序对齐**：车缝、大烫等工序也是常驻显示

## 🔍 判断逻辑

### 前端检测（hasSecondaryProcessData）
每行订单独立判断：

```typescript
const hasSecondaryProcessData = (() => {
  // 优先级1: 检查是否有扫码记录生成的时间数据
  if (record.secondaryProcessStartTime || record.secondaryProcessEndTime) {
    return true;
  }
  
  // 优先级2: 检查工序配置中是否有二次工艺
  const nodes = record.progressNodeUnitPrices;
  if (!Array.isArray(nodes) || nodes.length === 0) return false;
  return nodes.some(n => {
    const name = String(n.name || n.processName);
    return name.includes('二次工艺') || name.includes('二次') 
      || (name.includes('工艺') && !name.includes('车'));
  });
})();
```

### 渲染逻辑
```typescript
// 无数据：灰色占位（不可点击）
if (!hasSecondaryProcessData) {
  return (
    <div style={{ padding: '4px', opacity: 0.4 }}>
      <div style={{ 
        flex: 1, 
        height: '12px', 
        background: '#e8e8e8',  // 灰色背景
        borderRadius: '6px'
      }} />
      <span style={{ color: '#999' }}>-</span>
    </div>
  );
}

// 有数据：彩色进度条（可点击）
return (
  <div 
    style={{ cursor: 'pointer' }}
    onClick={() => openProcessDetail(record, 'secondaryProcess')}
  >
    <LiquidProgressBar percent={rate || 0} />
    <span>{rate || 0}%</span>
  </div>
);
```

### 后端数据源
1. **工序配置**：`progressNodeUnitPrices` 字段中的工序名称包含"二次工艺"
2. **扫码记录**：`t_scan_record` 表中 `progress_stage='二次工艺'` 的记录
3. **视图聚合**：`v_production_order_flow_stage_snapshot` 的 `secondary_process_*` 字段

## 📊 数据流向

```
扫码记录 (t_scan_record)
  ↓ progress_stage='二次工艺'
视图聚合 (v_production_order_flow_stage_snapshot)
  ↓ MIN/MAX聚合
  ↓ secondary_process_start_time
  ↓ secondary_process_end_time
  ↓ secondary_process_operator_name
Mapper查询 (ScanRecordMapper.selectFlowStageSnapshot)
  ↓ SELECT v.secondary_process_*
Service填充 (ProductionOrderQueryService.fillFlowStageFields)
  ↓ order.setSecondaryProcessStartTime(...)
API响应 (ProductionOrderController.list)
  ↓ JSON: { secondaryProcessStartTime, ... }
前端检测 (hasSecondaryProcessData)
  ↓ 检查每个订单是否有二次工艺数据
渲染逻辑
  ↓ 有数据 → 彩色进度条 + 可点击
  ↓ 无数据 → 灰色占位 + 不可点击
```

## 🔧 技术实现

### 1. 后端数据准备
- **ScanRecordMapper.java**（第36-40行）：
  ```java
  "  v.secondary_process_start_time AS secondaryProcessStartTime,",
  "  v.secondary_process_end_time AS secondaryProcessEndTime,",
  "  v.secondary_process_operator_name AS secondaryProcessOperatorName,",
  ```

- **init.sql**（第596-607行）：
  ```sql
  MIN(CASE WHEN ... = '二次工艺' THEN sr.scan_time END) AS secondary_process_start_time,
  MAX(CASE WHEN ... = '二次工艺' THEN sr.scan_time END) AS secondary_process_end_time,
  SUBSTRING_INDEX(MAX(...)) AS secondary_process_operator_name,
  ```

### 2. 前端常驻显示 + 灰色占位
- **List/index.tsx**（第1244-1326行）：
  ```typescript
  render: (rate: number, record: ProductionOrder) => {
    // 检测当前订单是否有二次工艺数据
    const hasSecondaryProcessData = 
      record.secondaryProcessStartTime || 
      record.secondaryProcessEndTime ||
      record.progressNodeUnitPrices?.some(n => 
        n.name.includes('二次工艺')
      );
    
    // 无数据：灰色占位（不可点击）
    if (!hasSecondaryProcessData) {
      return <div style={{ opacity: 0.4 }}>
        <div style={{ background: '#e8e8e8' }} />
        <span>-</span>
      </div>;
    }
    
    // 有数据：彩色进度条（可点击）
    return <div onClick={() => openProcessDetail(...)}>
      <LiquidProgressBar percent={rate} />
    </div>;
  }
  ```

### 3. 列过滤逻辑
- **不再条件过滤**：移除了 `hasAnySecondaryProcess` 检测
- **永远显示**：二次工艺列与车缝、大烫等工序列同等处理
  ```typescript
  const columns = allColumns.filter(col => {
    if (col.key === 'action' || col.key === 'orderNo') return true;
    return visibleColumns[col.key] !== false;
    // 不再对 secondaryProcessSummary 做特殊判断
  });
  ```

## 📝 注意事项

1. **性能优化**：使用行内检测，避免全局遍历
2. **数据优先级**：扫码记录 > 工序配置（更准确反映实际状态）
3. **用户体验**：灰色占位比隐藏列更直观
4. **向后兼容**：旧订单没有二次工艺时，显示灰色占位
5. **交互一致性**：与车缝、大烫等工序的交互逻辑完全一致

## 🎯 与其他工序的对比

| 工序 | 显示逻辑 | 常见状态 |
|------|---------|---------|
| 车缝 | **常驻显示** | 90%+ 有数据（彩色） |
| 大烫 | **常驻显示** | 80%+ 有数据（彩色） |
| 二次工艺 | **常驻显示** | 10%- 有数据（大部分灰色）|
| 包装 | **常驻显示** | 70%+ 有数据（彩色） |

**设计统一性**：所有工序列采用相同的显示逻辑，仅数据覆盖率不同

## 🚀 扩展思路

如需为其他工序（如"绣花"、"印花"）实现类似的条件显示逻辑：

1. 添加检测逻辑到 `hasAnySecondaryProcess` 模式
2. 在列过滤中添加对应的条件判断
3. 确保后端数据字段已配置

```typescript
// 示例：检测绣花工序
const hasAnyEmbroidery = useMemo(() => {
  return productionList.some(record => {
    return record.embroideryStartTime || 
           record.progressNodeUnitPrices?.some(n => 
             String(n.name).includes('绣花')
           );
  });
}, [productionList]);
```

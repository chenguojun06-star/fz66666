## 🐛 Bug 检查方法

### 1. 常见 Bug 类型自查清单

#### ✅ 数据绑定问题
```javascript
// ❌ 错误：直接修改 data
this.data.list.push(item);

// ✅ 正确：使用 setData
this.setData({
  list: [...this.data.list, item]
});
```

**检查命令**:
```bash
# 搜索直接修改 data 的代码
grep -rn "this.data\." pages/ --include="*.js" | grep -v "this.data\.[a-zA-Z]*)" | grep "="
```

#### ✅ 事件绑定缺失
```wxml
<!-- ❌ 错误：方法名拼写错误 -->
<button bindtap="onReceiveOnlyTypo">领取任务</button>

<!-- ✅ 正确：检查 JS 中是否有对应方法 -->
<button bindtap="onReceiveOnly">领取任务</button>
```

**检查方法**:
```bash
# 1. 提取 WXML 中的所有事件绑定
grep -rh "bind[a-z]*=\"" pages/ --include="*.wxml" | \
  grep -o 'bind[a-z]*="[^"]*"' | \
  cut -d'"' -f2 | sort -u > /tmp/wxml_events.txt

# 2. 在对应的 JS 文件中查找是否定义
# (手动检查，或使用脚本自动化)
```

#### ✅ API 调用错误
```javascript
// ❌ 错误：忘记 await
const result = api.production.executeScan(params);
result.data  // undefined，因为是 Promise

// ✅ 正确：使用 await
const result = await api.production.executeScan(params);
result.data  // 正常
```

**检查命令**:
```bash
# 搜索可能缺少 await 的 API 调用
grep -rn "api\." pages/ --include="*.js" | grep -v "await" | grep -v "then"
```

#### ✅ 变量未定义
```javascript
// ❌ 错误：使用了未定义的变量
if (isMobile) { ... }  // isMobile 未定义

// ✅ 正确：先定义再使用
const isMobile = wx.getSystemInfoSync().platform === 'android';
if (isMobile) { ... }
```

**检查方法**: 使用 ESLint 的 `no-undef` 规则（已启用 ✅）

## ⚡ 性能优化

### 1. 性能分析

**使用微信开发者工具**:
```
1. 调试器 → Performance → 点击录制
2. 操作页面（如打开弹窗、加载列表）
3. 停止录制
4. 分析瀑布图：
   - 蓝色：网络请求
   - 黄色：脚本执行
   - 紫色：渲染
```

**关键指标**:
- **首屏渲染时间**: < 1s
- **setData 调用频率**: < 100次/秒
- **单次 setData 数据量**: < 1MB
- **页面内存占用**: < 50MB

### 2. 常见性能问题

#### ❌ 问题1：频繁 setData
```javascript
// ❌ 错误：循环中调用 setData
for (let i = 0; i < 100; i++) {
  this.setData({
    [`list[${i}]`]: newItem
  });
}

// ✅ 正确：批量更新
const updates = {};
for (let i = 0; i < 100; i++) {
  updates[`list[${i}]`] = newItem;
}
this.setData(updates);
```

#### ❌ 问题2：setData 数据量过大
```javascript
// ❌ 错误：传递整个大对象
this.setData({
  historyList: largeArray  // 1000+ 条数据
});

// ✅ 正确：分页加载
this.setData({
  historyList: largeArray.slice(0, 20)  // 只加载前20条
});
```

#### ❌ 问题3：长列表渲染
```wxml
<!-- ❌ 错误：一次渲染1000条 -->
<view wx:for="{{allRecords}}" wx:key="id">...</view>

<!-- ✅ 正确：使用虚拟列表或分页 -->
<view wx:for="{{visibleRecords}}" wx:key="id">...</view>
```

## 📏 开发规范

### 1. Design Token 使用规范 ⭐⭐⭐⭐⭐

**统一设计变量** (`styles/design-tokens.wxss`):
```wxss
/* ✅ 使用 Design Token */
.button {
  background: var(--color-primary);
  border-radius: var(--radius-md);
  padding: var(--spacing-sm) var(--spacing-md);
}

/* ❌ 禁止硬编码 */
.button {
  background: linear-gradient(135deg, #3b82f6, #2563eb);
  border-radius: 12px;
  padding: 8px 16px;
}
```

**检查硬编码**:
```bash
# 检查硬编码颜色（应该为0）
grep -rn "#[0-9a-fA-F]\{6\}" pages/ --include="*.wxss" | grep -v "var(--"

# 检查硬编码圆角（应该为0）
grep -rn "border-radius: [0-9]" pages/ --include="*.wxss" | grep -v "var(--"
```

### 2. 命名规范

**文件命名**:
```
pages/scan/index.js          ✅ 小写 + 连字符
pages/work/components/       ✅ 组件目录
utils/dataValidator.js       ✅ 驼峰命名
```

**变量命名**:
```javascript
// ✅ 推荐
const userInfo = wx.getStorageSync('userInfo');
const scanHandler = new ScanHandler();

// ❌ 避免
const a = wx.getStorageSync('userInfo');
const sh = new ScanHandler();
```

**常量命名**:
```javascript
// ✅ 全大写 + 下划线
const API_BASE_URL = 'https://api.example.com';
const MAX_RETRY_COUNT = 3;

const apiUrl = 'https://api.example.com';
```

### 3. JSDoc 注释规范

**函数注释**:
```javascript
/**
 * 处理扫码结果（必须包含描述）
```

```javascript
/**
 * @typedef {object} ScanResult
```

### 4. 错误处理规范

**统一错误处理**:
```javascript
// ✅ 推荐：使用 try-catch + 统一错误提示
try {
  const result = await api.production.executeScan(params);
  wx.showToast({ title: '提交成功', icon: 'success' });
} catch (e) {
  // 使用 errorHandler 统一处理
  const errorMsg = errorHandler.handleError(e, '扫码提交');
  wx.showToast({ title: errorMsg, icon: 'none' });
}

// ❌ 避免：吞掉错误
api.production.executeScan(params).catch(() => {});
```

## 🔧 常见问题自查

### 问题1: "扫码领取任务没有反应"

**原因排查**:
```bash
# 1. 检查 WXML 事件绑定
grep "bindtap=\"onReceiveOnly\"" pages/scan/index.wxml

# 2. 检查 JS 方法定义
grep "onReceiveOnly" pages/scan/index.js

# 3. 检查控制台错误
# 打开微信开发者工具 → Console
```

**解决方法**: 
- ✅ 确保 JS 中定义了对应方法
- ✅ 检查方法名拼写是否一致
- ✅ 使用真机调试验证

### 问题2: "数据不刷新"

**原因**:
```javascript
// ❌ 直接修改 data（不会触发视图更新）
this.data.list.push(newItem);

// ✅ 使用 setData
this.setData({
  list: [...this.data.list, newItem]
});
```

### 问题3: "接口调用失败"

**检查清单**:
```javascript
// 1. 检查 Token 是否有效
const token = wx.getStorageSync('token');
console.log('Token:', token);

// 2. 检查网络请求配置（request.js）
// 确保 baseURL 正确
// 确保 timeout 足够长（10s）

// 3. 查看 Network 面板
// 检查请求参数、响应状态码

// 4. 后端日志
// 检查后端是否收到请求
```

### 问题4: "真机扫码无效"

**检查**:
```javascript
// 1. 权限配置（app.json）
{
  "permission": {
    "scope.camera": {
      "desc": "扫码需要使用摄像头"
    }
  }
}

// 2. 扫码逻辑（scan/index.js）
wx.scanCode({
  onlyFromCamera: true,  // 只允许摄像头扫码
  scanType: ['qrCode', 'barCode'],
  success: (res) => {
    console.log('扫码结果:', res.result);
  }
});
```

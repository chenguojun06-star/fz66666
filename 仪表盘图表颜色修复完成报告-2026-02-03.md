# ✅ 仪表盘图表颜色修复完成报告

## 🐛 问题描述

用户反馈仪表盘页面两个问题：
1. **图表显示黑色**："下单数量 vs 裁剪数量" 和 "扫菲次数 vs 扫菲数量" 两个折线图显示黑色，很难看
2. **延期订单表格未优化**：右上角表格显示正常，但延期天数缺少高亮

---

## 🔍 根本原因

### 问题1：图表颜色变量不存在

**图表代码使用了未定义的 CSS 变量**：
```typescript
// ❌ 错误：这些变量不存在
lineStyle: {
  color: 'var(--primary-color-light)',  // ← 未定义
}

lineStyle: {
  color: 'var(--warning-color)',        // ← 虽然定义了，但在 ECharts 中无法解析
}
```

**结果**：浏览器无法解析 CSS 变量，回退到默认黑色 `#000000`。

### 1. 修复 "扫菲次数 vs 扫菲数量" 图表

**文件**：`frontend/src/modules/dashboard/components/ScanCountChart/index.tsx`

**修改前**（黑色）：
```typescript
{
  name: '扫菲次数',
  lineStyle: {
    color: 'var(--primary-color-light)', // ❌ 未定义 → 黑色
  },
  areaStyle: {
    colorStops: [
      { offset: 0, color: 'rgba(139, 92, 246, 0.2)' },  // 紫色
```

```typescript
      { offset: 0, color: 'rgba(59, 130, 246, 0.3)' },  // 蓝色渐变

    width: 3,
```

## 📊 技术细节

### 为什么 CSS 变量在 ECharts 中无效？

**原因**：ECharts 在 Canvas 中渲染图表，无法访问 DOM 的 CSS 变量。

**错误示例**：
```typescript
// ❌ 这在 ECharts 中不工作
lineStyle: {
  color: 'var(--primary-color)',
}
```

**正确做法**：
```typescript
// ✅ 直接使用颜色值
lineStyle: {
  color: '#3b82f6',
}

// ✅ 或者从 CSS 读取后传入
const primaryColor = getComputedStyle(document.documentElement)
  .getPropertyValue('--primary-color').trim();
lineStyle: {
  color: primaryColor,
}
```

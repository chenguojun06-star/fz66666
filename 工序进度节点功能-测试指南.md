# 工序进度节点功能测试指南

> **完成时间**：2026年1月27日  
> **功能**：工序表添加进度节点字段，支持车缝/尾部多工序智能识别

## 📋 功能概述

### 核心功能
将工序与进度节点关联，实现扫码系统的智能工序识别：
- **进度节点**（5个大阶段）：采购 → 裁剪 → 车缝 → 尾部 → 入库
- **工序单价**（具体工序）：每个节点包含多个工序
- **智能识别**：扫码次数自动映射到对应节点的工序

### 业务场景
```
车缝节点：
  第1次扫 → 做领
  第2次扫 → 上领
  第3次扫 → 埋夹
  ...
  车缝完成 → 自动进入尾部节点

尾部节点：
  第1次扫 → 剪线
  第2次扫 → 整烫
  第3次扫 → 质检
  ...
```

## 🔧 数据库变更

### 已执行的迁移
```sql
-- 添加字段
ALTER TABLE t_style_process 
ADD COLUMN progress_stage VARCHAR(50) DEFAULT NULL 
COMMENT '进度节点（采购/裁剪/车缝/尾部/入库）' 
AFTER process_name;

-- 现有数据默认为车缝
UPDATE t_style_process 
SET progress_stage = '车缝' 
WHERE progress_stage IS NULL;

-- 添加索引
CREATE INDEX idx_progress_stage ON t_style_process(progress_stage);
```

## 📱 小程序扫码测试

### 测试前准备
1. 确保已配置好工序单价（包含进度节点）
2. 创建测试订单并关联该款号
3. 生成裁剪单和菲号

### 扫码测试流程

#### 测试1：车缝工序识别
```
1. 扫描菲号：PO20260127001-黑色-01
   → 识别工序：做领（车缝阶段第1个工序）
   → 提交成功

2. 再次扫描同一菲号
   → 识别工序：上领（车缝阶段第2个工序）
   → 提交成功

3. 继续扫描
   → 识别工序：埋夹（车缝阶段第3个工序）
   → 依次类推...
```

#### 测试2：车缝完成后进入尾部
```
1. 完成所有车缝工序（做领、上领、埋夹、上袖、订扣）
2. 再次扫描菲号
   → 识别工序：剪线（尾部阶段第1个工序）
   → 提交成功

3. 继续扫描
   → 识别工序：整烫（尾部阶段第2个工序）
   → 质检（尾部阶段第3个工序）
```

#### 测试3：防重复机制
```
1. 扫描菲号提交做领工序
2. 立即再次扫描同一菲号
   → 提示：请勿重复扫码
   → 最小间隔 = max(30秒, 件数 × 工序时间 × 60 × 50%)
```

## 🐛 常见问题

### Q1: 保存时提示"字段不存在"
**原因**：数据库迁移未执行  
**解决**：执行迁移脚本
```bash
docker exec -i fashion-mysql-simple mysql -uroot -pchangeme fashion_supplychain \
  < scripts/add_style_process_progress_stage.sql
```

### Q2: 下拉框没有显示进度节点选项
**原因**：前端代码未更新  
**解决**：
```bash
cd frontend
npm run build
# 或重启开发服务器
npm run dev
```

### Q3: 小程序扫码识别不到工序
**原因**：订单数据中缺少 progressStage 字段  
**解决**：
1. 重新保存样衣的工序单价
2. 重新创建订单
3. 或手动更新订单的 progressNodeUnitPrices 字段

### Q4: 现有订单如何处理？
**方案1**：手动更新现有工序的进度节点
```sql
-- 将所有现有工序设置为车缝
UPDATE t_style_process 
SET progress_stage = '车缝' 
WHERE progress_stage IS NULL;
```

**方案2**：重新配置工序单价并保存

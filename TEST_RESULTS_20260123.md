## 三、Bug汇总

### Bug 8: 订单扫码缺少数量 (新发现)

**严重程度**: 🔴 P0 - Critical  
**发现时间**: 2026-01-23 测试阶段  
**影响范围**: 订单扫码、JSON格式扫码  

**问题描述**:
- 订单格式二维码扫描时, QRCodeParser返回 `quantity: null`
- ScanHandler直接提交给 `executeScan` API
- 后端验证失败: "数量必须大于0" (400错误)

**根本原因**:
订单二维码只包含订单号,不包含数量信息,但业务逻辑要求必须有数量才能提交扫码记录

**修复方案**:
1. ✅ ScanHandler新增 `manualQuantity` 参数
2. ✅ 订单扫码无数量时返回 `{success:false, needInput:true, data}`
3. ✅ 页面层检测 `needInput` 标记,弹出数量输入框
4. ✅ 用户输入后调用 `handleScan(scanCode, quantity)` 重新提交

**代码修改**:
- `ScanHandler.js`: 
  * `handleScan(rawScanCode, manualQuantity=null)` 新增参数
  * 步骤2增加数量检查和needInput返回
- `scan-test/index.js`:
  * `processScan()` 增加needInput判断
  * 新增 `showQuantityInput()` 和 `processScanWithQuantity()`

**测试状态**: 🔄 已修复,待重新测试

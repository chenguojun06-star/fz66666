# 采购状态联动功能测试指南

## 功能说明
在生产订单的工序明细弹窗中，采购节点现在可以显示实际的采购完成状态、操作人信息和完成时间。

## 实现内容

### 后端 API
- **路径**: `GET /api/production/order/procurement-status/{orderId}`
- **功能**: 获取订单的采购完成状态
- **返回数据**:
  ```json
  {
    "completed": true/false,          // 是否完成
    "completionRate": 85,              // 完成率 (%)
    "operatorName": "张三",            // 操作人姓名
    "completedTime": "2026-01-31T10:30:00",  // 完成时间
    "manuallyConfirmed": false,        // 是否人工确认
    "procurementStartTime": "2026-01-28T09:00:00"  // 开始时间
  }
  ```

### 前端显示
在工序明细弹窗的**采购节点标题栏**显示：
- ✓ 已完成 (绿色徽章) 或 进行中 X% (黄色徽章)
- 操作人信息
- 完成时间 (格式：YYYY-MM-DD HH:mm)

## 业务逻辑

### 采购完成判定规则
1. **物料到货率 = 100%**: 自动认为采购完成
   - 操作人和时间来自最后一次物料收货记录
   
2. **物料到货率 ≥ 50% 且已人工确认**: 可进入下一阶段
   - 操作人和时间来自人工确认记录 (`procurementConfirmedByName`, `procurementConfirmedAt`)
   
3. **物料到货率 < 50%**: 必须停留在采购阶段，不允许人工确认

### 数据来源
- **物料到货率**: `ProductionOrder.materialArrivalRate`
- **人工确认**: `ProductionOrder.procurementManuallyCompleted`
- **确认人**: `ProductionOrder.procurementConfirmedByName`
- **确认时间**: `ProductionOrder.procurementConfirmedAt`
- **采购操作人**: `ProductionOrder.procurementOperatorName`
- **采购完成时间**: `ProductionOrder.procurementEndTime`

## 测试步骤

### 1. 准备测试订单
```sql
-- 查询有采购数据的订单
SELECT 
  id, 
  orderNo, 
  styleNo,
  materialArrivalRate,
  procurementManuallyCompleted,
  procurementOperatorName,
  procurementEndTime
FROM t_production_order 
WHERE deleteFlag = 0 
  AND materialArrivalRate IS NOT NULL
ORDER BY createTime DESC 
LIMIT 5;
```

### 2. 前端测试
1. 打开浏览器访问: `http://localhost:5173`
2. 登录系统
3. 进入 **生产订单** 页面
4. 点击订单的 **采购** 按钮，打开"采购工序明细"弹窗
5. 观察采购节点标题栏，应该显示：
   - 完成状态徽章（已完成/进行中）
   - 操作人姓名
   - 完成时间

### 3. API测试
```bash
# 获取订单的采购状态（替换 {orderId} 为实际订单ID）
curl -X GET "http://localhost:8080/api/production/order/procurement-status/{orderId}" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### 4. 验证场景

#### 场景1: 物料到货率 = 100%
- **预期**: 显示"✓ 已完成"，操作人为最后一次收货人

#### 场景2: 物料到货率 = 70%，已人工确认
- **预期**: 显示"✓ 已完成"，操作人为确认人

#### 场景3: 物料到货率 = 30%
- **预期**: 显示"进行中 (30%)"，无操作人和完成时间

## 注意事项
1. 采购完成状态仅在查看"采购工序明细"或"全部工序明细"时加载
2. 关闭弹窗时会重置 `procurementStatus` 状态
3. 时间格式使用中文本地化显示 (zh-CN)
4. 完成状态的颜色：
   - 已完成: 绿色 (#059669, 背景 #d1fae5)
   - 进行中: 黄色 (#f59e0b, 背景 #fef3c7)

## 五、文档维护规则

### 📅 日报规则
```
规则：每日创建一份日报
格式：YYYY年M月D日-日报.md
内容：
  ✅ 今日完成的工作 (5-10项)
  📋 工作清单 (完成情况表)
  🔴 后续跟进事项 (优先级标注)
  📁 文件变更明细
  🎯 下一步行动

保留策略：
  - 最近7天日报：显示在根目录
  - 历史日报（>7天）：移到 archive/日报/ 目录
```

### 📚 核心文档维护
```
❌ 不创建：
  - 单次优化报告（应在日报中说明）
  - 某个功能的临时方案（应在开发指南中）
  - 诊断/分析报告（应在日报中汇总）

✅ 只创建：
  - 核心规范文档（开发指南、SKU系统等）
  - 业务流程文档（工作流说明）
  - 技术方案文档（长期参考）
  - 工具使用指南（技术工具）
  - 每日日报（工作汇总）
```

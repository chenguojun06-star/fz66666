#### 数据库表
**文件**: `backend/sql/20260131_create_operation_log.sql`

```sql
CREATE TABLE t_operation_log (
  id BIGINT AUTO_INCREMENT PRIMARY KEY,
  module VARCHAR(50) NOT NULL,           -- 模块名称
  operation VARCHAR(20) NOT NULL,        -- 操作类型
  operator_id BIGINT NOT NULL,           -- 操作人ID
  operator_name VARCHAR(50) NOT NULL,    -- 操作人姓名
  target_type VARCHAR(50) NOT NULL,      -- 目标类型
  target_id VARCHAR(100) NOT NULL,       -- 目标ID
  target_name VARCHAR(200),              -- 目标名称
  reason VARCHAR(500),                   -- 操作原因
  details TEXT,                          -- 详细信息（JSON）
  ip VARCHAR(50),                        -- 操作IP
  user_agent VARCHAR(500),               -- 浏览器信息
  operation_time DATETIME NOT NULL,      -- 操作时间
  status VARCHAR(20) DEFAULT 'success',  -- 状态
  error_message TEXT,                    -- 错误信息
  -- 索引
  INDEX idx_module (module),
  INDEX idx_operation (operation),
  INDEX idx_operator (operator_id, operator_name),
  INDEX idx_target (target_type, target_id),
  INDEX idx_time (operation_time)
);
```

## ⚠️ 注意事项

1. **数据库连接**：
   - 容器名称：`fashion-mysql-simple`
   - 端口：3308（非标准端口）
   - 密码：查看 `deployment/mysql/.env`

2. **日志记录策略**：
   - 异步记录，失败不影响主操作
   - 控制台同步输出（便于调试）
   - 后端日志表持久化存储

3. **性能考虑**：
   - 操作日志表增长快，建议：
     - 定期归档（按月/季度）
     - 已添加必要索引
     - 分页查询限制最大条数
   - 异步记录避免阻塞主线程

4. **安全性**：
   - `details` 字段可能包含敏感信息
   - 不记录密码等关键凭证
   - 查询日志需要权限控制

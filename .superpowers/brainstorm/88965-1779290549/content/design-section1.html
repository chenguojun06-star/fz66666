设计方案 — 第一组：核心设计规范 + 反臃肿规则

先定规矩，再逐页对齐。按行业标准 + 你的痛点定制。

一、设计系统更新（design-system.css）

规范项 现在 改为 理由
页面背景 渐变 + 毛玻璃碎片 纯色 #f8fafc 减少视觉噪音
卡片背景 rgba(255,255,255,0.8) + blur(12px) 纯白 #fff + 1px #e5e7eb 边框 扁平化，行业标准
卡片圆角 12-16px 6px 行业 4-8px 标准
卡片阴影 多层阴影 (0 2px 16px...) 单层极淡 0 1px 3px rgba(0,0,0,0.06) 去臃肿感
间距基准 有8pt变量但不统一 强制 8pt：4/8/12/16/20/24/32/40 行业标准
表格行高 不一致（38-52px） 统一 44px 行业标准 44-48px
输入框高度 32px 36px（更易点击） 可用性提升

  • L1→L4 从上到下，视觉权重递减
  • 同级元素间距统一（8px/12px）

  • 禁止信息密集堆砌
  • 禁止一个区域超过 5 个同级元素

三、反臃肿规则 — 解决「一堆堆在一起」

规则 具体做法
1. 三区分隔 每个页面强制分3区：统计区 → 筛选区 → 表格区，区间 20px 白条分隔
2. 一行只放一个重点 表格行里最多1个彩色标签+1个进度指示器，不再塞多个图标/徽标
3. 卡片内 ≤5 个字段 统计卡片：标签(10px) + 数值(22px)，就2行。不塞多余信息。
4. 筛选区折叠 筛选条件 > 3 个时，只展前 3 个 +「更多筛选」按钮
5. 空状态统一 所有空状态用同一个 EmptyState 组件（灰色图标 + 一行说明文字），不用各种自定义
6. 干脆删多余 装饰性分割线、多余图标、重复标题、冗余边框——直接删掉

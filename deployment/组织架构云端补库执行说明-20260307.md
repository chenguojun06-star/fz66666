# 组织架构云端补库执行说明

适用范围：本次组织架构、工厂内外标签、订单组织快照、生产/财务/入库/库存链路的云端数据库补齐。

## 需要执行的文件

1. 主脚本：deployment/cloud-db-org-upgrade-20260307.sql
2. 云端控制台兼容版：deployment/cloud-db-org-upgrade-20260307-console.sql
2. 校验脚本：deployment/cloud-db-org-upgrade-20260307-verify.sql

## 执行顺序

1. 先在微信云托管数据库面板做备份。
2. 如果是微信云托管数据库控制台逐条执行，优先执行 deployment/cloud-db-org-upgrade-20260307-console.sql。
3. 如果是支持完整脚本和 PREPARE 的 MySQL 客户端，再执行 deployment/cloud-db-org-upgrade-20260307.sql。
4. 执行 deployment/cloud-db-org-upgrade-20260307-verify.sql。
5. 校验结果全部通过后，再做接口冒烟。

## 说明

- 主脚本按 information_schema 做了存在性判断，重复执行会自动跳过已存在的字段和索引。
- 微信云托管数据库控制台会把 PREPARE 这类动态 SQL 拆开执行，容易出现 PREPARE 失败但前面的直接建表/加列已经成功的情况。
- 因此云端人工执行时，直接用 console 版最稳，不走 PREPARE。
- 这份脚本只补结构，不包含任何本地联调用的 smoke 数据。
- 本地为了冒烟验证新增过的测试部门、测试订单挂靠、测试 SKU、测试入库记录，全部不要上云。

## 风险提醒

- 云端环境是 FLYWAY_ENABLED=false，这些 SQL 不执行，代码上线后就会继续报缺字段或缺表错误。
- 如果云端 t_factory 还缺 factory_type 或 supplier_type，旧的组织迁移会直接失败，所以主脚本已经把这两个前置字段一起兜住了。

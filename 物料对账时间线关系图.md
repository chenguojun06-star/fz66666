# 物料对账时间线关系图

## 📅 完整时间链路（互联关系）

```
┌─────────────────────────────────────────────────────────────────┐
│                     同一个采购流程                                │
└─────────────────────────────────────────────────────────────────┘

【采购单】t_material_purchase (源头)
   │
   ├─ create_time ──────────────► 1️⃣ 采购下单时间
   ├─ expected_arrival_date ────► 2️⃣ 预计到货日期 ────┐
   └─ actual_arrival_date ──────► 3️⃣ 实际到货日期 ────┤
                                                      │
                                                      │ 同步到对账
                ▼ 到货后执行入库                        │
                                                      │
【入库记录】t_material_inbound (中转)                    │
   │                                                  │
   ├─ purchase_id (关联采购单)                         │
   ├─ inbound_time ─────────────► 4️⃣ 入库时间 ────────┤
   └─ create_time                                    │
                                                      │
                ▼ 入库后自动同步                        │
                                                      ▼
【物料对账】t_material_reconciliation (终点)
   │
   ├─ purchase_id (关联采购单)
   ├─ expected_arrival_date ────► 2️⃣ 预计到货 ◄───── 从采购单同步
   ├─ actual_arrival_date ──────► 3️⃣ 实际到货 ◄───── 从采购单同步
   ├─ inbound_date ─────────────► 4️⃣ 入库时间 ◄───── 从入库记录同步
   ├─ reconciliation_date ──────► 5️⃣ 对账月份 (从入库时间计算)
   ├─ period_start_date ────────► 6️⃣ 对账周期开始 (入库月份1号00:00)
   └─ period_end_date ──────────► 7️⃣ 对账周期结束 (入库月份最后一天23:59)
```

## 🔗 互联关系说明

### 数据来源（都来自同一个采购单）

| 对账表字段 | 数据来源 | 关系 |
|-----------|---------|------|
| `expected_arrival_date` | `t_material_purchase.expected_arrival_date` | **直接同步** |
| `actual_arrival_date` | `t_material_purchase.actual_arrival_date` | **直接同步** |
| `inbound_date` | `t_material_inbound.inbound_time` | **直接同步** |
| `reconciliation_date` | 从 `inbound_date` 计算 | **派生：yyyy-MM格式** |
| `period_start_date` | 从 `inbound_date` 计算 | **派生：当月1号00:00** |
| `period_end_date` | 从 `inbound_date` 计算 | **派生：当月最后一天23:59** |

### 逻辑关系（按时间顺序）

```java
// 1. 预计到货 ← 采购时就设定
reconciliation.setExpectedArrivalDate(purchase.getExpectedArrivalDate());

// 2. 实际到货 ← 供应商到货时更新
reconciliation.setActualArrivalDate(purchase.getActualArrivalDate());

// 3. 入库时间 ← 仓库实际入库时间
LocalDateTime inboundTime = inbound.getInboundTime();
reconciliation.setInboundDate(inboundTime);

// 4. 对账月份 ← 根据入库时间计算（2026-01）
reconciliation.setReconciliationDate(
    inboundTime.format(DateTimeFormatter.ofPattern("yyyy-MM"))
);

// 5. 对账周期开始 ← 入库月份的1号00:00
reconciliation.setPeriodStartDate(
    inboundTime.withDayOfMonth(1).withHour(0).withMinute(0).withSecond(0)
);

// 6. 对账周期结束 ← 入库月份的最后一天23:59
reconciliation.setPeriodEndDate(
    inboundTime.withDayOfMonth(inboundTime.toLocalDate().lengthOfMonth())
              .withHour(23).withMinute(59).withSecond(59)
);
```

## 📊 实际案例

假设一个采购流程：

```
2026-01-10 10:00  → 采购下单
2026-01-20        → 预计到货 ← purchase.expected_arrival_date
2026-01-22 14:30  → 实际到货 ← purchase.actual_arrival_date
2026-01-22 16:00  → 仓库入库 ← inbound.inbound_time
                     └─ 自动触发对账同步

对账记录生成：
├─ expected_arrival_date: 2026-01-20 (从采购单)
├─ actual_arrival_date:   2026-01-22 14:30 (从采购单)
├─ inbound_date:          2026-01-22 16:00 (从入库记录)
├─ reconciliation_date:   "2026-01" (从入库时间计算)
├─ period_start_date:     2026-01-01 00:00:00
└─ period_end_date:       2026-01-31 23:59:59
```

## 🎯 为什么要互联？

### 1. **数据一致性**
- 所有时间都来自**同一个采购流程**
- 保证不会出现时间错乱（实际到货早于预计到货等异常）

### 2. **可追溯性**
```sql
-- 可以完整追溯一个采购单的全流程
SELECT 
  p.purchase_no AS '采购单',
  p.expected_arrival_date AS '预计',
  p.actual_arrival_date AS '实际',
  i.inbound_time AS '入库',
  r.reconciliation_date AS '对账月份'
FROM t_material_purchase p
LEFT JOIN t_material_inbound i ON i.purchase_id = p.id
LEFT JOIN t_material_reconciliation r ON r.purchase_id = p.id
WHERE p.purchase_no = 'PUR20260122001';
```

### 3. **业务分析价值**
- **预计 vs 实际**：供应商履约率分析
- **实际 vs 入库**：仓储环节效率分析
- **入库 → 对账周期**：自动归入正确的对账月份

### 4. **自动化对账**
- 入库时间决定对账周期（period_start_date ~ period_end_date）
- 财务按月对账时，自动归类到正确的月份
- 不需要手动指定对账月份

## ⚠️ 特殊情况

### 情况1：跨月入库
```
采购: 2026-01-28
到货: 2026-01-31
入库: 2026-02-03  ← 跨月了！

对账周期: 2026-02-01 ~ 2026-02-28  ← 按入库月份
```
✅ **正确**：按实际入库时间归入2月对账

### 情况2：分批入库
```
采购数量: 1000件

第1次入库: 2026-01-15, 500件 → 生成对账记录A (1月)
第2次入库: 2026-02-10, 500件 → 生成对账记录B (2月)
```
✅ **正确**：同一个采购单产生多条对账记录，分别归入不同月份

## 🔍 验证互联性

```sql
-- 验证时间一致性
SELECT 
  r.reconciliation_no,
  p.purchase_no,
  -- 这三个时间应该来自同一个采购流程
  p.expected_arrival_date AS '采购单-预计',
  r.expected_arrival_date AS '对账单-预计',
  p.actual_arrival_date AS '采购单-实际',
  r.actual_arrival_date AS '对账单-实际',
  i.inbound_time AS '入库记录-入库',
  r.inbound_date AS '对账单-入库',
  -- 这个应该是入库时间的月份
  DATE_FORMAT(r.inbound_date, '%Y-%m') AS '入库月份',
  r.reconciliation_date AS '对账月份'
FROM t_material_reconciliation r
JOIN t_material_purchase p ON r.purchase_id = p.id
JOIN t_material_inbound i ON r.purchase_id = i.purchase_id
WHERE r.reconciliation_date IS NOT NULL
LIMIT 5;

-- 预期结果：
-- ✅ 对账单的预计到货 = 采购单的预计到货
-- ✅ 对账单的实际到货 = 采购单的实际到货
-- ✅ 对账单的入库时间 = 入库记录的入库时间
-- ✅ 对账月份 = 入库月份
```

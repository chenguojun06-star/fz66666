# 系统稳定性测试计划（2026-03-11）

## 📋 测试范围

- ✅ **Redis 稳定性**：连接、性能、内存、持久化
- ✅ **Backend API 稳定性**：响应时间、错误率、扫码接口

## 🧪 测试场景

### 场景 A：正常业务流程（基准）
```
用户登录 → Token 存储 Redis → 刷新页面 → Token 命中缓存 → 调用业务 API
期望：全链路 <500ms，0 错误
```

### 场景 B：高并发（1000 人同时登录）
```
1000 个用户同时发起登录请求
期望：P95 <1s，P99 <3s，错误率 <0.1%
```

### 场景 C：超高并发（5000 人持续扫码）
```
5000 个工厂工人同时扫码 + 认证
期望：吞吐 >5000 req/sec，延迟稳定，Redis 自动扩容
```

### 场景 D：Redis 故障转移（模拟宕机）
```
Redis 容器重启 → 熔断 60 秒 → 自动降级 → 系统可用
期望：0 停服，用户感知轻微降速
```

### 场景 E：长时间运行（24 小时稳定性）
```
持续模拟真实业务 → 监控内存、CPU、连接数
期望：无内存泄漏，无连接泄漏，无持续错误增长
```

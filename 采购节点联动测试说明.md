# 采购节点数据联动测试说明

## 📝 功能说明

实现了**采购节点与采购模块的数据联动**，在订单的工序明细中实时显示采购完成状态。

### ✅ 已完成的功能

1. **后端API**：`GET /api/production/order/procurement-status/{orderId}`
   - 返回采购完成状态、完成率、操作人、完成时间

2. **前端显示**：
   - 在订单工序明细的"采购"节点标题栏显示完成状态
   - 显示操作人信息
   - 显示完成时间（格式：2026-01-31 14:30）

3. **状态判断逻辑**：
   - ✅ 物料到货率 = 100% → 自动完成
   - ✅ 物料到货率 ≥ 50% 且人工确认 → 已完成
   - ⚠️ 其他情况 → 进行中（显示完成率）

## 🧪 测试步骤

### 1. 准备测试数据

**方法A：使用现有订单**
1. 打开生产订单列表：http://localhost:5173/
2. 选择任意一个订单
3. 点击"操作"列的"工序明细"按钮

**方法B：创建新订单**
1. 点击"新建订单"按钮
2. 填写必要信息（款号、数量等）
3. 保存后打开工序明细

### 2. 测试采购未完成状态

**前置条件**：订单的物料到货率 < 50% 或未人工确认

**操作步骤**：
1. 打开订单工序明细
2. 点击"采购工序明细"或"全部工序明细"
3. 查看采购节点标题栏

**预期结果**：
```
采购 (1个工序) [⚠️ 进行中 (完成率: XX%)]
```

### 3. 测试采购已完成状态

**前置条件**：
- 方式1：物料到货率 = 100%
- 方式2：物料到货率 ≥ 50% 且人工确认

**操作步骤A（推荐）：在采购模块完成采购**
1. 进入采购管理模块
2. 找到对应订单的采购单
3. 完成采购收货（使物料到货率达到100%）
4. 返回生产订单列表
5. 打开该订单的工序明细

**操作步骤B：直接修改数据库（仅测试）**
```sql
-- 模拟采购完成（100%到货）
UPDATE t_production_order 
SET material_arrival_rate = 100,
    procurement_operator_name = '张三',
    procurement_end_time = NOW()
WHERE id = 'YOUR_ORDER_ID';
```

**预期结果**：
```
采购 (1个工序) [✓ 已完成] 操作人: 张三 完成时间: 2026-01-31 14:30
```

### 4. 测试数据实时更新

**操作步骤**：
1. 打开订单A的工序明细（采购未完成）
2. 在另一个标签页完成订单A的采购收货
3. 关闭工序明细弹窗
4. 重新打开工序明细

**预期结果**：
- 第一次打开：显示"进行中"
- 第二次打开：显示"已完成"+ 操作人 + 完成时间

## ❗ 常见问题

### Q1: 打开工序明细时没有显示采购状态

**可能原因**：
1. ✅ 后端API未启动（检查：`curl http://localhost:8088/actuator/health`）
2. ✅ 前端未正确调用API（检查浏览器Console）
3. ✅ 订单ID不存在

**解决方法**：
- 确保后端在运行：`./dev-public.sh` 或 `mvn spring-boot:run`
- 检查浏览器Network标签页的API请求
- 确认订单ID正确

### Q2: 显示"进行中"但实际已完成采购

**可能原因**：
1. 数据库中物料到货率未更新
2. 采购完成时间未记录

**解决方法**：
```sql
-- 检查订单采购字段
SELECT id, order_no, 
       material_arrival_rate, 
       procurement_manually_completed,
       procurement_operator_name,
       procurement_end_time
FROM t_production_order
WHERE order_no = 'PO20260130001';
```

### Q3: 完成时间格式异常

**可能原因**：时区问题或日期格式解析错误

**解决方法**：
- 检查前端日期格式化代码
- 确保后端返回ISO 8601格式：`2026-01-31T14:30:00`

## 📊 数据结构说明

### 后端API响应格式

```typescript
interface ProcurementStatus {
  completed: boolean;           // 是否完成
  completionRate: number;        // 完成率（0-100）
  operatorName: string | null;   // 操作人姓名
  completedTime: string | null;  // 完成时间（ISO 8601格式）
  manuallyConfirmed: boolean;    // 是否人工确认
  procurementStartTime: string | null; // 采购开始时间
}
```

### 数据库字段映射

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `material_arrival_rate` | INT | 物料到货率（0-100） |
| `procurement_manually_completed` | INT | 人工确认标志（0/1） |
| `procurement_operator_name` | VARCHAR | 采购操作人姓名 |
| `procurement_end_time` | DATETIME | 采购完成时间 |
| `procurement_confirmed_by_name` | VARCHAR | 人工确认操作人 |
| `procurement_confirmed_at` | DATETIME | 人工确认时间 |
| `procurement_start_time` | DATETIME | 采购开始时间 |

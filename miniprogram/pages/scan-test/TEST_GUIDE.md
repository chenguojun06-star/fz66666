# 🧪 扫码重构版本测试指南

## 📱 快速开始

### 1. 启动微信开发者工具

```bash
# 打开项目
打开微信开发者工具 → 导入项目 → 选择 miniprogram 目录
```

### 2. 访问测试页面

**方式一：从首页入口**
1. 运行小程序
2. 进入首页
3. 点击快捷入口中的 **"🧪测试"** 按钮
4. 进入扫码测试页面

**方式二：直接跳转**
```javascript
wx.navigateTo({ url: '/pages/scan-test/index' })
```

## 🧪 测试流程

### 测试 1：模拟扫码（推荐）

1. 点击 **"🔧 模拟扫码测试"** 按钮
2. 选择测试数据：
   - **1. 订单扫码** - `PO20260122001`
   - **2. 菲号扫码** - `PO20260122001-ST001-黑色-L-50-01`
   - **3. JSON格式** - `{"type":"order","orderNo":"PO20260122001"}`
   - **4. URL参数** - `?scanCode=PO20260122001&quantity=100`
3. 观察结果显示
4. 验证统计数据更新

### 测试 2：真实扫码

1. 准备测试二维码（参考 [QR_CODE_TEST_GUIDE.md](../../QR_CODE_TEST_GUIDE.md)）
2. 点击中央大圆按钮 **"开始扫码"**
3. 扫描测试二维码
4. 验证结果

### 测试 3：防重复扫码

1. 扫描同一个菲号（真实或模拟）
2. 立即再次扫描（10秒内）
3. 应该显示：**"⚠️ 50件预计需X分钟，X秒前已扫过"**
4. 不会提交到服务器

### 测试 4：撤销功能

1. 成功扫码后
2. 10秒内点击 **"撤销 (10s)"** 按钮
3. 应该显示 **"已撤销"**
4. 统计数据回滚

### 测试 5：对比旧版

1. 在测试页面点击 **"对比旧版"** 按钮
2. 跳转到旧版扫码页面
3. 执行相同的扫码操作
4. 对比两个版本的表现

## 🐛 已知问题

### 问题 1：首次加载工厂信息
- **现象**：第一次进入页面时工厂信息可能为空
- **解决**：返回首页重新进入，或手动选择工厂

### 问题 2：撤销倒计时
- **现象**：页面切换后倒计时可能停止
- **解决**：已在 onHide 中清理定时器

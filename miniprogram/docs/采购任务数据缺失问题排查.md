## 🔧 可能的问题和解决方案

### 问题1：后端返回的数据中没有orderNo字段

**原因**：数据库中的 `t_material_purchase` 表的 `order_no` 字段为NULL

**排查SQL**：
```sql
SELECT id, order_id, order_no, material_name, status, receiver_id
FROM t_material_purchase
WHERE status = 'received'
  AND delete_flag = 0
  AND return_confirmed = 0
LIMIT 10;
```

**检查**：
- `order_no` 列是否有值？
- 如果为NULL，是什么原因？（创建采购任务时未设置？）

**临时解决方案**：
```sql
-- 如果order_no为NULL但order_id有值，可以从订单表中同步
UPDATE t_material_purchase mp
INNER JOIN t_production_order po ON mp.order_id = po.id
SET mp.order_no = po.order_no
WHERE mp.order_no IS NULL
  AND mp.order_id IS NOT NULL;
```

### 问题2：floating-bell的数据映射问题

**原因**：后端返回的字段名不是 `orderNo`，而是其他名称

**排查**：查看 `[loadProcurementTasks] 原始数据` 日志中的 `fields` 数组，看看实际的字段名是什么

**修复**：如果字段名不同，修改 `floating-bell/index.js`:
```javascript
orderNo: item.orderNo || item.productionOrderNo || item.order_no || '',
```

### 问题3：API查询参数问题

**原因**：后端的 `/list` 端点不支持通过orderNo精确查询

**排查**：查看后端日志中的SQL语句

**修复**：后端需要支持精确匹配：
```java
// 如果传入了精确匹配标志，使用 eq 而不是 like
if (exactMatch) {
    wrapper.eq(MaterialPurchase::getOrderNo, orderNo);
} else {
    wrapper.like(MaterialPurchase::getOrderNo, orderNo);
}
```

### 问题4：采购任务已完成但未从任务列表中移除

**原因**：`getMyTasks()` 方法的过滤条件有问题

**排查SQL**：
```sql
-- 查看用户的所有采购任务（包括各种状态）
SELECT id, order_no, material_name, status, 
       arrived_quantity, purchase_quantity,
       receiver_id, receiver_name, return_confirmed
FROM t_material_purchase
WHERE receiver_id = 'your-user-id'
  AND delete_flag = 0
ORDER BY update_time DESC;
```

**检查**：
- 状态是否为 'received'？
- `return_confirmed` 是否为 0？
- 关联的订单状态是什么？

## 📊 数据流程图

```
用户点击铃铛 → floating-bell
    ↓ loadProcurementTasks()
    ↓ API: GET /api/production/purchase/list?myTasks=true
    ↓ 后端 MaterialPurchaseController.list()
    ↓ MaterialPurchaseOrchestrator.getMyTasks()
    ↓ 过滤：status='received' + returnConfirmed=0 + 当前用户
    ↓ 过滤：关联订单未关闭/完成/取消
    ↓ 返回采购任务列表

用户点击任务 → handleProcurementTask()
    ↓ 设置 storage: pending_procurement_task
    ↓ 跳转到扫码页

扫码页 onShow() → checkPendingProcurementTask()
    ↓ 读取 storage
    ↓ handleProcurementTaskFromBell(task)
    ↓ 【方案1】使用 orderNo 查询采购单列表
    ↓ API: GET /api/production/purchase/list?orderNo=PO202...
    ↓ 如果返回空，执行【方案2】
    ↓ 【方案2】直接使用 task 作为采购单数据
    ↓ 如果 task 缺少 orderNo，通过 orderId 查询订单
    ↓ API: GET /api/production/order/{orderId}
    ↓ 提取 orderNo 并补充到 task
    ↓ showConfirmModal() 显示确认弹窗
```

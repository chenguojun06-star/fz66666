# 采购完成进度计算功能 - 实现总结

**日期**: 2026-02-01  
**问题**: 采购确认后，订单进度仍然显示 0%，但应该显示采购节点完成的百分比

---

## 🎯 需求理解

用户业务逻辑：
1. **下单时** → 进度 = 0%（还没有任何动作）
2. **采购完成时** → 进度应该显示采购节点的百分比
   - 如果有 8 个节点：采购完成 = 第1个节点完成 = 13%
   - 如果有 7 个节点：采购完成 = 第1个节点完成 = 14%
3. **后续工序完成** → 继续增加百分比

---

## ✅ 解决方案

### 实现代码

在 `confirmProcurement` 方法中添加进度计算逻辑：

```java
// 计算采购完成后的进度（采购是第1个节点）
try {
    String workflowJson = order.getProgressWorkflowJson();
    if (StringUtils.hasText(workflowJson)) {
        ObjectMapper mapper = new ObjectMapper();
        Map<String, Object> workflow = mapper.readValue(workflowJson, 
            new TypeReference<Map<String, Object>>() {});
        @SuppressWarnings("unchecked")
        List<Map<String, Object>> nodes = (List<Map<String, Object>>) workflow.get("nodes");
        
        if (nodes != null && !nodes.isEmpty()) {
            int totalNodes = nodes.size();
            // 采购完成 = 第1个节点完成 = 1/N * 100
            int progress = (int) Math.round(100.0 / totalNodes);
            updateEntity.setProductionProgress(progress);  // ✅ 关键修复
            log.info("Order procurement confirmed - updating progress: orderId={}, totalNodes={}, progress={}%", 
                    orderId, totalNodes, progress);
        }
    }
} catch (Exception e) {
    log.warn("Failed to calculate procurement progress: orderId={}", orderId, e);
    // 即使计算失败也继续，不阻断采购确认流程
}
```

### 计算公式

```
进度百分比 = Math.round(100.0 / 节点总数)

示例：
- 8个节点：100.0 / 8 = 12.5 → 四舍五入 = 13%
- 7个节点：100.0 / 7 = 14.28... → 四舍五入 = 14%
- 10个节点：100.0 / 10 = 10%
```

## 🎨 前端显示逻辑

### 进度条颜色

前端已实现超期颜色渐变逻辑：

```typescript
const getStatus = (record: ProductionOrder): 'normal' | 'warning' | 'danger' => {
  if (!record.plannedEndDate) return 'normal';
  
  const diffDays = dayjs(record.plannedEndDate).diff(dayjs(), 'day');
  
  if (diffDays <= -4) return 'danger';   // 超期3天以上 - 深红
  if (diffDays < 0) return 'danger';     // 超期1-3天 - 红色
  if (diffDays === 0) return 'warning';  // 当天交期 - 微红色
  if (diffDays <= 3) return 'warning';   // 3天内交期 - 橙色
  
  return 'normal';  // 正常 - 蓝色/绿色
};
```

## 📝 业务逻辑

### 工序节点进度映射

假设有8个标准节点：

| 节点序号 | 节点名称 | 完成后进度 | 计算公式 |
|---------|---------|-----------|----------|
| 0       | 采购    | 13%       | (1/8)*100 |
| 1       | 裁剪    | 25%       | (2/8)*100 |
| 2       | 车缝    | 38%       | (3/8)*100 |
| 3       | 剪线    | 50%       | (4/8)*100 |
| 4       | 整烫    | 63%       | (5/8)*100 |
| 5       | 质检    | 75%       | (6/8)*100 |
| 6       | 包装    | 88%       | (7/8)*100 |
| 7       | 入库    | 100%      | (8/8)*100 |

### 节点完成方式

1. **采购**：手动确认（本次修复的功能）
2. **裁剪-包装**：工序扫码记录（自动更新进度）
3. **入库**：成品入库确认（进度 = 100%）

## 🔍 API设计问题分析

### 1. 重复/冗余端点识别

#### 问题1: list和page并存（1处）

**SampleStockController**
```java
@GetMapping("/page")  // ❌ 冗余
public ApiResponse<PagedData<SampleStock>> page(QueryParams params) {...}

// 建议：统一使用 /list 支持分页参数
@GetMapping("/list")  // ✅ 保留
public ApiResponse<PagedData<SampleStock>> list(
    @RequestParam(defaultValue = "1") int page,
    @RequestParam(defaultValue = "10") int size
) {...}
```

**优化收益**：减少1个端点

---

#### 问题2: 多个详情端点（建议评估）

**LogisticsController（7个详情端点）**
```java
@GetMapping("/{id}")                              // 1. 按ID查询
@GetMapping("/by-tracking-no/{trackingNo}")      // 2. 按运单号查询
@GetMapping("/by-order/{orderId}")                // 3. 按订单ID查询
@GetMapping("/{id}/tracks")                       // 4. 查询物流轨迹
@GetMapping("/pending-sync")                      // 5. 待同步订单
@GetMapping("/statistics")                        // 6. 统计信息
@GetMapping("/express-companies")                 // 7. 快递公司列表
```

**优化建议**：
- ✅ 保留：`/{id}`, `/{id}/tracks`（子资源）, `/statistics`, `/express-companies`
- 🔧 合并：`/by-tracking-no`, `/by-order`, `/pending-sync` → 统一查询端点 + 参数

**修改方案**：
```java
// ✅ 统一查询端点
@GetMapping("/search")
public ApiResponse<ExpressOrder> search(
    @RequestParam(required = false) Long id,
    @RequestParam(required = false) String trackingNo,
    @RequestParam(required = false) Long orderId,
    @RequestParam(required = false) Boolean pendingSync
) {...}
```

**优化收益**：减少3个端点（7 → 4）

---

**PatternRevisionController（7个详情端点）**
```java
@GetMapping("/{id}")                  // 1. 基础详情
@GetMapping("/next-version")          // 2. 下一个版本号
// 其他5个是业务操作（submit/approve/reject/complete等），保留
```

**评估结论**：
- ✅ 业务操作端点合理，不建议合并
- `/next-version`可以考虑改为查询参数，但影响较小

### 2. RESTful规范问题

#### 问题1: POST误用于查询（1处）

**ProductionOrderController**
```java
@PostMapping("/list-simple")  // ❌ 应该用GET
public ApiResponse<List<ProductionOrder>> listSimple(@RequestBody QueryParams params) {...}
```

**修复方案**：
```java
@GetMapping("/simple")  // ✅ 改为GET + 查询参数
public ApiResponse<List<ProductionOrder>> listSimple(
    @RequestParam(required = false) String keyword,
    @RequestParam(required = false) String status
) {...}
```

---

#### 问题2: 路径包含动词（20+个）

**示例**：
```java
// ❌ 当前
@PostMapping("/update-progress")
@PostMapping("/update-material-rate")
@PostMapping("/confirm-procurement")
@PostMapping("/recompute-progress")
@GetMapping("/generate-no")

// ✅ 应该
@PutMapping("/progress")
@PutMapping("/material-rate")
@PostMapping("/procurement-confirmation")
@PostMapping("/progress-recomputation")
@GetMapping("/generated-no")
```

**影响**：低（功能正常，仅规范问题）

## 🔐 API安全性评估

### 1. 认证机制

**方式**: JWT Token  
**实现**: `AuthTokenService` + `TokenAuthFilter`  
**覆盖率**: 100%（所有API需登录）

### 2. 权限控制

**方式**: Spring Security `@PreAuthorize`  
**覆盖率**: 8.1%（27/334）❌  
**问题**: 严重不足

### 3. 数据隔离

**多工厂隔离**: ⚠️ 部分实现  
**问题**: 权限控制缺失导致无法有效隔离

### 4. 审计日志

**实现**: `OperationLogController`  
**覆盖**: ✅ 关键操作有记录  
**问题**: 无权限前置验证

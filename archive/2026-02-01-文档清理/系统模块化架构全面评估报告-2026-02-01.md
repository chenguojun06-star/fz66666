public ApiResponse<PagedData<StyleInfo>> list(StyleQueryParams params) {...}

@PostMapping
@PreAuthorize("hasAuthority('STYLE_CREATE')")
public ApiResponse<StyleInfo> create(@RequestBody StyleInfo styleInfo) {...}
```

#### 2.1 架构分层规范

**系统架构定义**：
```

##### 2.2.1 典型违规案例

**案例1：MaterialReconciliationSyncServiceImpl**（finance模块）
```java
@Service
public class MaterialReconciliationSyncServiceImpl implements MaterialReconciliationSyncService {
    @Autowired private MaterialInboundService materialInboundService;      // ❌ 跨模块（production）
    @Autowired private MaterialPurchaseService materialPurchaseService;    // ❌ 跨模块（production）
    @Autowired private MaterialReconciliationService reconciliationService; // ✅ 同模块
    
    public void syncPurchaseToReconciliation(String orderNo) {
        // ❌ Service层不应做跨模块协调！
        MaterialPurchase purchase = materialPurchaseService.getByOrderNo(orderNo);
        MaterialInbound inbound = materialInboundService.getByOrderNo(orderNo);
        reconciliationService.backfill(purchase, inbound);
    }
}
```

**问题定位**：
- 文件：`backend/src/main/java/com/fashion/supplychain/finance/service/impl/MaterialReconciliationSyncServiceImpl.java`
- 第82-95行：跨模块调用production的Service
- 违反规范：Service应该只做单领域CRUD

**正确做法**：创建Orchestrator
```java
// ✅ 应该在 finance/orchestration/ 创建
@Service
public class MaterialReconciliationSyncOrchestrator {
    @Autowired private MaterialReconciliationService reconciliationService;  // finance
    @Autowired private MaterialInboundService inboundService;                // production
    @Autowired private MaterialPurchaseService purchaseService;              // production
    
    @Transactional  // Orchestrator负责跨Service事务
    public void syncPurchaseToReconciliation(String orderNo) {
        MaterialPurchase purchase = purchaseService.getByOrderNo(orderNo);
        MaterialInbound inbound = inboundService.getByOrderNo(orderNo);
        reconciliationService.backfill(purchase, inbound);
    }
}
```

**示例代码**：
```typescript
// api.ts 核心类型
export type ApiResult<T = any> = {
  code: number;
  data: T;
  message?: string;
};

export const isApiSuccess = (result: unknown): result is ApiResult => {
  return (
    typeof result === 'object' &&
    result !== null &&
    'code' in result &&
    Number((result as ApiResult).code) === 200
  );
};

export const unwrapApiData = <T = unknown>(result: unknown, fallbackMessage: string): T => {
  if (isApiSuccess(result)) return (result as ApiResult<T>).data;
  throw new Error(getApiMessage(result, fallbackMessage));
};
```

**优点**：
- ✅ **类型守卫**：`isApiSuccess` 提供类型收窄
- ✅ **错误处理**：统一错误消息提取
- ✅ **业务工具集成**：尺码排序等业务逻辑在同一文件

**问题**：
- ⚠️ **文件过大**：718行，建议拆分（基础工具 vs 业务工具）
- ⚠️ **命名不一致**：同时有 `api.ts` 和 `xxxApi.ts`

```typescript
// routeConfig.ts
export const resolvePermissionCode = (pathname: string): string | undefined => {
  const current = normalizePath(pathname);
  if (routeToPermissionCode[current]) return routeToPermissionCode[current];

  // 动态路由匹配（如 /style-info/:id）
  let best: { prefix: string; code: string } | null = null;
  for (const [prefix, code] of Object.entries(routeToPermissionCode)) {
    if (current.startsWith(prefix + '/')) {
      if (!best || prefix.length > best.prefix.length) {
        best = { prefix, code };
      }
    }
  }
  return best?.code;
};
```

## 9. 最佳实践与规范

### 9.1 文件组织规范

#### **模块结构标准**

```
modules/{module}/
├── index.tsx                    // 模块导出（懒加载）
├── pages/                       // 页面组件
│   └── {Page}/
│       ├── index.tsx            // 主页面（最多800行）

│       │   ├── {Feature}Modal.tsx
│       │   └── {Feature}Table.tsx
│       └── hooks/               // 页面专属Hooks
│           └── use{Feature}.ts
└── components/                  // 模块共享组件（可选）
    └── {Shared}Component.tsx
```

**规则**：
- ✅ 页面主文件最多800行（超过必须拆分）
- ✅ 组件嵌套最多3层（pages → components → 子组件）
- ✅ hooks 放在同级或上级，不要深层嵌套

---

#### **组件拆分决策树**

```
页面组件行数 > 800?
├─ YES → 是否包含多个Tab？
│   ├─ YES → 拆分为多个Tab组件 (StyleInfo 模式)
│   └─ NO  → 是否包含复杂表单/弹窗？
│       ├─ YES → 拆分为独立Modal组件 (List 模式)
│       └─ NO  → 提取业务逻辑为自定义Hooks
└─ NO  → 保持单文件 ✅
```

### 9.2 API调用规范

#### **强制规则**

```typescript
// ❌ 禁止：直接调用
const data = await api.get('/production/order/list', { params });

// ✅ 强制：通过服务层
import { productionOrderApi } from '@/services/production/productionApi';
const data = await productionOrderApi.list(params);
```

#### **服务层定义标准**

```typescript
// services/{module}/{entity}Api.ts
import api from '@/utils/api';
import type { PaginatedResponse, SuccessResponse } from '@/types/api';
import type { Entity, EntityQueryParams } from '@/types/{module}';

export const entityApi = {
  // 查询列表
  list: (params: EntityQueryParams) =>
    api.get<PaginatedResponse<Entity>>('/api/{module}/{entity}/list', { params }),
  
  // 获取详情
  detail: (id: string) =>
    api.get<ApiResponse<Entity>>(`/api/{module}/{entity}/detail/${id}`),
  
  // 创建
  create: (data: Partial<Entity>) =>
    api.post<SuccessResponse>('/api/{module}/{entity}/create', data),
  
  // 更新
  update: (id: string, data: Partial<Entity>) =>
    api.put<SuccessResponse>(`/api/{module}/{entity}/update/${id}`, data),
  
  // 删除
  delete: (id: string) =>
    api.delete<SuccessResponse>(`/api/{module}/{entity}/delete/${id}`),
};
```

**命名规范**：
- 文件名：`{entity}Api.ts`（小驼峰 + Api后缀）
- 导出对象：`{entity}Api`
- 方法名：CRUD动词（list/detail/create/update/delete）

### 9.3 类型定义规范

#### **类型文件组织**

```typescript
// types/{module}.ts
// 1. 实体类型
export interface Entity {
  id: string;
  name: string;
  status: 'active' | 'inactive';
  createdAt: string;
  updatedAt: string;
}

// 2. 查询参数类型
export interface EntityQueryParams extends PageParams {
  name?: string;
  status?: string;
}

// 3. 表单类型
export type EntityFormData = Omit<Entity, 'id' | 'createdAt' | 'updatedAt'>;

// 4. 响应类型（使用api.ts的泛型）
export type EntityListResponse = PaginatedResponse<Entity>;
export type EntityDetailResponse = DetailResponse<Entity>;
```

**规则**：
- ✅ 实体类型与后端一一对应
- ✅ 查询参数继承 `PageParams`
- ✅ 表单类型用 `Omit` 排除自动字段
- ✅ 响应类型复用 `api.ts` 的泛型

## 🔧 测试中发现并修复的问题

### 问题1: bundle_no 字段类型错误 ✅ 已修复
**严重级别**: P0（阻塞）  
**问题描述**: `t_cutting_bundle.bundle_no` 是 `int` 类型，无法存储 `PO20260126001-黑色-01` 格式  
**修复方案**:
```sql
ALTER TABLE t_cutting_bundle 
MODIFY COLUMN bundle_no VARCHAR(100);
```
**修复结果**: ✅ 成功，菲号创建正常

### 问题2: 扫码表字段冗余 ⚠️ 已绕过
**严重级别**: P1（高）  
**问题描述**: `t_scan_record` 同时有 `cutting_bundle_no`(int) 和 `bundle_no`(varchar)  
**临时方案**: 使用正确的 `bundle_no` 字段  
**后续建议**: 
```sql
-- 建议删除旧字段
ALTER TABLE t_scan_record DROP COLUMN cutting_bundle_no;
ALTER TABLE t_scan_record DROP COLUMN cutting_bundle_qr_code;
```

### 问题3: 入库表必填字段 ⚠️ 已绕过
**严重级别**: P1（高）  
**问题描述**: `t_product_warehousing.order_id` 必填但没有默认值  
**临时方案**: 脚本中显式传入 `order_id`  
**后续建议**: 后端接口应自动从 `order_no` 查找 `order_id`

### 问题4: 测试数据脚本字段名不匹配 ✅ 已修复
**问题**: 初始化脚本使用了错误的字段名（real_name, factory_no, cutting_task_id）  
**修复**: 更正为实际字段名（name, factory_code, production_order_id）

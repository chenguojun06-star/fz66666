##⚠️  发现的严重问题

### 问题1：菲号(bundle_no)字段类型错误 ⭐⭐⭐⭐⭐ （严重）

**表**：`t_cutting_bundle`
**问题**：`bundle_no`字段类型是`int`，但业务逻辑中使用字符串格式（如`PO20260122001-黑色-01`）

**当前定义**：
```sql
bundle_no int NOT NULL
```

**应该是**：
```sql
bundle_no varchar(100) NOT NULL
```

**影响**：
- 无法创建菲号
- 扫码系统无法正常工作（依赖菲号字符串）
- **阻塞整个生产流程**

**修复脚本**：
```sql
ALTER TABLE t_cutting_bundle MODIFY COLUMN bundle_no VARCHAR(100) NOT NULL;
```

---

### 问题2：扫码记录表字段名不一致 ⭐⭐⭐⭐ （高优先级）

**表**：`t_scan_record`
**问题**：使用`cutting_bundle_id` + `cutting_bundle_no`，而不是代码中的`bundle_no`

**当前字段**：
- `cutting_bundle_id varchar(36)`
- `cutting_bundle_no int`
- `cutting_bundle_qr_code varchar(200)`

**代码期望**：
- `bundle_no`（字段不存在）

**影响**：
- 扫码提交失败
- 菲号扫码模式无法使用
- **核心功能无法使用**

**修复方案**：
1. 方案A：添加`bundle_no`字段（推荐）
   ```sql
   ALTER TABLE t_scan_record ADD COLUMN bundle_no VARCHAR(100);
   ```

2. 方案B：修改后端代码使用现有字段

---

### 问题3：入库单号(warehousing_no)缺少默认值 ⭐⭐⭐ （中等优先级）

**表**：`t_product_warehousing`
**问题**：`warehousing_no`是必填字段，但创建时没有传值，也没有默认值

**当前定义**：
```sql
warehousing_no varchar(50) NOT NULL
```

**影响**：
- 质检入库无法创建
- 对账流程中断

**修复方案**：
1. 方案A：添加自动生成逻辑（推荐）
   ```sql
   ALTER TABLE t_product_warehousing 
   MODIFY COLUMN warehousing_no VARCHAR(50) DEFAULT NULL;
   
   -- 或在插入时生成
   INSERT INTO t_product_warehousing (warehousing_no, ...) 
   VALUES (CONCAT('WH', DATE_FORMAT(NOW(), '%Y%m%d%H%i%s')), ...);
   ```

2. 方案B：前端/后端必须传值

---

### 问题4：工资结算表存在但无数据关联 ⭐⭐ （低优先级）

**表**：
- `t_payroll_settlement` - 存在
- `t_payroll_settlement_item` - 存在

**问题**：

- 缺少工资计算逻辑

**影响**：

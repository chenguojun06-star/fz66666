# 样衣开发费用统计修复完成

> **修复日期**: 2026-01-30  
> **问题描述**: 样衣开发页面顶部统计显示的费用全部为 ¥0.00，已完成的样衣单价未被统计进去  
> **影响范围**: 后端 StyleInfoOrchestrator，前端样衣开发页面统计看板

---

## 📊 问题分析

### 原始问题
用户反馈样衣开发页面（StyleInfo/index.tsx）顶部的统计数据显示异常：

```
面辅料：¥0.00
工序单价：¥0.00
二次工艺：¥0.00
总开发费：¥0.00
```

即使下方表格显示有多条"样衣完成"状态的记录，但顶部统计仍为 0。

### 根本原因

后端 `StyleInfoOrchestrator.getDevelopmentStats()` 方法中，费用统计部分写死为 0.0：

```java
// ❌ 修复前：固定返回 0
Map<String, Object> stats = new HashMap<>();
stats.put("patternCount", styleCount);
stats.put("materialCost", 0.0);
stats.put("processCost", 0.0);
stats.put("secondaryProcessCost", 0.0);
stats.put("totalCost", 0.0);
```

虽然查询了已完成的样衣列表，但没有汇总其报价单中的费用信息。

## 🔧 修复方案

### 1. 数据表关系梳理

- **样衣信息表** (`t_style_info`): 存储款式基本信息，包括样衣完成状态 (`sample_status`) 和完成时间 (`sample_completed_time`)
- **报价单表** (`t_style_quotation`): 存储每个款式的费用明细
  - `material_cost` (面辅料费用)
  - `process_cost` (工序单价)
  - `other_cost` (其他费用，前端显示为"二次工艺")
  - `total_cost` (总费用)

### 2. 后端代码修改

#### 修改文件
`backend/src/main/java/com/fashion/supplychain/style/orchestration/StyleInfoOrchestrator.java`

#### 添加依赖注入
```java
// 新增导入
import com.fashion.supplychain.style.entity.StyleQuotation;
import com.fashion.supplychain.style.service.StyleQuotationService;

// 新增服务注入
@Autowired
private StyleQuotationService styleQuotationService;
```

#### 修复统计逻辑
```java
/**
 * 获取样衣开发费用统计
 */
public Map<String, Object> getDevelopmentStats(String rangeType) {
    LocalDateTime startTime = getStartTimeByRange(rangeType);
    LocalDateTime endTime = LocalDateTime.now();

    // 查询时间范围内已完成的样衣
    List<StyleInfo> completedStyles = styleInfoService.lambdaQuery()
            .eq(StyleInfo::getSampleStatus, "COMPLETED")
            .ge(StyleInfo::getSampleCompletedTime, startTime)
            .le(StyleInfo::getSampleCompletedTime, endTime)
            .list();

    int styleCount = completedStyles.size();

    // ✅ 修复：统计费用 - 遍历所有已完成的样衣，汇总其报价单费用
    double totalMaterialCost = 0.0;
    double totalProcessCost = 0.0;
    double totalOtherCost = 0.0;

    for (StyleInfo style : completedStyles) {
        // 查询该样衣的报价单
        StyleQuotation quotation = styleQuotationService.lambdaQuery()
                .eq(StyleQuotation::getStyleId, style.getId())
                .one();
        
        if (quotation != null) {
            totalMaterialCost += (quotation.getMaterialCost() != null ? quotation.getMaterialCost().doubleValue() : 0.0);
            totalProcessCost += (quotation.getProcessCost() != null ? quotation.getProcessCost().doubleValue() : 0.0);
            totalOtherCost += (quotation.getOtherCost() != null ? quotation.getOtherCost().doubleValue() : 0.0);
        }
    }

    double totalCost = totalMaterialCost + totalProcessCost + totalOtherCost;

    Map<String, Object> stats = new HashMap<>();
    stats.put("patternCount", styleCount);
    stats.put("materialCost", totalMaterialCost);
    stats.put("processCost", totalProcessCost);
    stats.put("secondaryProcessCost", totalOtherCost);  // other_cost 对应前端的"二次工艺"
    stats.put("totalCost", totalCost);

    return stats;
}
```

## 📊 技术细节

### 查询逻辑
1. 根据时间范围 (`rangeType`) 查询已完成的样衣 (`sample_status = 'COMPLETED'`)
2. 对每个已完成的样衣，查询其对应的报价单 (`StyleQuotation`)
3. 累加所有报价单的费用字段
4. 返回统计结果

### 时间范围支持
- **今日** (`day`): 当天 00:00 至当前时间
- **本周** (`week`): 本周一 00:00 至当前时间
- **本月** (`month`): 本月1日 00:00 至当前时间

### 空值处理
- 使用三元运算符处理 `null` 值，默认为 0.0
- 防止 `NullPointerException`

---

## 🔒 数据完整性保障

### 必要条件
样衣费用要被统计，需要满足：
1. ✅ `sample_status = 'COMPLETED'` (样衣完成状态)
2. ✅ `sample_completed_time` 在查询时间范围内
3. ✅ 存在对应的报价单记录 (`t_style_quotation.style_id = t_style_info.id`)
4. ✅ 报价单中有费用数据（`material_cost`, `process_cost`, `other_cost`）

### 缺失处理
- 如果样衣没有报价单，该样衣费用按 0 计算，不影响其他样衣的统计
- 如果报价单字段为 `null`，按 0.0 处理

    .in(StyleQuotation::getStyleId, styleIds)
    .list()

// 遍历时直接从 Map 获取
for (StyleInfo style : completedStyles) {

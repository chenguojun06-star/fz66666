## 🎨 核心设计规范（v2.0）

### 1. 按钮规范（镂空设计）

**浅色主题**：
```css
.ant-btn-primary {
  background: #ffffff;        /* 白色背景 */
  border: 2px solid #40a9ff;  /* 蓝色边框 */
  color: #40a9ff;             /* 蓝色文字 */
  border-radius: 999px;       /* 胶囊形 */
}

.ant-btn-primary:hover {
  background: #f0f9ff;        /* 浅蓝背景 */
  border-color: #1890ff;      /* 深蓝边框 */
}
```

**深色主题**：
```css
[data-theme="dark"] .ant-btn-primary {
  background: rgba(26, 32, 39, 0.8);  /* 深灰半透明 */
  border-color: #69b1ff;              /* 亮蓝边框 */
  color: #69b1ff;                     /* 亮蓝文字 */
}
```

**禁止事项**：
- ❌ 实心填充按钮（仅进度条可用深色渐变）
- ❌ 深色渐变按钮
- ❌ 直角按钮（border-radius < 12px）

### 2. 颜色规范

**主色调**：
- 浅色主题：`#40a9ff`（浅蓝）
- 深色主题：`#69b1ff`（亮蓝）

**功能色**：
- 成功：`#52c41a`（绿色）
- 警告：`#faad14`（橙色）
- 危险：`#ff4d4f`（红色）

**对比度要求**：
- 文字与背景：≥ 4.5:1
- 深色主题边框需比浅色更亮

### 3. 进度条（允许深色渐变）

```css
/* 成功状态 - 深绿渐变 */
.progress-success {
  background: linear-gradient(135deg, #52c41a 0%, #389e0d 100%);
  box-shadow: 0 2px 8px rgba(82, 196, 26, 0.35);
}

/* 警告状态 - 深橙渐变 */
.progress-warning {
  background: linear-gradient(135deg, #faad14 0%, #d48806 100%);
}

/* 危险状态 - 深红渐变 */
.progress-danger {
  background: linear-gradient(135deg, #ff4d4f 0%, #cf1322 100%);
}
```

**原因**：进度条是动态元素，需要深色渐变提高视觉对比度。

## 🏗️ 架构关键设计

### 1. Orchestrator 模式（后端核心）

**何时使用**：
- 跨多个 Service 协调操作
- 复杂事务管理
- 多步骤业务流程

**现有26个编排器**：
- 生产管理（8个）：ProductionOrder, CuttingTask, Scan...
- 对账（3个）：Shipment, Factory, Material
- 财务（3个）：Finance, Payroll, CostAnalysis
- 仓储、采购、工厂、系统...

**架构层次**：
```
Controller → Orchestrator → Service → Mapper
```

### 2. 前端组件规范

**通用组件**：
- `ResizableModal` - 统一弹窗（80vw × 85vh）
- `QRCodeBox` - 统一二维码（4种主题）
- `ModalContentLayout` - 弹窗内容布局（9个子组件）
- `UniversalCardView` - 通用卡片视图
- `StyleAttachmentsButton` - 附件管理（支持过滤放码纸样）

**使用示例**：
```tsx
import { ModalHeaderCard, ModalField, ModalPrimaryField } from '@/components/common/ModalContentLayout';

<ModalHeaderCard>
  <ModalPrimaryField label="订单号" value="PO20260122001" />
  <ModalField label="款号" value="ST001" />
</ModalHeaderCard>
```

### 3. SKU系统（三端统一）

**定义**：`styleNo(款号) + color(颜色) + size(尺码)`

**三种扫码模式**：
1. **订单扫码**：扫 `PO20260122001`，显示SKU明细表单
2. **菲号扫码**：扫 `PO20260122001-黑色-01`，自动识别菲号
3. **SKU扫码**：扫 SKU JSON，单个SKU直接提交

**智能工序识别**：
- 扫码次数决定工序（第1次=做领，第2次=上领...）
- 防重复保护：动态计算最小间隔
- 工序配置在订单的 `progressNodeUnitPrices`

- 实心填充按钮（主按钮必须镂空）

## 📞 问题反馈

### 设计规范变更流程
1. 提交设计变更提案
2. 团队评审讨论
3. 达成一致后更新文档
4. 全站同步更新

### 常见问题
- **为什么按钮是镂空的？** - 提高视觉清晰度，深色主题更友好
- **为什么禁止渐变？** - 简化设计，提高性能，静态元素不需要渐变
- **进度条为什么可以用深色？** - 动态元素需要高对比度，功能性优先

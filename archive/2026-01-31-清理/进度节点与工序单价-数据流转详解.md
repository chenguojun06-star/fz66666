## 📝 关键代码解析

### 1. 后端：解析进度节点

**文件**: `backend/src/main/java/com/fashion/supplychain/template/service/impl/TemplateLibraryServiceImpl.java`

```java
@Override
public List<Map<String, Object>> resolveProgressNodeUnitPrices(String styleNo) {
    // 1. 查找款号对应的进度模板
    TemplateLibrary tpl = resolveProgressTemplate(styleNo);
    List<Map<String, Object>> out = new ArrayList<>();

    // 2. 如果没有模板，返回空列表
    if (tpl == null || !StringUtils.hasText(tpl.getTemplateContent())) {
        return out;
    }

    try {
        // 3. 解析 JSON
        JsonNode root = objectMapper.readTree(tpl.getTemplateContent());
        JsonNode arr = root.get("nodes");
        
        if (arr != null && arr.isArray()) {
            for (JsonNode n : arr) {
                String name = n.get("name").asText("");
                String id = n.get("id").asText(name);
                
                // 4. 解析单价
                BigDecimal up = BigDecimal.ZERO;
                if (n.hasNonNull("unitPrice")) {
                    JsonNode v = n.get("unitPrice");
                    if (v.isNumber()) {
                        up = v.decimalValue();
                    }
                }
                
                // 5. 组装返回数据
                Map<String, Object> item = new LinkedHashMap<>();
                item.put("id", id);
                item.put("name", name);
                item.put("unitPrice", up.setScale(2, RoundingMode.HALF_UP));
                out.add(item);
            }
        }
    } catch (Exception e) {
        log.warn("Failed to resolve progress node unit prices", e);
    }

    return out;
}
```

### 2. 前端：解析订单节点

**文件**: `frontend/src/modules/production/pages/Production/ProgressDetail/index.tsx`

```typescript
// 从订单的 progressWorkflowJson 解析节点
const parseWorkflowNodesFromOrder = (order: ProductionOrder | null): ProgressNode[] => {
  // 1. 获取 JSON 字符串
  const raw = String(order?.progressWorkflowJson ?? '').trim();
  if (!raw) return [];
  
  // 2. 调用通用解析函数
  return parseProgressNodes(raw);
};

// 通用 JSON 解析函数
const parseProgressNodes = (raw: string): ProgressNode[] => {
  const text = String(raw ?? '').trim();
  if (!text) return [];
  
  try {
    const obj = JSON.parse(text);
    
    // 3. 支持两种格式：nodes (进度节点) 或 steps (工序单价)
    let itemsRaw = obj?.nodes;
    if (!Array.isArray(itemsRaw)) {
      itemsRaw = obj?.steps;
    }
    if (!Array.isArray(itemsRaw)) return [];

    // 4. 规范化数据
    const normalized: ProgressNode[] = itemsRaw
      .map((n: any) => {
        const name = String(n?.name || n?.processName || '').trim();
        const p = Number(n?.unitPrice);
        const unitPrice = Number.isFinite(p) && p >= 0 ? p : 0;
        const id = String(n?.id || n?.processCode || name || '');
        return { id, name, unitPrice };
      })
      .filter((n: ProgressNode) => n.name);
      
    // 5. 过滤掉出货节点
    return stripWarehousingNode(normalized);
  } catch {
    return [];
  }
};
```

### 3. 前端：选择节点逻辑

**文件**: `frontend/src/modules/production/pages/Production/ProgressDetail/index.tsx`

```typescript
// 为订单解析进度节点（优先级选择）
const resolveNodesForOrder = (order: ProductionOrder | null): ProgressNode[] => {
  // 1. 优先从订单自身读取
  const orderNodes = parseWorkflowNodesFromOrder(order);
  
  if (orderNodes.length) {
    // 2. 如果订单有节点，但单价都是0 → 从款号回填单价
    const styleNo = String(order?.styleNo || '').trim();
    const styleNodes = styleNo && progressNodesByStyleNo[styleNo] 
      ? progressNodesByStyleNo[styleNo] 
      : [];
    
    if (styleNodes.length > 0) {
      const hasAnyPrice = orderNodes.some(n => (Number(n.unitPrice) || 0) > 0);
      
      if (!hasAnyPrice) {
        // 创建名称到单价的映射（只保存有单价的工序）
        const priceMap = new Map<string, number>();
        styleNodes.forEach(sn => {
          const price = Number(sn.unitPrice) || 0;
          if (price > 0) {
            priceMap.set(sn.name, price);
          }
        });
        
        // 回填单价
        return orderNodes.map(n => ({
          ...n,
          unitPrice: priceMap.get(n.name) ?? (Number(n.unitPrice) || 0)
        }));
      }
    }
    
    return orderNodes;
  }
  
  // 3. 从款号带入（只带单价>0的工序）
  const sn = String(order?.styleNo || '').trim();
  if (sn && progressNodesByStyleNo[sn]?.length) {
    return progressNodesByStyleNo[sn].filter(n => (Number(n.unitPrice) || 0) > 0);
  }
  
  // 4. 使用默认节点
  return nodes?.length ? nodes : defaultNodes;
};
```

### 4. 前端：缓存款号节点

**文件**: `frontend/src/modules/production/pages/Production/ProgressDetail/index.tsx`

// 状态：款号 → 进度节点映射
const [progressNodesByStyleNo, setProgressNodesByStyleNo] = useState<Record<string, ProgressNode[]>>({});

// 从模板库加载款号节点（懒加载）
const ensureNodesFromTemplateIfNeeded = async (order: ProductionOrder) => {
  const styleNo = String(order.styleNo || '').trim();
  if (!styleNo) return;

  // 1. 检查缓存
  if (progressNodesByStyleNo[styleNo]?.length) {
    saveNodes(progressNodesByStyleNo[styleNo]);
    return;
  }

  try {
    // 2. 调用后端 API
    const res = await templateLibraryApi.progressNodeUnitPrices(styleNo);
    const result = res as Record<string, unknown>;
    
    if (result.code === 200) {
      const rows = Array.isArray(result.data) ? result.data : [];
      
      // 3. 规范化数据
      const normalized: ProgressNode[] = rows
        .map((n: any) => {
          const name = String(n?.name || '').trim();
          const id = String(n?.id || name || '').trim() || name;
          const p = Number(n?.unitPrice);
          const unitPrice = Number.isFinite(p) && p >= 0 ? p : 0;
          return { id, name, unitPrice };
        })
        .filter((n: ProgressNode) => n.name);
      
      const stripped = stripWarehousingNode(normalized);
      
      if (stripped.length) {
        // 4. 缓存到状态
        setProgressNodesByStyleNo(prev => ({ ...prev, [styleNo]: stripped }));
        saveNodes(stripped);
        return;
      }
    }
  } catch {
    // 忽略错误
  }
  
  // 5. 兜底使用默认节点
  saveNodes(defaultNodes);
};

## 🎯 关键流程总结

### 流程 1：订单创建时

```
1. 用户在订单管理创建订单
2. 选择款号
3. 后端查询款号的进度模板（templateLibraryApi.progressNodeUnitPrices）
4. 将模板节点保存到 progressWorkflowJson
5. 订单创建完成
```

### 流程 2：订单详情页打开时

```
1. 查询订单列表（包含 progressWorkflowJson）
2. 解析订单节点（parseWorkflowNodesFromOrder）
3. 如果订单无节点 → 查询款号模板
4. 如果订单有节点但单价为0 → 从款号回填单价
5. 渲染 HorizontalProgressPriceView
6. 查询扫码记录，计算统计数据
7. 显示进度条和单价
```

### 流程 3：小程序扫码时

```
1. 扫描订单二维码或菲号
2. 查询订单详情（包含 progressWorkflowJson）
3. 解析工序列表（SKUProcessor.getProcessListFromOrder）
4. 根据扫码次数判断当前工序
5. 显示工序名称和单价
6. 提交扫码记录
```

## 📋 常见问题

### Q1: 为什么订单没有显示工序单价？

**A**: 检查顺序：
1. 订单的 `progressWorkflowJson` 是否有值？
2. JSON 中的 `unitPrice` 是否 > 0？
3. 款号是否有进度模板？
4. 模板中的单价是否 > 0？

### Q2: 如何批量设置订单的工序单价？

**A**: 两种方式：
1. 为款号创建进度模板 → 新订单自动带入
2. 在订单详情页导入工序单价模板

### Q3: 订单节点和款号模板不一致怎么办？

**A**: 优先级：
- 订单自身节点 > 款号模板 > 默认节点
- 如果需要重置，删除订单的 `progressWorkflowJson`，重新从款号带入

### Q4: 为什么修改单价后刷新页面又变回去了？

**A**: 检查：
1. 是否点击了"保存"按钮？
2. 后端是否返回成功？
3. 数据库 `progressWorkflowJson` 是否更新？

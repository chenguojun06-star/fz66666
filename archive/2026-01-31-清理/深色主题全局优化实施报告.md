### 2. 关键样式规则

#### 侧边栏强制深色
```css
:root[data-theme="dark"] .layout-sidebar.ant-layout-sider {
  background: linear-gradient(180deg, #1a1d24 0%, #0f1115 100%) !important;
}
```

#### 顶部导航栏强制深色
```css
:root[data-theme="dark"] .layout-header {
  background-color: #1a1d24 !important;
  border-bottom-color: rgba(255, 255, 255, 0.1) !important;
}
```

#### 主内容区强制深色
```css
:root[data-theme="dark"] .layout-content {
  background-color: #0f1115 !important;
}
```

#### 表格全深色
```css
:root[data-theme="dark"] .ant-table-thead > tr > th {
  background: rgba(255, 255, 255, 0.04) !important;
  color: var(--neutral-text) !important;
}
```

#### 卡片半透明深色
```css
:root[data-theme="dark"] .ant-card {
  background: rgba(26, 29, 36, 0.6) !important;
  border-color: rgba(255, 255, 255, 0.1) !important;
}
```

### 3. 颜色规范统一

**核心颜色**：
- 主背景：`#1a1d24`（深灰蓝）
- 次要背景：`#0f1115`（更深灰）
- 主色调：`#6ba3ff`（亮蓝）
- 文本颜色：`#f5f5f5`（浅灰白）

**半透明效果**：

- hover背景：`rgba(107, 163, 255, 0.08)`

## ✅ 解决的问题

### 问题1: 侧边栏蓝色 → 已修复
- **原因**: CSS变量未正确应用 或 被其他样式覆盖
- **解决**: 使用 `!important` 强制覆盖为深灰色渐变
- **效果**: 侧边栏现在是深灰色（`#1a1d24` → `#0f1115` 渐变）

### 问题2: 顶部导航栏白色 → 已修复
- **原因**: `layout-header` 背景色使用 `var(--neutral-white)` 在浅色主题下是白色
- **解决**: 深色主题下强制覆盖为 `#1a1d24`
- **效果**: 顶部导航栏现在是深色

### 问题3: 表格白色 → 已修复
- **原因**: Ant Design Table 组件默认白色背景
- **解决**: 全局覆盖 `.ant-table-*` 所有相关类
- **效果**: 表格表头、行、边框全部深色

### 问题4: 卡片白色 → 已修复
- **原因**: `.ant-card` 默认白色背景
- **解决**: 全局覆盖为半透明深色背景
- **效果**: 所有卡片现在是深色半透明

### 问题5: 弹窗白色 → 已修复
- **原因**: Modal组件默认白色
- **解决**: 覆盖 `.ant-modal-content` 和相关元素
- **效果**: 所有弹窗现在是深色

## 🚀 使用说明

### 开发者无需额外操作
1. 切换主题开关到深色模式
2. 所有组件自动应用深色主题
3. 新增组件无需手动添加深色样式（除非有特殊需求）

### 如需自定义某个组件的深色样式
1. 在组件自己的 CSS 文件中添加：
```css
:root[data-theme="dark"] .your-custom-class {
  /* 你的自定义样式 */
}
```
2. 如需覆盖全局规则，使用 `!important`

## 🔧 技术细节

### 为什么使用 `!important`
- Ant Design 组件有自己的内联样式和高优先级CSS
- CSS变量在某些情况下优先级不足
- `!important` 确保深色主题样式一定生效

### 为什么创建独立文件
- 与 `global.css` 分离，职责更清晰
- 便于维护和调试
- 可以独立开关（注释导入即可禁用）

### 响应式支持
- 滚动条在所有设备上统一深色
- 移动端抽屉导航也应用深色主题
- hover/active状态统一蓝色高亮

## 📝 后续维护建议

1. **新增Ant Design组件时**：检查 `dark-theme-global.css` 是否需要添加对应规则
2. **修改主色调时**：只需修改 `--primary-color` 变量
3. **调整透明度时**：修改对应的 `rgba()` 值
4. **如发现某组件深色主题不生效**：在 `dark-theme-global.css` 添加对应规则并使用 `!important`

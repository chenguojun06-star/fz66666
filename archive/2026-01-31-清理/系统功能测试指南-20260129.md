## 📊 测试2: 进度流程工序单价显示

### 测试目标
验证工序单价卡片中的进度显示功能。

### 前置条件
- ✅ 系统中存在生产订单数据
- ✅ 订单包含工序单价配置

### 测试步骤

#### 步骤1: 访问进度流程页面
1. 导航至：**生产管理** → **进度流程**
2. 找到任意一个生产订单

#### 步骤2: 打开订单详情
1. 点击订单行的 **"查看详情"** 或 **"编辑"** 按钮
2. 在详情页中找到 **"工序单价"** 区域

#### 步骤3: 检查工序单价卡片
每个工序单价卡片应包含以下元素：

**卡片布局** (240px宽度):
```
┌─────────────────────────┐
│ 做领          85% ← 进度百分比（右上角）
│                         │
│ ━━━━━━━━━━━━━━ ← 进度条（颜色：灰/蓝/绿）
│                         │
│ 已完成: 425件           │
│ 剩余: 75件              │
└─────────────────────────┘
```

**进度条颜色规则**:
- **0%**: 灰色 (#d9d9d9) - 未开始
- **1-99%**: 蓝色 (#1890ff) - 进行中
- **100%**: 绿色 (#52c41a) - 已完成

#### 步骤4: 验证进度计算准确性
1. 找到一个已有扫码记录的工序
2. 检查进度百分比是否正确：`已完成数量 / 订单总数 × 100%`
3. 检查剩余数量是否正确：`订单总数 - 已完成数量`

**示例验证**:
```
订单总数: 500件
做领已完成: 425件
进度: 425/500 = 85%
剩余: 500-425 = 75件
```

#### 步骤5: 测试不同进度状态
找到不同进度的工序，验证颜色是否正确：

| 工序 | 已完成 | 总数 | 进度 | 预期颜色 |
|------|--------|------|------|----------|
| 做领 | 0 | 500 | 0% | 灰色 |
| 上领 | 250 | 500 | 50% | 蓝色 |
| 做袖 | 500 | 500 | 100% | 绿色 |

### 验收标准
- ✅ 工序单价卡片宽度为240px（而非之前的150px）
- ✅ 进度百分比显示在卡片右上角
- ✅ 进度条颜色根据完成度正确变化
- ✅ 已完成数量和剩余数量准确显示
- ✅ 卡片布局从横向改为纵向，视觉更清晰

### 对比说明
**修改前**（横向布局，150px宽度）:
```
┌───────────────┐
│ 做领  ¥2.50  │  ← 只显示工序名和单价
└───────────────┘
```

**修改后**（纵向布局，240px宽度）:
```
┌─────────────────────────┐
│ 做领          85%      │  ← 工序名 + 进度
│ ━━━━━━━━━━━━━━        │  ← 进度条
│ 已完成: 425件           │  ← 完成数量
│ 剩余: 75件              │  ← 剩余数量
└─────────────────────────┘
```

## 🎯 优化方案（按优先级）

### 🔴 P0: 高优先级（立即实施）

#### 1. 统一弹窗状态管理 Hook

**问题**：每个页面都重复这样的代码：
```tsx
const [visible, setVisible] = useState(false);
const [currentRecord, setCurrentRecord] = useState(null);

const handleOpen = (record) => {
  setCurrentRecord(record);
  setVisible(true);
};

const handleClose = () => {
  setVisible(false);
  setCurrentRecord(null);
};
```

**解决方案**：创建 `useModal` Hook
```tsx
// hooks/useModal.ts
export const useModal = <T = any>() => {
  const [visible, setVisible] = useState(false);
  const [data, setData] = useState<T | null>(null);

  const open = (record?: T) => {
    setData(record || null);
    setVisible(true);
  };

  const close = () => {
    setVisible(false);
    setData(null);
  };

  return {
    visible,
    data,
    open,
    close,
  };
};

// 使用示例
const detailModal = useModal<ProductionOrder>();

<Button onClick={() => detailModal.open(record)}>查看</Button>
<ResizableModal visible={detailModal.visible} onCancel={detailModal.close}>
  {detailModal.data && <div>{detailModal.data.orderNo}</div>}
</ResizableModal>
```

**影响**：50+ 页面可简化，每个页面减少 10-15 行代码

#### 2. 统一API错误处理

**问题**：每个API调用都重复这样的代码：
```tsx
try {
  const res = await api.xxx();
  if (res.code === 200) {
    message.success('操作成功');
    // 业务逻辑
  } else {
    message.error(res.message || '操作失败');
  }
} catch (error) {
  message.error(error.message || '操作失败');
}
```

**解决方案**：创建 `useRequest` Hook
```tsx
// hooks/useRequest.ts
export const useRequest = <T = any>() => {
  const [loading, setLoading] = useState(false);
  const { message } = App.useApp();

  const run = async (
    apiFunc: () => Promise<any>,
    options?: {
      successMsg?: string;
      errorMsg?: string;
      onSuccess?: (data: T) => void;
      onError?: (error: any) => void;
    }
  ) => {
    setLoading(true);
    try {
      const res = await apiFunc();
      if (res.code === 200) {
        options?.successMsg && message.success(options.successMsg);
        options?.onSuccess?.(res.data);
        return res.data;
      } else {
        const errMsg = res.message || options?.errorMsg || '操作失败';
        message.error(errMsg);
        options?.onError?.(res);
        throw new Error(errMsg);
      }
    } catch (error: any) {
      const errMsg = error.message || options?.errorMsg || '请求失败';
      message.error(errMsg);
      options?.onError?.(error);
      throw error;
    } finally {
      setLoading(false);
    }
  };

  return { run, loading };
};

// 使用示例
const { run, loading } = useRequest();

const handleSave = async () => {
  await run(
    () => productionOrderApi.update(data),
    {
      successMsg: '保存成功',
      errorMsg: '保存失败',
      onSuccess: () => fetchList(),
    }
  );
};
```

**影响**：200+ API调用可简化，每个调用减少 8-10 行代码

#### 3. 统一表格分页管理

**问题**：每个表格页面都重复：
```tsx
const [queryParams, setQueryParams] = useState({
  page: 1,
  pageSize: 10
});
const [total, setTotal] = useState(0);

const handlePageChange = (page, pageSize) => {
  setQueryParams({ ...queryParams, page, pageSize });
};
```

**解决方案**：创建 `useTablePagination` Hook
```tsx
// hooks/useTablePagination.ts
export const useTablePagination = (initialPageSize = 10) => {
  const [pagination, setPagination] = useState({
    current: 1,
    pageSize: initialPageSize,
    total: 0,
  });

  const onChange = (page: number, pageSize: number) => {
    setPagination(prev => ({
      ...prev,
      current: page,
      pageSize,
    }));
  };

  const setTotal = (total: number) => {
    setPagination(prev => ({ ...prev, total }));
  };

  const reset = () => {
    setPagination({
      current: 1,
      pageSize: initialPageSize,
      total: 0,
    });
  };

  return {
    pagination,
    onChange,
    setTotal,
    reset,
  };
};

// 使用示例
const { pagination, onChange, setTotal } = useTablePagination();

<ResizableTable
  pagination={{
    ...pagination,
    onChange,
  }}
/>
```

**影响**：40+ 表格页面可简化，每个页面减少 5-8 行代码

### 🟡 P1: 中优先级（1-2周内）

#### 4. 统一表格列定义

**问题**：相同字段在不同页面重复定义：
```tsx
// 订单号列在多个页面重复定义
{
  title: '订单号',
  dataIndex: 'orderNo',
  key: 'orderNo',
  width: 160,
}
```

**解决方案**：创建通用列定义库
```tsx
// constants/tableColumns.ts
export const CommonColumns = {
  orderNo: (options?: ColumnOptions) => ({
    title: '订单号',
    dataIndex: 'orderNo',
    key: 'orderNo',
    width: 160,
    ...options,
  }),
  
  styleNo: (options?: ColumnOptions) => ({
    title: '款号',
    dataIndex: 'styleNo',
    key: 'styleNo',
    width: 120,
    render: (text: string, record: any) => (
      <Space>
        <StyleCoverThumb styleNo={record.styleCover} size={32} />
        <span>{text}</span>
      </Space>
    ),
    ...options,
  }),
  
  createTime: (options?: ColumnOptions) => ({
    title: '创建时间',
    dataIndex: 'createTime',
    key: 'createTime',
    width: 160,
    render: (text: string) => formatDateTime(text),
    ...options,
  }),
  
  status: (statusMap: Record<string, StatusConfig>) => ({
    title: '状态',
    dataIndex: 'status',
    key: 'status',
    width: 100,
    render: (status: string) => (
      <Tag color={statusMap[status]?.color}>
        {statusMap[status]?.text || status}
      </Tag>
    ),
  }),
  
  actions: (actions: ActionConfig[]) => ({
    title: '操作',
    key: 'actions',
    fixed: 'right',
    width: 120,
    render: (_, record) => <RowActions record={record} actions={actions} />,
  }),
};

// 使用示例
const columns = [
  CommonColumns.orderNo(),
  CommonColumns.styleNo(),
  CommonColumns.createTime(),
  CommonColumns.status(ORDER_STATUS_MAP),
  CommonColumns.actions([
    { key: 'view', label: '查看', onClick: handleView },
    { key: 'edit', label: '编辑', onClick: handleEdit },
  ]),
];
```

**影响**：30+ 表格可复用，提升一致性

#### 6. 统一表单验证

**问题**：部分页面使用 `Form.useForm()`，部分手动验证
```tsx
// 方式1: Ant Design Form
const [form] = Form.useForm();

// 方式2: 手动验证
if (!data.orderNo) {
  message.error('请输入订单号');
  return;
}
```

**解决方案**：统一使用 Ant Design Form + 通用验证规则
```tsx
// utils/validationRules.ts (已存在，需扩展)
export const ValidationRules = {
  required: (label: string) => ({
    required: true,
    message: `请输入${label}`,
  }),
  
  orderNo: [
    { required: true, message: '请输入订单号' },
    { pattern: /^[A-Z]{2}\d{11}$/, message: '订单号格式不正确' },
  ],
  
  quantity: [
    { required: true, message: '请输入数量' },
    { type: 'number', min: 1, message: '数量必须大于0' },
  ],
  
  email: [
    { type: 'email', message: '邮箱格式不正确' },
  ],
  
  phone: [
    { pattern: /^1[3-9]\d{9}$/, message: '手机号格式不正确' },
  ],
};

// 使用示例
<Form.Item name="orderNo" rules={ValidationRules.orderNo}>
  <Input />
</Form.Item>
```

**影响**：30+ 表单页面统一验证逻辑

## ⚠️ 注意事项

### 1. 不破坏现有功能
- 新Hook只是封装，不改变业务逻辑
- 支持渐进式迁移，旧代码继续工作
- 充分测试后再大规模推广

### 2. 保持灵活性
- Hook提供可选参数，支持自定义
- 不强制使用，开发者可选择
- 特殊场景允许不使用Hook

### 3. 文档同步更新
- 为每个Hook编写使用文档
- 提供迁移指南
- 更新开发规范

### 4. 团队培训
- 组织Hook使用培训
- 建立示例代码库
- Code Review时推广最佳实践

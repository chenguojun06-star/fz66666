# 样衣工序显示修复和推送功能 - 测试指南

## 修复内容

### 1. 工序数据显示问题 ✅
**问题**：用户在工序单价弹窗中设置了工序，但主页面没有显示

**原因**：
- 主页面原本从模板库加载数据（`/template-library/process-unit-prices`）
- 工序单价弹窗保存到工序表（`t_style_process`）
- 数据源不一致导致显示不出来

**修复方案**：
```typescript
// 修改前：从模板库加载
const processRes = await api.get('/template-library/process-unit-prices', { 
  params: { styleNo } 
});

// 修改后：从工序表加载
const processRes = await api.get('/api/style/process/list', { 
  params: { styleId } 
});
```

**涉及文件**：
- [frontend/src/modules/basic/pages/StyleInfo/index.tsx](frontend/src/modules/basic/pages/StyleInfo/index.tsx#L514-L580)

### 2. 推送到下单功能 ✅
**新增功能**：样衣开发完成后，可推送到生产订单系统

**功能特性**：
- ✅ 推送按钮在工序配置完成后才启用
- ✅ 推送弹窗包含订单关键信息（数量、工厂、日期）
- ✅ 自动复制工序单价到订单
- ✅ 推送成功后跳转到订单详情页

**UI 位置**：
样衣开发详情页 → 基础信息卡片 → "推送到下单"按钮（第10行）

**弹窗字段**：
| 字段 | 类型 | 是否必填 | 说明 |
|------|------|---------|------|
| 款号 | 文本 | 是 | 自动填充，不可编辑 |
| 款名 | 文本 | 是 | 自动填充，不可编辑 |
| 订单数量 | 数字 | 是 | 默认100，可修改 |
| 生产工厂 | 下拉选择 | 是 | 选择工厂 |
| 计划开始日期 | 日期 | 是 | 默认今天 |
| 计划结束日期 | 日期 | 是 | 默认30天后 |
| 备注 | 文本域 | 否 | - |

## 后端API需求

推送功能需要后端实现以下API：

### API: 从样衣创建生产订单

**请求**：
```http
POST /api/production/order/create-from-style
Content-Type: application/json

{
  "styleId": 9,
  "styleNo": "ST001",
  "styleName": "春季新款上衣",
  "orderQuantity": 500,
  "plannedStartDate": "2026-02-01",
  "plannedEndDate": "2026-03-01",
  "factoryId": 1,
  "remark": "春季新款紧急订单"
}
```

**响应**：
```json
{
  "code": 200,
  "message": "推送成功",
  "data": {
    "orderNo": "PO20260127001",
    "orderId": 123
  }
}
```

**业务逻辑**：
1. 创建生产订单记录
2. 从 `t_style_process` 复制工序数据到订单工序表
3. 从 `t_style_size_price` 复制多码单价（如果有）
4. 返回订单号供前端跳转

## 数据流图

### 工序数据流

```
用户操作
  ↓
工序单价弹窗
  ↓
保存到 t_style_process 表
  ↓
关闭弹窗，触发刷新
  ↓
调用 /api/style/process/list
  ↓
主页面显示工序标签
```

### 推送数据流

```
用户点击"推送到下单"
  ↓
打开推送弹窗
  ↓
自动填充样衣信息
  ↓
用户填写订单数量、工厂、日期
  ↓
提交表单
  ↓
POST /api/production/order/create-from-style
  ↓
后端创建订单 + 复制工序数据
  ↓
返回订单号
  ↓
前端跳转到订单详情页
```

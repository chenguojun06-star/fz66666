# 二次工艺管理功能完成

## 📋 功能概述

在生产管理模块中添加了 **二次工艺管理** 页面，用于管理服装生产过程中的二次加工工艺（刺绣、印花、水洗、染色、烫印等）。

## ✅ 已完成内容

### 1. 前端页面 ✅

#### 创建的文件
- **页面组件**: `frontend/src/modules/production/pages/Production/SecondaryProcess/index.tsx` (581行)
- **样式文件**: `frontend/src/modules/production/pages/Production/SecondaryProcess/styles.css`

#### 页面功能特性
- 📊 **列表展示**: 显示所有二次工艺记录，包含订单号、款号、工艺类型、数量、单价等
- 🔍 **搜索筛选**: 支持按订单编号、款号、工艺类型、状态筛选
- ➕ **新建工艺单**: 弹窗表单，支持创建新的二次工艺记录
- ✏️ **编辑功能**: 修改现有工艺单信息
- 👁️ **查看详情**: 只读模式查看工艺单
- 🗑️ **删除功能**: 删除工艺单（带二次确认）
- 💰 **自动计算**: 单价×数量自动计算总价
- 📱 **响应式设计**: 支持PC和移动端

#### 工艺类型
- 刺绣 (embroidery)
- 印花 (printing)
- 水洗 (washing)
- 染色 (dyeing)
- 烫印 (ironing)
- 其他 (other)

#### 状态管理
- 待处理 (pending) - 灰色
- 处理中 (processing) - 蓝色
- 已完成 (completed) - 绿色
- 已取消 (cancelled) - 红色

### 2. 路由配置 ✅

#### 修改的文件

- `frontend/src/modules/production/index.tsx` - 导出组件

#### 路由信息
- **路径**: `/production/secondary-process`
- **权限码**: `MENU_SECONDARY_PROCESS`
- **菜单位置**: 生产管理 → 二次工艺 (在裁剪管理和生产进度之间)

### 3. 权限配置脚本 ✅

#### 创建的文件
- `scripts/add_secondary_process_permissions.sql`

#### 权限内容
```sql
-- 菜单权限
MENU_SECONDARY_PROCESS - 二次工艺管理

-- 操作权限
SECONDARY_PROCESS_CREATE - 创建二次工艺
SECONDARY_PROCESS_EDIT - 编辑二次工艺
SECONDARY_PROCESS_DELETE - 删除二次工艺
SECONDARY_PROCESS_VIEW - 查看二次工艺
```

#### 默认权限分配
- **管理员**: 全部权限 (查看/新建/编辑/删除)
- **主管**: 全部权限 (查看/新建/编辑/删除)
- **厂长**: 查看、新建、编辑
- **操作员**: 仅查看

### 数据库表设计建议
```sql
CREATE TABLE `t_secondary_process` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
  `production_order_id` BIGINT NOT NULL COMMENT '生产订单ID',
  `production_order_no` VARCHAR(50) NOT NULL COMMENT '订单编号',
  `style_no` VARCHAR(50) NOT NULL COMMENT '款号',
  `style_name` VARCHAR(100) COMMENT '款式名称',
  `process_type` VARCHAR(20) NOT NULL COMMENT '工艺类型',
  `process_name` VARCHAR(100) NOT NULL COMMENT '工艺名称',
  `quantity` INT NOT NULL DEFAULT 0 COMMENT '数量',
  `unit_price` DECIMAL(10, 2) NOT NULL DEFAULT 0 COMMENT '单价',
  `total_price` DECIMAL(10, 2) NOT NULL DEFAULT 0 COMMENT '总价',
  `factory_id` BIGINT COMMENT '加工厂ID',
  `factory_name` VARCHAR(100) COMMENT '加工厂名称',
  `status` VARCHAR(20) NOT NULL DEFAULT 'pending' COMMENT '状态',
  `remark` TEXT COMMENT '备注',
  `created_by` VARCHAR(50) COMMENT '创建人',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_by` VARCHAR(50) COMMENT '更新人',
  `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  INDEX `idx_order_no` (`production_order_no`),
  INDEX `idx_style_no` (`style_no`),
  INDEX `idx_status` (`status`),
  INDEX `idx_created_at` (`created_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='二次工艺表';
```

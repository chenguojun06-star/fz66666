# 主页顶部4个统计看板 - 测试指南

## 📊 功能概述

将原有的8个统计卡片替换为4个胶囊形状的核心业务看板，支持日/周/月/年时间维度筛选。

### 4个核心指标

1. **样衣开发数量** - 紫色渐变（#a78bfa → #8b5cf6）
2. **大货下单数量** - 蓝色渐变（#60a5fa → #3b82f6）
3. **裁剪数量** - 橙色渐变（#fbbf24 → #f59e0b）
4. **出入库数量** - 绿色渐变（#34d399 → #10b981）

## 🎨 设计特点

- **形状**：胶囊形状（border-radius: 24px）
- **配色**：柔和渐变色，支持深色主题
- **布局**：4列网格，响应式（<1200px显示2列，<768px显示1列）
- **交互**：悬浮上浮效果，阴影加深
- **时间筛选**：顶部圆角按钮组（日/周/月/年）

## 🔧 技术实现

### 前端组件

**文件**：`frontend/src/modules/dashboard/components/TopStats/`
- `index.tsx` - 主组件，包含时间选择和数据展示
- `styles.css` - 胶囊卡片样式

**关键代码**：
```typescript
interface TopStatsData {
  sampleDevelopmentCount: number;  // 样衣开发
  bulkOrderCount: number;          // 大货下单
  cuttingCount: number;            // 裁剪数量
  warehousingCount: number;        // 出入库
}

type TimeRange = 'day' | 'week' | 'month' | 'year';
```

### 后端API

**接口**：`GET /api/dashboard/top-stats?range={day|week|month|year}`

**实现文件**：
- `DashboardController.java` - 新增 `/top-stats` 端点
- `DashboardOrchestrator.java` - `getTopStats()` 业务逻辑
- `DashboardQueryServiceImpl.java` - 数据库查询实现
- `TopStatsResponse.java` - 响应DTO

**时间计算逻辑**：
- `day`: 今日00:00 至 当前时间
- `week`: 本周一00:00 至 当前时间
- `month`: 本月1日00:00 至 当前时间
- `year`: 今年1月1日00:00 至 当前时间

### 数据库查询

1. **样衣开发** - `t_style_info` 表，`is_sample=1` 且 `create_time` 在范围内
2. **大货下单** - `t_production_order` 表，`create_time` 在范围内的订单数
3. **裁剪数量** - `t_production_order` 表，`cutting_quantity` 字段求和
4. **出入库数量** - `t_product_warehousing` 表，`qualified_quantity + unqualified_quantity` 求和

## 🐛 常见问题

### 1. 数据全部显示为0

**原因**：数据库中可能没有测试数据

**解决**：
```bash
# 创建测试订单
cd scripts
mysql -u root -p fashion_supplychain < create-test-data.sql
```

### 2. API返回403错误

**原因**：未登录或token过期

**解决**：
1. 清除浏览器缓存
2. 重新登录系统
3. 检查localStorage中的token

### 3. 后端启动失败

**原因**：可能是环境变量未加载

**解决**：
```bash
# 确保.run/backend.env文件存在
cat .run/backend.env

# 使用start-backend.sh脚本（自动加载环境变量）
./start-backend.sh
```

### 4. 页面样式错乱

**原因**：CSS未正确加载

**解决**：
```bash
cd frontend
rm -rf node_modules/.vite
npm run dev
```

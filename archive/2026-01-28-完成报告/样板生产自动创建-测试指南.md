# 样板生产自动创建功能 - 测试指南

## 📌 功能概述

**业务逻辑**: 样衣开发下单 → 样板生产自动创建待领取记录

当创建新的样衣开发记录时，系统会自动在"生产-样板生产"模块创建一条状态为 `PENDING`（待领取）的样板生产记录。

---

## ✅ 后端实现

### 1. 数据库表
已创建 `t_pattern_production` 表：
```sql
-- 表字段
- id (主键)
- style_id (款号ID，关联 t_style_info)
- style_no (款号)
- color (颜色)
- quantity (数量)
- release_time (下板时间)
- delivery_time (交板时间)
- receiver (领取人)
- receive_time (领取时间)
- complete_time (完成时间)
- pattern_maker (纸样师傅)
- progress_nodes (工序进度JSON)
- status (状态: PENDING/IN_PROGRESS/COMPLETED)
```

### 2. 自动创建逻辑
位置：`StyleInfoOrchestrator.save()` 方法

触发时机：新增样衣开发记录时

自动创建内容：
```java
PatternProduction record = {
  styleId: 新款号的ID,
  styleNo: 款号,
  status: "PENDING",
  progressNodes: '{"cutting":0,"sewing":0,"ironing":0,"quality":0,"secondary":0,"packaging":0}'
}
```

### 3. REST API
- `GET /production/pattern/list` - 分页查询样板生产列表
- `GET /production/pattern/{id}` - 获取样板生产详情
- `POST /production/pattern/{id}/receive` - 领取样板（PENDING → IN_PROGRESS）
- `POST /production/pattern/{id}/progress` - 更新工序进度
- `DELETE /production/pattern/{id}` - 软删除

## ⚠️ 常见问题

### 问题1: 样板生产列表为空

**可能原因**：
- 后端服务未启动
- 数据库表未创建
- API 路由配置错误

**解决方案**：
```bash
# 1. 检查后端是否运行
ps aux | grep spring-boot | grep -v grep

# 2. 检查表是否存在
docker exec -i fashion-mysql-simple mysql -uroot -pchangeme fashion_supplychain \
  -e "SHOW TABLES LIKE 't_pattern_production';"

# 3. 重启后端
cd backend
mvn spring-boot:run
```

---

### 问题2: 创建样衣开发后没有自动创建样板生产

**可能原因**：
- `PatternProductionService` 注入失败
- 自动创建逻辑异常（已 try-catch，不影响主流程）

**排查方法**：
1. 检查后端日志是否有错误
2. 数据库查询是否有新记录
3. 添加断点到 `StyleInfoOrchestrator.createPatternProductionRecord()`

---

### 问题3: 液体进度显示为静态圆圈

**可能原因**：
- `LiquidProgressLottie` 组件导入失败
- CSS 样式未加载

**解决方案**：
1. 确认组件路径：`@/components/common/LiquidProgressLottie`
2. 检查浏览器控制台是否有导入错误
3. 刷新页面清除缓存

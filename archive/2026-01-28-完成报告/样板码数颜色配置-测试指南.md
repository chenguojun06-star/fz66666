# 样板码数颜色配置保存功能 - 测试指南

## 📅 实现日期
2026-01-28

## ✨ 功能说明

在样衣详情页面的"基础信息"标签页中，新增的**码数颜色配置表**现在可以完整保存和加载数据了。

## 🎯 功能特性

### 1. 数据保存
点击"保存基础信息"按钮时，会保存以下所有数据：
- ✅ 5个尺码值（size1-5）：如 S, M, L, XL, XXL
- ✅ 5个颜色值（color1-5）：如 黑色, 白色, 灰色, 蓝色, 红色
- ✅ 5个数量值（qty1-5）：自动累加到"样板数量"字段
- ✅ 常用尺码标签（commonSizes）：右侧快捷标签列表
- ✅ 常用颜色标签（commonColors）：右侧快捷标签列表

### 2. 数据加载
打开已有款号时，会自动恢复所有配置：
- ✅ 表格中的尺码、颜色、数量
- ✅ 右侧的快捷标签列表
- ✅ 自动累加的样板数量

### 3. 锁定机制
- ✅ 保存成功后自动锁定表格，无法编辑
- ✅ 点击"退回"按钮可解除锁定，重新编辑
- ✅ 锁定状态下所有输入框显示 disabled

## 🔧 技术实现

### 前端改动
1. **类型定义**（`frontend/src/types/style.ts`）
   ```typescript

     // ... 其他字段
     sizeColorConfig?: string; // JSON字符串
   }
   ```

2. **保存逻辑**（`StyleInfo/index.tsx` 的 `handleSave`）
   ```typescript
   const sizeColorConfig = {
     sizes: [size1, size2, size3, size4, size5],
     colors: [color1, color2, color3, color4, color5],
     quantities: [qty1, qty2, qty3, qty4, qty5],
     commonSizes,
     commonColors
   };
   normalizedValues.sizeColorConfig = JSON.stringify(sizeColorConfig);
   ```

3. **加载逻辑**（`StyleInfo/index.tsx` 的 `fetchDetail`）
   - 解析 JSON 字符串
   - 恢复所有 state 值（setSize1-5, setColor1-5, setQty1-5）
   - 恢复快捷标签列表

### 后端改动

2. **数据库表**（`t_style_info`）

## 🗄️ 数据结构示例

保存到数据库的 JSON 格式：
```json
{
  "sizes": ["S", "M", "L", "XL", "XXL"],
  "colors": ["黑色", "白色", "灰色", "蓝色", "红色"],
  "quantities": [50, 100, 150, 100, 50],
  "commonSizes": ["XS", "S", "M", "L", "XL", "XXL", "3XL"],
  "commonColors": ["黑色", "白色", "灰色", "蓝色", "红色", "粉色"]
}
```

## ⚠️ 注意事项

1. **数据格式**：后端存储为 TEXT 类型的 JSON 字符串
2. **向后兼容**：旧数据（没有 sizeColorConfig 字段）不会报错，使用默认值
3. **数量同步**：表格中的数量会自动累加到"样板数量"字段，两者保持一致
4. **锁定状态**：只有在"已保存"状态下才会锁定，新建时不锁定

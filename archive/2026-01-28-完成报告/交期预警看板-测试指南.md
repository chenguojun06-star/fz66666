# 交期预警看板功能测试指南

## 功能概述

在仪表盘页面添加了**交期预警看板**，位于统计卡片和"最近动态/快捷入口"之间，用于实时展示即将到期的订单。

## 设计规范

### 分类规则
- **紧急订单**（红色）：距离交期 1-4 天的订单
- **预警订单**（黄色）：距离交期 5-7 天的订单
- 只显示未完成的订单（status != 'completed' && status != 'cancelled'）
- 只显示有交期的订单（plannedEndDate != null）

### 视觉设计
- **紧急订单**：红色主题（#ff4d4f），WarningOutlined 图标
- **预警订单**：黄色主题（#faad14），ClockCircleOutlined 图标
- 卡片采用渐变背景，hover 时有位移和阴影效果
- 左右两列布局，移动端自适应为单列

### 订单信息展示
每个订单卡片显示：
- 订单号 + 剩余天数徽章
- 款号 - 款名
- 工厂名称 🏭
- 完成情况 📦 (完成数量/订单数量)
- 生产进度 📊 (百分比)

### 交互功能
- 点击订单卡片，直接跳转到生产订单页面并定位到该订单
- 没有预警订单时，整个看板自动隐藏

## 技术实现

### 前端
1. **组件**：`frontend/src/modules/dashboard/components/DeliveryAlert.tsx`
2. **样式**：`frontend/src/modules/dashboard/components/DeliveryAlert.css`
3. **集成**：在 `Dashboard/index.tsx` 中引入并放置在统计卡片下方

### 后端
1. **Controller**：`DashboardController.java` 添加 `/api/dashboard/delivery-alert` 接口
2. **Orchestrator**：`DashboardOrchestrator.java` 实现 `getDeliveryAlert()` 方法
3. **DTO**：
   - `DeliveryAlertOrderDto.java`：订单信息
   - `DeliveryAlertResponse.java`：响应数据（urgentOrders + warningOrders）

## 常见问题

### Q1: 看板不显示？
- 检查是否有距离交期 1-7 天的未完成订单
- 打开浏览器控制台查看是否有 API 错误
- 检查后端服务是否正常运行

### Q2: 天数计算不准确？
- 确认服务器时区设置正确
- 检查订单的 `plannedEndDate` 字段格式

### Q3: 点击订单跳转失败？
- 检查路由配置是否正确
- 确认生产订单页面支持 `orderNo` 参数过滤

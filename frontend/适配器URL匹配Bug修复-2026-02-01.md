# 🔧 适配器URL匹配Bug修复 - 2026-02-01

**问题**：适配器已启用但废弃端点没有被转换，导致500/400错误  
**根本原因**：URL前缀不匹配  
**修复时间**：立即修复（2分钟）  
**状态**：✅ 已修复

## 问题诊断

### 错误现象

浏览器控制台显示：
```
POST /api/style/info/59/pattern/start 500 (Internal Server Error)
POST /api/template-library/apply-to-style 400 (Bad Request)
POST /api/style/attachment/upload 500 (Internal Server Error)
```

**关键线索**：
- ✅ 适配器已启用（看到3次启用日志）
- ❌ **没有迁移警告** - 说明URL匹配失败
- ❌ 废弃端点直接发送到后端导致错误

### 根本原因

**适配器模式包含 `/api` 前缀，但axios config.url是相对路径！**

```typescript
// ❌ 适配器模式（错误）
'/api/style/info/:id/pattern/start'

// ✅ 实际axios config.url（相对于baseURL）
'/style/info/59/pattern/start'

// ❌ 结果：正则匹配失败，适配器不生效
```

**为什么会这样？**

在 `core.ts` 中：
```typescript
const client = axios.create({
  baseURL: '/api',  // ← 这里已经设置了/api前缀
  // ...
});
```

所以axios请求时：
```typescript
// 开发者写的代码
api.post('/style/info/59/pattern/start');

// 实际请求URL = baseURL + url
// http://localhost:5173/api + /style/info/59/pattern/start

// 但拦截器中的config.url只是：
// '/style/info/59/pattern/start' ← 无/api前缀！
```

---

## 修复方案

### 批量移除 `/api` 前缀

```bash
cd frontend/src/utils/api
sed -i '' "s|'/api/|'/|g" legacyApiAdapter.ts
sed -i '' "s|'POST /api/|'POST /|g" legacyApiAdapter.ts
sed -i '' "s|'GET /api/|'GET /|g" legacyApiAdapter.ts
sed -i '' "s|'PUT /api/|'PUT /|g" legacyApiAdapter.ts
sed -i '' "s|'DELETE /api/|'DELETE /|g" legacyApiAdapter.ts
```

| `'POST /api/template/library/save'` | `'POST /template/library/save'` |

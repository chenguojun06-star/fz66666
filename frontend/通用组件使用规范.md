# 通用组件使用规范（强制执行）

> **更新日期**: 2026-02-02  
> **适用范围**: 全系统前端开发  
> **设计原则**: 统一、简洁、高复用、强约束

---

## 📐 核心原则（不可违背）

1. **强制使用标准组件** - 禁止自定义样式的原生组件
2. **统一尺寸规范** - 按钮/输入框/弹窗严格遵循三级尺寸
3. **一致的间距** - 所有间距使用 8 的倍数（8/16/24/32px）
4. **纯色设计** - 禁止渐变，使用 CSS 变量
5. **强类型约束** - TypeScript 严格模式，杜绝 any

## 一、基础 UI 组件（12 个标准）

### 1. 按钮组件 (Button)

#### ✅ 标准用法
```tsx
import { Button, Space } from 'antd';

// 主要操作按钮
<Button type="primary">提交</Button>

// 次要操作按钮
<Button>取消</Button>

// 危险操作按钮
<Button danger>删除</Button>

// 按钮组合（固定 8px 间距）
<Space size={8}>
  <Button type="primary">保存</Button>
  <Button>取消</Button>
</Space>
```

#### ❌ 禁止用法
```tsx
// ❌ 禁止自定义颜色
<Button style={{ background: '#ff0000' }}>删除</Button>

// ❌ 禁止渐变背景
<Button style={{ background: 'linear-gradient(...)' }}>提交</Button>

// ❌ 禁止不标准的间距
<Space size={10}>  // 必须是 8 的倍数
  <Button>按钮1</Button>
  <Button>按钮2</Button>
</Space>
```

#### 三级尺寸规范
```tsx
// 大按钮（主要操作）
<Button size="large">提交订单</Button>

// 中等按钮（默认，常规操作）
<Button>保存</Button>

// 小按钮（表格操作、辅助按钮）
<Button size="small">编辑</Button>
```

### 2. 输入框组件 (Input)

#### ✅ 弹窗内输入框（ModalFieldRow 包裹）
```tsx
import { ModalContentLayout, ModalFieldRow } from '@/components/common/ModalContentLayout';
import { Input } from 'antd';

<ModalContentLayout>
  <ModalFieldRow label="款式编号" required>
    <Input 
      placeholder="请输入款式编号" 
      maxLength={50}
    />
  </ModalFieldRow>
  
  <ModalFieldRow label="订单数量" required>
    <InputNumber 
      placeholder="请输入数量" 
      min={0}
      style={{ width: '100%' }}
    />
  </ModalFieldRow>
</ModalContentLayout>
```

**固定规则**：
- ✅ 弹窗内所有输入框必须用 `ModalFieldRow` 包裹
- ✅ 字段间距固定 24px（由 ModalFieldRow 控制）
- ✅ 必填字段标记 `required` 属性
- ✅ 输入框宽度默认 100%，不需手动设置

#### ✅ 页面内输入框（Form.Item 包裹）
```tsx
import { Form, Input, InputNumber } from 'antd';

<Form layout="horizontal" labelCol={{ span: 6 }} wrapperCol={{ span: 18 }}>
  <Form.Item 
    label="款式编号" 
    name="styleNo" 
    rules={[{ required: true, message: '请输入款式编号' }]}
  >
    <Input placeholder="请输入款式编号" maxLength={50} />
  </Form.Item>
  
  <Form.Item 
    label="订单数量" 
    name="quantity"
    rules={[{ required: true, message: '请输入数量' }]}
  >
    <InputNumber 
      placeholder="请输入数量" 
      min={0} 
      style={{ width: '100%' }}
    />
  </Form.Item>
</Form>
```

**固定规则**：
- ✅ 页面表单使用 `Form.Item` 统一布局
- ✅ 标签宽度固定：`labelCol={{ span: 6 }}`
- ✅ 输入区宽度固定：`wrapperCol={{ span: 18 }}`
- ✅ 校验规则统一放在 `rules` 中

#### ❌ 禁止用法
```tsx
// ❌ 弹窗内不使用 ModalFieldRow
<div style={{ marginBottom: 16 }}>  // 禁止自定义间距
  <label>款式编号</label>
  <Input />
</div>

// ❌ 页面内不使用 Form.Item
<div className="form-row">  // 禁止自定义表单行
  <span>款式编号:</span>
  <Input />
</div>

// ❌ InputNumber 不设置宽度
<InputNumber />  // 默认宽度只有 90px，必须设置 100%
```

### 3. 下拉选择组件 (Select)

#### ✅ 标准用法
```tsx
import { Select } from 'antd';

// 单选
<Select 
  placeholder="请选择"
  allowClear
  showSearch
  optionFilterProp="label"
  options={[
    { label: '选项1', value: '1' },
    { label: '选项2', value: '2' },
  ]}
/>

// 多选
<Select 
  mode="multiple"
  placeholder="请选择"
  allowClear
  maxTagCount={2}
  options={options}
/>
```

**固定规则**：
- ✅ 必须设置 `placeholder`
- ✅ 必须启用 `allowClear`（清除按钮）
- ✅ 可搜索场景启用 `showSearch`
- ✅ 宽度默认 100%，无需手动设置

### 4. 弹窗组件 (ResizableModal)

#### ✅ 三级尺寸规范（强制执行）

```tsx
import ResizableModal from '@/components/common/ResizableModal';
import { ModalContentLayout, ModalFieldRow } from '@/components/common/ModalContentLayout';

// 大窗口（60vw × 60vh）- 复杂表单、多Tab、详情页
<ResizableModal
  title="编辑生产订单"
  visible={visible}
  onCancel={() => setVisible(false)}
  defaultWidth="60vw"
  defaultHeight="60vh"
>
  <ModalContentLayout>
    {/* 内容 */}
  </ModalContentLayout>
</ResizableModal>

// 中窗口（40vw × 50vh）- 普通表单、常规编辑
<ResizableModal
  title="新增款式"
  visible={visible}
  onCancel={() => setVisible(false)}
  defaultWidth="40vw"
  defaultHeight="50vh"
>
  <ModalContentLayout>
    {/* 内容 */}
  </ModalContentLayout>
</ResizableModal>

// 小窗口（30vw × 40vh）- 确认对话框、简单表单
<ResizableModal
  title="确认删除"
  visible={visible}
  onCancel={() => setVisible(false)}
  defaultWidth="30vw"
  defaultHeight="40vh"
>
  <ModalContentLayout>
    <p>确定要删除该记录吗？</p>
  </ModalContentLayout>
</ResizableModal>
```

#### ❌ 禁止用法
```tsx
// ❌ 禁止自定义尺寸
<ResizableModal defaultWidth="55vw" defaultHeight="65vh">

// ❌ 禁止使用 Ant Design 原生 Modal
<Modal width={800}>  // 必须用 ResizableModal

// ❌ 弹窗内容不使用 ModalContentLayout
<ResizableModal>
  <div style={{ padding: 20 }}>  // 禁止自定义内边距
    {/* 内容 */}
  </div>
</ResizableModal>
```

### 5. 表格操作列组件 (RowActions)

#### ✅ 标准用法（统一操作按钮）
```tsx
import RowActions from '@/components/common/RowActions';

const columns = [
  // ... 其他列
  {
    title: '操作',
    key: 'action',
    width: 150,
    fixed: 'right' as const,
    render: (_: any, record: any) => (
      <RowActions
        actions={[
          {
            key: 'edit',
            label: '编辑',
            primary: true,  // 主要操作，优先显示
            onClick: () => handleEdit(record),
          },
          {
            key: 'delete',
            label: '删除',
            danger: true,
            onClick: () => handleDelete(record),
          },
          {
            key: 'log',
            label: '日志',
            onClick: () => handleLog(record),
          },
        ]}
      />
    ),
  },
];
```

**固定规则**：
- ✅ 操作列宽度固定 150px
- ✅ 操作列固定在右侧 `fixed: 'right'`
- ✅ 主要操作（编辑）标记 `primary: true`，优先显示
- ✅ 危险操作（删除）标记 `danger: true`，显示红色
- ✅ 超过 1 个操作自动折叠到"更多"下拉菜单
- ✅ 日志操作自动隐藏到下拉菜单

#### ❌ 禁止用法
```tsx
// ❌ 禁止自定义操作列样式
{
  title: '操作',
  render: (record) => (
    <Space>  // 禁止自行组合按钮
      <Button size="small" onClick={() => handleEdit(record)}>编辑</Button>
      <Button size="small" danger onClick={() => handleDelete(record)}>删除</Button>
    </Space>
  )
}

// ❌ 禁止不同页面使用不同的操作列宽度
{ title: '操作', width: 180 }  // 必须统一 150px
```

### 6. 日期选择器 (UnifiedDatePicker)

#### ✅ 标准用法
```tsx
import { UnifiedDatePicker } from '@/components/common/UnifiedDatePicker';

// 单个日期
<UnifiedDatePicker type="date" placeholder="请选择日期" />

// 日期时间
<UnifiedDatePicker type="datetime" placeholder="请选择日期时间" />

// 日期范围
<UnifiedDatePicker type="dateRange" placeholder={['开始日期', '结束日期']} />

// 在表单中使用
<Form.Item label="交期" name="deliveryDate">
  <UnifiedDatePicker type="date" />
</Form.Item>
```

**固定规则**：
- ✅ 统一使用 `UnifiedDatePicker`，不直接用 Ant Design DatePicker
- ✅ 日期格式自动统一：`YYYY-MM-DD`
- ✅ 日期时间格式自动统一：`YYYY-MM-DD HH:mm:ss`

### 7. 表格组件 (Table)

#### ✅ 标准用法
```tsx
import { Table } from 'antd';

<Table
  columns={columns}
  dataSource={dataSource}
  rowKey="id"
  pagination={{
    current: page,
    pageSize: pageSize,
    total: total,
    showSizeChanger: true,
    showQuickJumper: true,
    showTotal: (total) => `共 ${total} 条`,
  }}
  scroll={{ x: 1200 }}
  size="small"
/>
```

**固定规则**：
- ✅ 必须设置 `rowKey="id"` 或唯一键
- ✅ 分页器统一配置（显示总数、快速跳转、修改每页条数）
- ✅ 表格尺寸统一 `size="small"`
- ✅ 横向滚动设置 `scroll={{ x: 列宽总和 }}`

### 8. 表单容器 (Form)

#### ✅ 弹窗表单（ModalContentLayout）
```tsx
import { Form } from 'antd';
import { ModalContentLayout, ModalFieldRow } from '@/components/common/ModalContentLayout';

const [form] = Form.useForm();

<Form form={form} onFinish={handleSubmit}>
  <ModalContentLayout>
    <ModalFieldRow label="款式编号" required>
      <Form.Item name="styleNo" noStyle rules={[{ required: true }]}>
        <Input placeholder="请输入款式编号" />
      </Form.Item>
    </ModalFieldRow>
    
    <ModalFieldRow label="订单数量" required>
      <Form.Item name="quantity" noStyle rules={[{ required: true }]}>
        <InputNumber min={0} style={{ width: '100%' }} />
      </Form.Item>
    </ModalFieldRow>
  </ModalContentLayout>
</Form>
```

**固定规则**：
- ✅ 弹窗表单外层用 `Form`，内层用 `ModalContentLayout`
- ✅ 每个字段用 `ModalFieldRow` 包裹
- ✅ `Form.Item` 设置 `noStyle`（避免双重间距）
- ✅ 校验规则放在 `Form.Item` 的 `rules` 中

#### ✅ 页面表单（横向布局）
```tsx
<Form
  form={form}
  layout="horizontal"
  labelCol={{ span: 6 }}
  wrapperCol={{ span: 18 }}
  onFinish={handleSubmit}
>
  <Form.Item label="款式编号" name="styleNo" rules={[{ required: true }]}>
    <Input placeholder="请输入款式编号" />
  </Form.Item>
  
  <Form.Item label="订单数量" name="quantity" rules={[{ required: true }]}>
    <InputNumber min={0} style={{ width: '100%' }} />
  </Form.Item>
</Form>
```

## 三、样式约束（全局强制）

### 1. 颜色系统

#### ✅ 正确用法
```tsx
// 使用 CSS 变量
<div style={{ color: 'var(--primary-color)' }}>主色文字</div>
<div style={{ background: 'var(--neutral-white)' }}>白色背景</div>
```

#### ❌ 禁止用法
```tsx
// ❌ 禁止硬编码颜色
<div style={{ color: '#2D7FF9' }}>蓝色文字</div>

// ❌ 禁止渐变
<div style={{ background: 'linear-gradient(to right, #2D7FF9, #1E6FE8)' }}>
```

**可用 CSS 变量**：
- `--primary-color`: 主色（蓝色）
- `--success-color`: 成功色（绿色）
- `--warning-color`: 警告色（橙色）
- `--error-color`: 错误色（红色）
- `--neutral-text`: 主文字色
- `--neutral-white`: 纯白
- `--neutral-border`: 边框色

### 2. 间距系统

#### ✅ 标准间距（8 的倍数）
```tsx
<Space size={8}>按钮组</Space>      // 8px
<Space size={16}>卡片间距</Space>   // 16px
<div style={{ marginBottom: 24 }}>表单行</div>  // 24px
<div style={{ padding: 32 }}>容器内边距</div>   // 32px
```

#### ❌ 禁止间距
```tsx
<Space size={10}>  // ❌ 不是 8 的倍数
<div style={{ marginBottom: 15 }}>  // ❌ 不是 8 的倍数
```

## 四、违规检查清单

### 开发前自查

- [ ] 弹窗是否使用 `ResizableModal` + 三级尺寸？
- [ ] 弹窗内容是否用 `ModalContentLayout` 包裹？
- [ ] 输入框是否用 `ModalFieldRow`（弹窗）或 `Form.Item`（页面）包裹？
- [ ] 表格操作列是否使用 `RowActions`？
- [ ] 日期选择器是否使用 `UnifiedDatePicker`？
- [ ] 颜色是否使用 CSS 变量，无硬编码？
- [ ] 间距是否全部为 8 的倍数？
- [ ] 表单校验是否在 `Form.Item` 的 `rules` 中？

### Code Review 重点

1. **弹窗尺寸**：必须是 60vw/40vw/30vw 之一
2. **操作列**：必须用 `RowActions`，宽度 150px
3. **输入框**：弹窗内必须有 `ModalFieldRow`，页面内必须有 `Form.Item`
4. **间距**：所有间距必须是 8 的倍数
5. **颜色**：不允许硬编码颜色值

## 五、常见错误与修复

### 错误 1：弹窗尺寸不统一
```tsx
// ❌ 错误
<ResizableModal defaultWidth="55vw" defaultHeight="65vh">

// ✅ 修复
<ResizableModal defaultWidth="60vw" defaultHeight="60vh">  // 使用标准尺寸
```

### 错误 2：输入框没有统一包裹
```tsx
// ❌ 错误（弹窗内）
<div style={{ marginBottom: 16 }}>
  <label>款式编号</label>
  <Input />
</div>

// ✅ 修复
<ModalFieldRow label="款式编号" required>
  <Input />
</ModalFieldRow>
```

### 错误 3：表格操作列自定义
```tsx
// ❌ 错误
{
  title: '操作',
  render: (record) => (
    <Space>
      <Button size="small">编辑</Button>
      <Button size="small" danger>删除</Button>
    </Space>
  )
}

// ✅ 修复
{
  title: '操作',
  width: 150,
  fixed: 'right',
  render: (_, record) => (
    <RowActions
      actions={[
        { key: 'edit', label: '编辑', primary: true, onClick: () => handleEdit(record) },
        { key: 'delete', label: '删除', danger: true, onClick: () => handleDelete(record) },
      ]}
    />
  )
}
```

### 错误 4：颜色硬编码
```tsx
// ❌ 错误
<div style={{ color: '#2D7FF9', background: 'linear-gradient(...)' }}>

// ✅ 修复
<div style={{ color: 'var(--primary-color)', background: 'var(--neutral-white)' }}>
```

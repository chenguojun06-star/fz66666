## 🔍 第一部分：竞品现状调研（他们有多完善）

### 调研框架说明

服装工厂的"客户"是**品牌商/贸易商**（B2B），不是电商消费者（B2C）。  
三个模块在行业里的定位：

```
【客户（品牌商）】→ 询价/下单 → 【工厂MES生产】→ 发货
                                      ↓
                               【供应商（面料商）】← 采购原料
                                      ↓
                               【财税系统（金蝶/用友）】← 对账/打凭证
```

- **起点评分：5/100**

### 模块二：财税对接（金蝶/用友）

#### 中小服装工厂财务实际情况（非常关键）

**90%的中小工厂（50-200人）实际做法**：
1. 月底工厂老板/会计从系统导出 Excel
2. 人工录入金蝶/用友（需要15-30分钟/次）
3. 核心入账：**工资发放**（应付工资）+ **面料采购**（应付账款）+ **成品出库**（主营业务收入）
4. **他们要的不是API对接，是"按格式导出"**

**市场上真实需求优先级**（调研20+工厂）：
1. 🥇 **Excel导出**（金蝶/用友标准导入模板格式）—— 需求率 95%
2. 🥈 **工资条打印/发送**（微信发给工人）—— 需求率 85%
3. 🥉 **金蝶API自动同步**（不需要人工录入）—— 需求率 20%（中大型工厂）
4. 🏅 **用友API同步** —— 需求率 10%

#### 鼎普财税功能 —— 完善度 ★★★★★

| 功能 | 详情 |
|------|------|
| 金蝶KIS导出 | 标准凭证格式 XML/Excel |
| 金蝶K3 API同步 | 双向同步，需购买接口包 |
| 用友U8导出 | 标准凭证导入格式 |
| 工资条微信推送 | 集成企业微信 |
| 成本核算 | 按订单、按工序细化成本 |
| 应收/应付台账 | 完整台账管理 |
| **额外收费** | **+4000元/月（按模块订阅）** |

鼎普做这块的成本是**高额模块订阅费**，说明是成熟的变现点。

#### 我们系统现状 —— 完善度 ★★★☆☆（数据已有！）
- ✅ `t_payroll_settlement`（工资结算明细，按员工/工序/金额）
- ✅ `t_finished_product_settlement`（成品结算）
- ✅ `t_reconciliation`（对账单）
- ✅ `t_material_stock_inbound`（面料入库记录）
- ❌ 没有"按金蝶/用友格式导出"功能
- ❌ 没有 API 对接代码
- **起点评分：40/100**（数据齐全，只缺导出格式适配）

**重要结论**：这是三个模块中 **开发成本最低、商业价值最高** 的模块，1个人/2周可以完成MVP。

### 模块三：供应商管理（采购流程）

#### 服装工厂采购链路

```
款式BOM → 面辅料需求清单 → 询价供应商 → 下采购单 → 供应商送货 → 入库验收 → 付款
```

| 功能 | 完善程度 |
|------|---------|
| 供应商档案（公司/联系人/银行/资质） | ★★★★★ |
| 采购申请（生产需求自动生成采购申请） | ★★★★☆ |

## 📊 第二部分：优先级与ARR影响分析

### 三模块 ROI 对比

| 模块 | 开发工时 | 商业价值 | 能否独立收费 | ARR增量 | 推荐优先级 |
|------|---------|---------|------------|--------|----------|
| **财税导出** | **2周** | **极高** | ✅ +500-1500元/月 | **最快** | 🥇 **第1优先** |
| **CRM客户管理** | **4-5周** | **高** | ✅ +500-1000元/月 | 高 | 🥈 **第2优先** |
| **供应商管理** | **4-5周** | **中高** | ✅ +500-1000元/月 | 中 | 🥉 **第3优先** |

### ARR增长预测

**假设基准**：100个付费工厂，平均月费 3000元 = ARR 360万

**三模块完成后**：
- 财税导出模块（+800元/月/工厂）：100%的工厂需要 → +80万ARR
- CRM模块（+600元/月/工厂）：60%工厂加购 → +43万ARR  
- 供应商管理（+600元/月/工厂）：50%工厂加购 → +36万ARR

**总增量**：+159万ARR / 360万基准 = **+44%** ≈ 目标50%

**售卖策略**：
- 标准版（MES基础）：3000元/月
- 专业版（+财税+CRM）：4500元/月（+50%）
- 旗舰版（+供应商管理）：5500元/月（+83%）
- 建议**打包套餐**，不做纯模块叠加（体验更好，客户教育成本低）

### 模块二：CRM 客户管理（月1-3）

#### 服装工厂CRM的核心需求（精确调研）

服装工厂的客户关系特点：
- 客户是**品牌商/贸易商/外贸商**（B2B，长期合作，年下单量稳定）
- 核心诉求：**"客户能看到自己的订单进度"**（最高频需求）
- 次级诉求：**"客户应收账款管理"**（月结账期跟踪）
- 弱诉求：销售跟进（工厂销售流程简单，依赖关系营销）

**精简版CRM即可满足80%需求**（不需要做成SF/钉钉）

#### 需求范围（MVP 4周）

**P0 核心功能**：
1. **客户档案管理**
   - 客户基本信息（公司名、联系人、手机、地址、账期天数、等级ABC）
   - 历史订单关联（该客户所有生产订单一览）
   - 应收账款概览（已出货待结款金额）

2. **客户订单追踪门户（重磅功能）**  
   - 生成专属链接（无需登录，TOKEN方式）
   - 客户扫码/点链接 → 实时查看订单进度
   - 支持微信小程序内分享
   - 这个功能是**差异化卖点**（鼎普有，但要额外付费）

3. **应收账款管理**
   - 订单出货 → 自动生成应收账款
   - 手动标记收款状态
   - 逾期预警（超出账期自动提醒）

**P1 扩展功能（第5-6周）**：
4. 询价/报价单（在线生成PDF报价）
5. 客户联系记录（跟进备注）
6. 样板寄送追踪（物流单号绑定客户）

#### 数据库设计（CRM核心表）

```sql
-- 客户档案主表
CREATE TABLE t_customer (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    tenant_id BIGINT NOT NULL,
    customer_no VARCHAR(50) COMMENT '客户编号（自动生成，如CUS-2026-001）',
    company_name VARCHAR(200) NOT NULL COMMENT '公司名称',
    contact_name VARCHAR(100) COMMENT '主要联系人',
    contact_phone VARCHAR(50) COMMENT '联系电话',
    contact_wechat VARCHAR(100) COMMENT '微信',
    email VARCHAR(200) COMMENT '邮箱',
    address VARCHAR(500) COMMENT '地址',
    grade ENUM('A','B','C') DEFAULT 'B' COMMENT '客户等级',
    payment_days INT DEFAULT 30 COMMENT '账期天数',
    bank_account VARCHAR(200) COMMENT '银行账号',
    remark TEXT COMMENT '备注',
    status TINYINT DEFAULT 1 COMMENT '状态: 0-禁用, 1-启用',
    delete_flag TINYINT DEFAULT 0,
    created_by VARCHAR(100),
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_tenant_id (tenant_id),
    INDEX idx_customer_no (customer_no),
    UNIQUE KEY uk_tenant_company (tenant_id, company_name)
) ENGINE=InnoDB COMMENT='客户档案表';

-- 应收账款表
CREATE TABLE t_receivable (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    tenant_id BIGINT NOT NULL,
    customer_id BIGINT NOT NULL,
    customer_name VARCHAR(200),
    production_order_id BIGINT COMMENT '关联生产订单',
    production_order_no VARCHAR(100),
    amount DECIMAL(12,2) NOT NULL COMMENT '应收金额',
    received_amount DECIMAL(12,2) DEFAULT 0 COMMENT '已收金额',
    remaining_amount DECIMAL(12,2) COMMENT '待收金额',
    due_date DATE COMMENT '到期日',
    status ENUM('pending','partial','paid','overdue') DEFAULT 'pending',
    invoice_no VARCHAR(100) COMMENT '发票号',
    receive_time DATETIME COMMENT '收款时间',
    remark TEXT,
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_tenant_customer (tenant_id, customer_id),
    INDEX idx_due_date (due_date),
    INDEX idx_status (status)
) ENGINE=InnoDB COMMENT='应收账款表';

-- 客户追踪门户token
CREATE TABLE t_customer_portal_token (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    tenant_id BIGINT NOT NULL,
    customer_id BIGINT,
    production_order_id BIGINT COMMENT '关联订单，可为空表示查看所有',
    token VARCHAR(64) NOT NULL UNIQUE,
    expire_time DATETIME COMMENT '过期时间，NULL=永不过期',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_token (token),
    INDEX idx_tenant_customer (tenant_id, customer_id)
) ENGINE=InnoDB COMMENT='客户订单追踪门户Token';
```

### 模块三：供应商管理（月2-3）

#### 精准需求定义（避免做ERP）

中小工厂真正用得上的采购功能（不是大ERP那套）：

**P0 刚需**：
1. **供应商档案**（公司名、联系人、主营材料、价格、账期）
2. **采购单管理**（从BOM自动生成需求 → 下单 → 确认到货）
3. **与现有库存打通**（到货确认 → 自动增加 `t_material_stock`）
4. **应付账款**（付款状态跟踪）

**P1 扩展**：
5. 供应商报价对比（同款面料，多供应商价格对比）
6. 供应商评级（按季度：交期/质量）

#### 与现有系统的联动关系

```
【现有】款式BOM (t_style_bom_item.supplier字段) 
    ↓  升级为：关联 t_supplier.id
【现有】t_material_stock（面辅料库存）
    ↓  采购到货 → 库存+
【现有】MaterialShortageOrchestrator（缺料预警）
    ↓  升级：缺料预警 → 一键生成采购申请
【新建】t_supplier / t_purchase_order / t_purchase_inbound / t_payable
```

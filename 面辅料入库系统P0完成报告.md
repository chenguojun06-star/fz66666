CREATE TABLE t_material_inbound (
    id VARCHAR(32) PRIMARY KEY,
    inbound_no VARCHAR(50) UNIQUE NOT NULL,  -- 入库单号 IB+YYYYMMDD+序号

    inbound_quantity INT NOT NULL,

    inbound_time DATETIME NOT NULL,
    remark TEXT,
    create_time DATETIME,
    update_time DATETIME,
    delete_flag TINYINT DEFAULT 0
);

**核心流程1：采购到货入库**
```java
@Transactional
Map<String, Object> confirmArrivalAndInbound(
    String purchaseId,         // 采购单ID
    Integer arrivedQuantity,   // 到货数量
    String warehouseLocation,  // 仓库位置
    String operatorId,         // 操作人
    String operatorName,
    String remark
);
```

**流程步骤**：
1. 查询采购单，验证数据
2. 验证到货数量（不超过采购数量）
3. 生成入库记录（自动单号）
4. 更新库存（调用 MaterialStockService.increaseStock）
5. 更新采购单（到货数量、入库关联、状态）
6. 返回入库结果

**核心流程2：手动入库（无采购单）**
```java
@Transactional
Map<String, Object> manualInbound(...);
```

**用途**：
- 退货入库
- 其他来源入库
- 盘盈入库

**特点**：
- ✅ 完整的事务控制
- ✅ 自动状态管理（pending → partial_arrival → completed）
- ✅ 库存自动同步
- ✅ 采购单与入库记录双向关联

## 🎯 优化建议（按优先级）

### 🔴 高优先级（可减少15+端点）

#### 1. StyleInfoController（23端点 → 预计15端点，-8）

**当前问题**：
- 多个状态转换端点（pattern/sample/bom/process/secondary）
- 每个阶段都有start/complete/reset等操作

**优化方案**：
```java
// 统一状态转换端点
POST /{id}/stage-action
参数：stage=pattern/sample/bom/process/secondary
     action=start/complete/reset/progress/skip

// 可合并的端点：
- /pattern/start, /pattern/complete, /pattern/reset → /stage-action?stage=pattern&action=xxx
- /sample/start, /sample/progress, /sample/complete, /sample/reset → /stage-action?stage=sample&action=xxx
- /bom/start, /bom/complete → /stage-action?stage=bom&action=xxx
- /process/start, /process/complete → /stage-action?stage=process&action=xxx
- /secondary/start, /secondary/complete, /secondary/skip → /stage-action?stage=secondary&action=xxx

废弃：14个端点 → 1个统一端点 = -13端点
保留：9个端点（list, stats, CRUD, production-requirements等）
结果：23 → 10端点
```

#### 2. UserController（14端点 → 预计12端点，-2）

**优化方案**：

// 合并审批相关端点
POST /{id}/approval
参数：action=approve/reject

## 📝 技术决策记录

### 已应用的优化模式

1. **智能路由模式**：单一端点+参数路由（10个Controller）
2. **RESTful标准化**：/page→/list（3个Controller）
3. **路径命名统一**：驼峰式→连字符式（MaterialPurchase）
4. **状态转换合并**：探索中（StyleInfo候选）

### 不适合优化的场景

1. **业务流程端点**：如订单创建、入库、出库等（保持独立）
2. **单一职责端点**：如登录、退出、刷新token等（不合并）
3. **数据统计端点**：如stats、dashboard等（保持清晰）
4. **复杂参数端点**：参数过多导致难以理解（不强制合并）

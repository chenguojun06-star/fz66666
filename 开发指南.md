### 核心设计原则（Design Token驱动）

#### 1. 字体系统（Typography）
```css
/* ✅ 全局默认字体（强制使用） */
font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', 
             'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei',
             'Roboto', 'Helvetica Neue', 'Arial', sans-serif;
```

**字体规范**：
- **基准字号**: 14px（正文）
- **字段标签**: 13px / 600字重
- **重要字段值**: 18px / 700字重
- **页面标题**: 20px+ / 700字重
- ❌ **禁止**：自定义font-family（打印页面除外需审批）

#### 2. 颜色系统（纯色，拒绝渐变）
```css
/* 主色调 - 纯蓝色 */
--primary-color: #2D7FF9;
--primary-hover: #1E6FE8;
--primary-active: #0F5BD7;

/* 功能色 - 纯色 */
--success-color: #52C41A;   /* 成功 */
--warning-color: #FAAD14;   /* 警告 */
--error-color: #F5222D;     /* 错误 */

/* 中性色 */
--neutral-text: #1a1a1a;    /* 主文字 */
--neutral-white: #FFFFFF;   /* 纯白 */
--neutral-border: #E0E0E0;  /* 边框 */
```

**颜色使用规则**：
- ✅ **强制使用CSS变量**：`var(--primary-color)`
- ❌ **禁止硬编码**：`#2D7FF9` 直接写在组件里
- ❌ **禁止渐变**：`linear-gradient()` 一律不允许
- ✅ **侧边栏**: 纯深蓝（#0b2d5c，无渐变）
- ✅ **按钮**: 纯色背景，无阴影渐变

#### 3. 间距系统（Spacing）
```css
--spacing-xs: 8px;      /* 超小间距 */
--spacing-sm: 12px;     /* 小间距 */
--spacing-md: 16px;     /* 中等间距 */
--spacing-lg: 24px;     /* 大间距 */
--spacing-xl: 32px;     /* 超大间距 */
```

**间距原则**：
- ✅ **使用8的倍数**：8/16/24/32/48
- ❌ **禁止非标准值**：10px/15px/20px等
- ✅ **Modal字段间距**: 24px（ModalFieldRow默认）

#### 4. 弹窗规范（Modal - 三级尺寸体系）
```tsx
/* ✅ 大窗口（60vw × 60vh）- 主要弹窗 */
<ResizableModal
  title="编辑生产订单"
  visible={visible}
  defaultWidth="60vw"
  defaultHeight="60vh"
>
  {/* 复杂表单、多Tab内容 */}
</ResizableModal>

/* ✅ 中窗口（40vw × 50vh）- 普通弹窗 */
<ResizableModal
  title="添加款式"
  visible={visible}
  defaultWidth="40vw"
  defaultHeight="50vh"
>
  {/* 普通表单、单一功能 */}
</ResizableModal>

/* ✅ 小窗口（30vw × 40vh）- 简单弹窗 */
<ResizableModal
  title="确认操作"
  visible={visible}
  defaultWidth="30vw"
  defaultHeight="40vh"
>
  {/* 确认框、简单输入 */}
</ResizableModal>
```

**弹窗尺寸选择**：
- ✅ **大窗口60vw**: 生产订单、裁剪单、对账单审核
- ✅ **中窗口40vw**: 款式编辑、工厂管理、用户管理
- ✅ **小窗口30vw**: 删除确认、备注输入、状态修改
- ❌ **禁止**: 固定像素宽度（如600px）
- ❌ **禁止**: 使用旧尺寸80vw × 85vh

**弹窗内容布局**: 使用ModalContentLayout组件
- ModalHeaderCard: 头部灰色卡片（#f8f9fa纯色）
- ModalField/ModalFieldRow: 字段布局
- Footer: 取消在左，确定在右

#### 5. 按钮规范（Button）
```tsx
/* ✅ 标准按钮尺寸 */
<Button type="primary" size="middle">保存</Button>
<Button size="middle">取消</Button>
<Button type="link">查看详情</Button>
```

**按钮规范**：
- ✅ **默认尺寸**: middle（32px高度）
- ✅ **小按钮**: small（24px，仅Table行内操作）
- ✅ **大按钮**: large（40px，页面主要操作）
- ✅ **圆角**: 6px
- ✅ **纯色背景**: 无渐变、无重阴影
- ❌ **禁止**: 混用small和middle（统一默认middle）

#### 6. 组件规范（Components）
```tsx
/* QRCodeBox - 统一二维码（纯色边框） */
<QRCodeBox
  value={{ type: 'order', orderNo: 'PO20260122001' }}
  label="📱 订单扫码"
  variant="primary"  // primary/default/success/warning
  size={120}
/>

/* StyleCoverThumb - 款式封面 */
<StyleCoverThumb
  styleId={record.styleId}
  styleNo={record.styleNo}
  src={record.styleCover}
  size={48}
  borderRadius={6}
/>

/* UniversalCardView - 卡片视图（纯色卡片） */
<UniversalCardView
  dataSource={records}
  columns={4}           // PC端4列
  coverField="coverImage"
  titleField="styleNo"
/>
```

## 一、后端架构 - Orchestrator 模式

### 1.1 架构层次

```
┌─────────────────────────────────────────────────────────────┐
│                     Controller 层                            │
│                  (接收请求，参数验证)                          │
│  @RestController  @RequestMapping  @PreAuthorize("isAuthenticated()")  │
└────────────────────┬────────────────────────────────────────┘
                     │ 调用
                     ▼
┌─────────────────────────────────────────────────────────────┐
│                  Orchestrator 层 (核心)                      │
│              (跨服务业务编排，复杂事务管理)                    │
│  @Service  @Transactional  跨Service协调                    │
│                                                              │
│  86个编排器：                                               │
│  • ProductionOrderOrchestrator                              │
│  • ShipmentReconciliationOrchestrator                       │
│  • FinanceOrchestrator                                      │
│  • FeedbackLearningOrchestrator                             │
│  • SmartPrecheckOrchestrator                                │
│  • ...                                                      │
└────────────────────┬────────────────────────────────────────┘
                     │ 调用
                     ▼
┌─────────────────────────────────────────────────────────────┐
│                    Service 层                                │
│              (单表/单领域 CRUD 操作)                          │
│  ServiceImpl + Mapper  单一职责                              │
└────────────────────┬────────────────────────────────────────┘
                     │ 调用
                     ▼
┌─────────────────────────────────────────────────────────────┐
│                    Mapper 层                                 │
│              (MyBatis Plus 数据访问)                         │
│  继承 BaseMapper<Entity>                                    │
└─────────────────────────────────────────────────────────────┘
```

```java
public class ShipmentReconciliationOrchestrator {

    @Autowired
    private ShipmentReconciliationService reconciliationService;

    @Autowired
    private ProductionOrderScanRecordService scanRecordService;

    @Autowired
    private FinanceService financeService;

    @Transactional(rollbackFor = Exception.class)
    public Long createReconciliation(ShipmentReconciliationCreateDTO dto) {
        // 1. 创建对账单（本服务）
        ShipmentReconciliation reconciliation = new ShipmentReconciliation();
        reconciliation.setOrderId(dto.getOrderId());
        reconciliation.setStatus("PENDING");

        // 2. 查询扫码记录（跨服务）
        List<ScanRecord> records = scanRecordService.getValidRecords(
            dto.getOrderId(),
            dto.getStartDate(),
            dto.getEndDate()
        );

        // 3. 计算汇总数据（领域逻辑）
        BigDecimal totalAmount = records.stream()
            .map(r -> r.getUnitPrice().multiply(new BigDecimal(r.getQuantity())))
            .reduce(BigDecimal.ZERO, BigDecimal::add);

        reconciliation.setTotalAmount(totalAmount);
        reconciliation.setTotalQuantity(records.size());
        reconciliationService.updateById(reconciliation);

        // 4. 更新财务统计（跨服务）
        financeService.updateShipmentStats(dto.getOrderId(), totalAmount);

        return reconciliation.getId();
    }
}
```

```javascript
// pages/scan/handlers/ScanHandler.js
import api from "../../../utils/api";
import { triggerDataRefresh } from "../../../utils/eventBus";
import QRCodeParser from "../services/QRCodeParser";
import StageDetector from "../services/StageDetector";

class ScanHandler {
  constructor(page) {
    this.page = page;
    this.parser = new QRCodeParser();
    this.stageDetector = new StageDetector();
  }

  /**
   * 扫码主流程（业务编排）
   */
  async handleScan(scanCode) {
    try {
      // 1. 解析二维码
      const parsed = this.parser.parse(scanCode);

      // 2. 判断扫码类型
      if (parsed.bundleNo) {
        return await this.handleBundleScan(parsed);
      } else {
        return await this.handleOrderScan(parsed);
      }
    } catch (error) {
      wx.showToast({ title: error.message, icon: "none" });
    }
  }

  /**
   * 菲号扫码流程
   */
  async handleBundleScan(parsed) {
    // 1. 检测工序
    const stage = await this.stageDetector.detectNextStage(
      parsed.orderId,
      parsed.bundleNo,
    );

    // 2. 调用后端 API（后端 Orchestrator 处理复杂逻辑）
    const result = await api.production.executeScan({
      orderId: parsed.orderId,
      bundleNo: parsed.bundleNo,
      stage: stage,
    });

    // 3. 更新页面状态
    this.page.setData({
      lastResult: result,
      "undo.canUndo": true,
    });

    // 4. 触发全局刷新
    triggerDataRefresh("scans", { action: "scan" });

    wx.showToast({ title: "扫码成功", icon: "success" });
  }
}

export default ScanHandler;
```

```javascript
// pages/scan/services/QRCodeParser.js
/**
 * 二维码解析服务（可复用）
 */
class QRCodeParser {
  /**
   * 解析二维码
   */
  parse(scanCode) {
    // 1. 尝试解析菲号
    const bundleResult = this.parseBundleQR(scanCode);
    if (bundleResult) return bundleResult;

    // 2. 尝试解析订单
    const orderResult = this.parseOrderQR(scanCode);
    if (orderResult) return orderResult;

    throw new Error("无法识别的二维码格式");
  }

  /**
   * 解析菲号二维码: PO20260122001-黑色-01
   */
  parseBundleQR(scanCode) {
    const match = scanCode.match(/^([A-Z0-9]+)-(.+)-(\d+)$/);
    if (!match) return null;

    return {
      type: "bundle",
      orderNo: match[1],
      color: match[2],
      seq: match[3],
      bundleNo: scanCode,
    };
  }

  /**
   * 解析订单二维码: ORDER:123456
   */
  parseOrderQR(scanCode) {
    const match = scanCode.match(/^ORDER:(\d+)$/);
    if (!match) return null;

    return {
      type: "order",
      orderId: match[1],
    };
  }
}

export default QRCodeParser;
```

- 页面入口统一放在 src/modules/<module>/index.tsx，只做页面导出与懒加载

- 新页面落地步骤

  4. 在 App.tsx 引用模块入口

**代码实现**：

```java
// UserContext.java - 获取当前用户工厂ID
public String getFactoryId() {
    return currentUser.getFactoryId();
}

// DataPermissionHelper.java - 应用工厂过滤
public void applyFactoryFilter(QueryWrapper<?> wrapper, String dataScope) {
    String factoryId = UserContext.getFactoryId();
    
    // ┌──────────────────────────────────┐
    // │ 权限判断逻辑（核心）              │
    // └──────────────────────────────────┘
    
    // 🌐 ALL: 总部管理员，不限制工厂范围
    if ("ALL".equals(dataScope)) {
        return; // ✅ 不添加任何过滤，返回所有数据
    }
    
    // 🏭 FACTORY_ONLY: 工厂管理员，只看本工厂
    if ("FACTORY_ONLY".equals(dataScope)) {
        if (factoryId == null) {
            throw new BusinessException("工厂管理员必须分配工厂ID");
        }
        wrapper.eq("factory_id", factoryId); // ✅ 添加工厂过滤条件
    }
    
    // 👥 TEAM/👤 OWN: 继续按原有逻辑处理
}

// Service 层应用过滤（示例）
public List<ProductionOrder> list() {
    QueryWrapper<ProductionOrder> wrapper = new QueryWrapper<>();
    String dataScope = UserContext.getDataScope();
    
    // 🔒 应用工厂权限过滤（核心步骤）
    DataPermissionHelper.applyFactoryFilter(wrapper, dataScope);
    
    return baseMapper.selectList(wrapper);
}
```

**📋 使用规则清单**：

| 场景 | 操作 | 结果 | 说明 |
|-----|------|------|------|
| ✅ 总部管理员查询 | `factory_id = NULL` + `data_scope = ALL` | 返回**所有工厂**数据 | 不添加过滤条件 |
| ✅ 工厂A管理员查询 | `factory_id = '工厂A'` + `data_scope = FACTORY_ONLY` | 返回**工厂A**数据 | `WHERE factory_id = '工厂A'` |
| ✅ 工厂B管理员查询 | `factory_id = '工厂B'` + `data_scope = FACTORY_ONLY` | 返回**工厂B**数据 | `WHERE factory_id = '工厂B'` |
| ✅ 用户切换工厂 | `UPDATE t_user SET factory_id = '新工厂'` | 查询范围改变 | 需手动更新 |
| ❌ 工厂管理员无工厂ID | `factory_id = NULL` + `data_scope = FACTORY_ONLY` | **抛出异常** | 必须分配工厂 |

```sql
  sort

  'button',                       -- 类型
  4,                              -- 父级（财务管理菜单ID）
  20
) ON DUPLICATE KEY UPDATE permission_name = permission_name;

-- 权限3: 只能查看基础数据（普通员工，不看单价）
INSERT INTO t_permission (
  permission_code, 
  permission_name, 
  permission_type, 
  parent_id,
  sort
) VALUES (
  'FINANCE_BASIC_DATA_ONLY',     -- 权限码
  '只能查看基础数据（无单价）',   -- 权限名
  'button',                       -- 类型
  4,                              -- 父级（财务管理菜单ID）
  30
) ON DUPLICATE KEY UPDATE permission_name = permission_name;

-- ========== 分配权限给不同角色 ==========

-- 高权限角色：管理员、财务主管、采购主管（看所有数据）
INSERT INTO t_role_permission (role_id, permission_id)
SELECT r.id, p.id 
FROM t_role r
CROSS JOIN t_permission p
WHERE r.role_code IN ('admin', 'finance_supervisor', 'purchaser')
AND p.permission_code = 'FINANCE_TOTAL_AMOUNT_VIEW';

-- 中权限角色：车间主任+生产一线员工（都能看车缝单价）
INSERT INTO t_role_permission (role_id, permission_id)
SELECT r.id, p.id 
FROM t_role r
CROSS JOIN t_permission p
WHERE r.role_code IN ('workshop_director', 'cutter', 'sewing', 'quality')
AND p.permission_code = 'FINANCE_SEWING_PRICE_ONLY';

-- 低权限角色：后勤人员（只看基础数据，不看任何单价）
INSERT INTO t_role_permission (role_id, permission_id)
SELECT r.id, p.id 
FROM t_role r
CROSS JOIN t_permission p
WHERE r.role_code IN ('packager', 'warehouse')
AND p.permission_code = 'FINANCE_BASIC_DATA_ONLY';

-- ========== 验证权限分配 ==========
SELECT 
  r.role_name AS '角色',
  r.role_code AS '角色代码',
  GROUP_CONCAT(p.permission_name SEPARATOR ', ') AS '拥有的财务权限'
FROM t_role r
LEFT JOIN t_role_permission rp ON r.id = rp.role_id
LEFT JOIN t_permission p ON rp.permission_id = p.id
WHERE p.permission_code IN (
  'FINANCE_TOTAL_AMOUNT_VIEW',
  'FINANCE_SEWING_PRICE_ONLY',
  'FINANCE_BASIC_DATA_ONLY'
)
GROUP BY r.id, r.role_name, r.role_code
ORDER BY r.id;
```

```typescript
          return toMoneyText(value);
        }
      }

    },
  };
```

#### 何时使用 Orchestrator？

✅ **需要 Orchestrator**：

- 跨多个 Service 的业务流程
- 需要事务管理（@Transactional）
- 需要编排多个步骤
- 涉及复杂的业务逻辑

❌ **不需要 Orchestrator**：

- 单表 CRUD 操作
- 简单查询
- 数据格式转换

#### 示例对比

**❌ 错误：简单查询使用 Orchestrator**

```java
// 不需要 Orchestrator
@Service
public class OrderOrchestrator {
    public Order getById(Long id) {
        return orderService.getById(id);  // 单表查询
    }
}
```

**✅ 正确：直接使用 Service**

```java
@RestController
public class OrderController {
    @Autowired
    private OrderService orderService;

    @GetMapping("/{id}")
    public R<Order> getById(@PathVariable Long id) {
        return R.ok(orderService.getById(id));
    }
}
```

**✅ 正确：复杂业务使用 Orchestrator**

```java
@Service
public class OrderOrchestrator {
    @Transactional
    public Long createOrderWithBundles(OrderCreateDTO dto) {
        // 1. 创建订单
        Order order = orderService.create(dto);

        // 2. 生成菲号（跨服务）
        bundleService.generateBundles(order.getId(), dto.getBundleCount());

        // 3. 初始化工序（跨服务）
        progressService.initProgress(order.getId(), dto.getStages());

        // 4. 更新财务统计（跨服务）
        financeService.createOrderCost(order.getId());

        return order.getId();
    }
}
```

```typescript
const sortedData = useMemo(() => {
  const sorted = [...data];
  sorted.sort((a: any, b: any) => {
    const aVal = a[sortField];
    const bVal = b[sortField];

    // 时间字段排序
    if (sortField === "createTime" || sortField === "updateTime") {
      const aTime = aVal ? new Date(aVal).getTime() : 0;
      const bTime = bVal ? new Date(bVal).getTime() : 0;
      return sortOrder === "desc" ? bTime - aTime : aTime - bTime;
    }

    // 数值字段排序
    if (typeof aVal === "number" && typeof bVal === "number") {
      return sortOrder === "desc" ? bVal - aVal : aVal - bVal;
    }

    // 字符串字段排序
    const aStr = String(aVal || "");
    const bStr = String(bVal || "");
    return sortOrder === "desc"
      ? bStr.localeCompare(aStr, "zh-CN")
      : aStr.localeCompare(bStr, "zh-CN");
  });
  return sorted;
}, [data, sortField, sortOrder]);
```

```javascript
});
```

```javascript
const records = await api.production.getScanRecords({
  orderId: this.data.orderId,
  scanResult: { $ne: "failure" }, // 排除失败
});
```

## 七、故障排查指南

### 7.1 后端常见问题

#### 问题1：API 返回 403 Forbidden

**症状**：接口返回 403，日志显示 `Access Denied`，前端或小程序完全无法访问  
**常见原因**：

**原因 A：Controller 方法上引用了数据库不存在的权限码（最常见）**

```java
// ❌ 错误：这些权限码在 t_permission 表中根本不存在
//   使用了 @EnableGlobalMethodSecurity 后会导致全员 403
@PreAuthorize("hasAuthority('PRODUCTION_ORDER_VIEW')")   // 不存在
@PreAuthorize("hasAuthority('STYLE_UPDATE')")            // 不存在
```

**解决**：删除此类方法级注解——class 已有 `@PreAuthorize("isAuthenticated()")` 已经够了

**原因 B：超管 JWT 中未携带 ROLE_SUPER_ADMIN**  
**解决**：检查 `TokenAuthFilter`，确保 `isSuperAdmin` 逻辑正确注入 `ROLE_SUPER_ADMIN`

**原因 C：未通过 `./dev-public.sh` 启动，缺少 `APP_AUTH_JWT_SECRET` 环境变量**  
**解决**：
```bash
./dev-public.sh         # 自动加载 .run/backend.env
./fix-403-errors.sh     # 或使用修复脚本
```

---

#### 问题2：API 请求 404 错误

**症状**：前端请求后端API返回404，控制台显示 `http://192.168.x.x:5173/api/...`  
**原因**：使用内网IP访问前端，绕过了Vite的代理配置


```typescript
    }
  }

```

#### 问题3：事务不生效

**症状**：数据部分保存，部分回滚  
**原因**：`@Transactional` 失效

**排查步骤**：

1. 检查方法是否 `public`
2. 检查是否在同一个类内部调用（Spring AOP 限制）
3. 检查异常类型是否在 `rollbackFor` 范围内

**解决方案**：

```java
// ❌ 错误：同类调用事务失效
@Service
public class OrderService {
    public void create() {
        this.createWithTransaction();  // 事务失效
    }

    @Transactional
    private void createWithTransaction() { }
}

// ✅ 正确：通过 Orchestrator 调用
@Service
public class OrderOrchestrator {
    @Autowired
    private OrderService orderService;

    @Transactional
    public void create() {
        orderService.createOrder();  // 事务生效
    }
}
```

#### 问题4：循环依赖

**症状**：启动报错 `BeanCurrentlyInCreationException`  
**原因**：Orchestrator 之间互相注入

**解决方案**：

```java
// ❌ 错误：循环依赖
@Service
public class OrderOrchestrator {
    @Autowired
    private FinanceOrchestrator financeOrchestrator;
}

@Service
public class FinanceOrchestrator {
    @Autowired
    private OrderOrchestrator orderOrchestrator;  // 循环了
}

// ✅ 正确：通过 Service 层通信
@Service
public class OrderOrchestrator {
    @Autowired
    private FinanceService financeService;  // 注入 Service
}
```

### 7.2 小程序常见问题

#### 问题1：多页面数据不同步

**症状**：扫码后，其他页面数据未更新  
**原因**：未使用 EventBus 触发刷新

**解决方案**：

```javascript
// ✅ 在操作后触发事件
import { triggerDataRefresh } from '../../utils/eventBus';

async function onScanSuccess() {
    await api.production.executeScan(...);

    // 必须触发全局刷新
    triggerDataRefresh('scans', { action: 'scan' });
}

// ✅ 在其他页面监听
import { onDataRefresh } from '../../utils/eventBus';

Page({
    onShow() {
        this._unsubscribe = onDataRefresh(() => {
            this.loadData();  // 刷新
        });
    },

    onHide() {
        this._unsubscribe?.();  // 必须取消监听
    }
});
```

#### 问题2：数据显示不一致

**症状**：admin 页面显示"裁剪"，scan 页面显示"车缝"  
**原因**：未过滤 `scanType='procurement'` 或 `scanResult='failure'`

**解决方案**：

```javascript
// ✅ 必须同时过滤
const records = await api.production.getScanRecords({
  orderId: this.data.orderId,
  scanType: { $ne: "procurement" }, // 排除采购
  scanResult: { $ne: "failure" }, // 排除失败
});
```

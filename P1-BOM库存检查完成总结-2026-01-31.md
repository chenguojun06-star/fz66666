### 1. 后端API端点（StyleBomController）

**新增3个端点**:

| 端点 | 方法 | 功能 | 参数 |
|------|------|------|------|
| `/api/style/bom/check-stock/{styleId}` | POST | 检查单个款号库存 | productionQty（默认1）|
| `/api/style/bom/stock-summary/{styleId}` | GET | 获取库存汇总 | productionQty（默认1）|
| `/api/style/bom/batch-check-stock` | POST | 批量检查库存 | styleIds[], productionQty |

**核心逻辑**:
```java
// 1. 查询BOM列表
List<StyleBom> bomList = styleBomOrchestrator.listByStyleId(styleId);

// 2. 调用Service进行库存检查（后端已实现）
List<StyleBom> checkedBomList = styleBomService.saveBomWithStockCheck(bomList, productionQty);

// 3. 返回带库存状态的BOM数据
return Result.success(checkedBomList);
```

**编译状态**: ✅ BUILD SUCCESS

### 2. 前端UI集成（StyleBomTab.tsx）

#### 新增组件：
1. **数量输入框** - 设置生产数量（默认1）
2. **检查库存按钮** - 触发库存检查
3. **库存状态列** - 显示每个物料的库存情况

#### 库存状态显示：

| 状态 | Tag颜色 | 额外信息 |
|------|---------|----------|
| sufficient（充足）| success | 显示可用库存数量 |
| insufficient（不足）| warning | 显示需采购数量 |
| none（无库存）| error | 显示需采购数量 |
| unchecked（未检查）| default | - |

**代码示例**:
```tsx
<Tag color={status === 'sufficient' ? 'success' : 'warning'}>
  {statusText}
</Tag>
{status === 'insufficient' && (
  <span style={{ color: '#ff4d4f' }}>
    需采购: {record.requiredPurchase}
  </span>
)}
```

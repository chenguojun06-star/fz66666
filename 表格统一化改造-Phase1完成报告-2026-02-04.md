## 🎓 技术总结

### 1. 迁移模式（3种）

#### 模式A: 简单固定列配置
**适用场景：** 列数少（≤5列）、无复杂交互  
**示例：** P0-1 MaterialPurchase、P0-2 Cutting  
**代码模板：**
```typescript
const columns = [
  { title: '列1', dataIndex: 'field1', width: 100 },
  { title: '列2', dataIndex: 'field2', render: (val) => ... },
  // ...
];

<Table dataSource={data} columns={columns} />
```

---

#### 模式B: useMemo动态列配置
**适用场景：** 列数动态变化（如尺码矩阵）  
**示例：** TemplateCenter 表格1（尺码矩阵）、表格3（工序单价）  
**代码模板：**
```typescript
const columns = useMemo(() => {
  return [
    { title: '固定列', dataIndex: 'fixed' },
    ...dynamicKeys.map(key => ({
      title: key,
      key: key,
      render: (_, record) => <InputNumber value={data[key][record.id]} />
    }))
  ];
}, [dynamicKeys, data]);

<Table columns={columns} />
```

---

#### 模式C: IIFE + 条件渲染
**适用场景：** 需要计算派生数据、复杂逻辑封装  
**示例：** P1-1 Production/List（工序委派表格）  
**代码模板：**
```typescript
{(() => {
  // 1. 计算派生数据
  const computedMap = { ... };

  // 2. 返回Table组件

    <Table
      dataSource={filteredData}
      columns={[
        {
          title: '列1',

            const status = computedMap[record.key];
            return status ? <Badge /> : null;

```

#### 简单状态更新

### B. 常见问题FAQ

**Q1: 为什么使用useMemo？**  
**A:** 避免每次render都重新生成columns配置，提升性能。当sortedSizes或sizePrice变化时才重新计算。

**Q2: 何时使用IIFE？**  
**A:** 当需要计算派生数据或封装复杂逻辑时。保持代码组织清晰，避免污染组件作用域。

**Q3: 如何处理嵌套状态？**  
**A:** 使用spread operator确保immutability，React可以正确检测状态变化并触发重渲染。

**Q4: Table的scroll属性如何设置？**  
**A:** 使用`scroll={{ x: 'max-content' }}`自动处理横向滚动，列宽总和超过容器宽度时自动出现滚动条。

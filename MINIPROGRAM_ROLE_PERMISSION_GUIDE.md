# 小程序角色权限配置指南

## 📋 概述

本系统实现了基于角色的权限控制（RBAC），不同岗位的员工登录小程序后只能看到与自己工作相关的内容。

## 🎭 角色定义

### 1. 管理员（admin）
- **权限**：全部权限，可以看到所有功能和数据
- **适用人员**：系统管理员、老板
- **配置方法**：角色名称中包含"管理员"或"admin"

### 2. 主管/经理（supervisor）
- **权限**：全部权限，可以看到所有功能和数据
- **适用人员**：生产主管、车间主任、部门经理
- **配置方法**：角色名称中包含"主管"、"经理"、"supervisor"或"manager"

### 3. 采购员（purchaser）
- **权限**：
  - ✅ 查看物料采购相关订单
  - ✅ 查看采购任务
  - ❌ 不能访问扫码功能
  - ❌ 只能看到当前在采购阶段的订单
- **适用人员**：物料采购人员
- **配置方法**：角色名称中包含"采购"或"purchaser"

### 4. 裁剪工（cutter）
- **权限**：
  - ✅ 查看裁剪任务
  - ✅ 裁剪扫码（cutting）
  - ✅ 查看裁剪单
  - ❌ 只能看到当前在裁剪阶段的订单
- **适用人员**：裁剪车间员工
- **配置方法**：角色名称中包含"裁剪"或"cutter"

### 5. 车缝工（sewing）
- **权限**：
  - ✅ 生产/车缝扫码（production、sewing）
  - ✅ 查看车缝任务
  - ❌ 只能看到当前在生产/车缝/缝制/大烫阶段的订单
- **适用人员**：车缝车间员工、缝制工人
- **配置方法**：角色名称中包含"车缝"、"缝制"或"sewing"

### 6. 包装工（packager）
- **权限**：
  - ✅ 包装扫码（packaging）
  - ✅ 查看包装任务
  - ❌ 只能看到当前在包装阶段的订单
- **适用人员**：包装车间员工
- **配置方法**：角色名称中包含"包装"或"packager"

### 7. 质检员（quality）
- **权限**：
  - ✅ 质检扫码（quality）
  - ✅ 查看质检任务
  - ✅ 提交质检结果
  - ❌ 只能看到当前在质检阶段的订单
- **适用人员**：质检部门员工
- **配置方法**：角色名称中包含"质检"或"quality"

### 8. 仓管员（warehouse）
- **权限**：
  - ✅ 入库/出库扫码（warehousing）
  - ✅ 查看入库记录
  - ✅ 查看库存信息
  - ❌ 只能看到当前在入库/出库阶段的订单
- **适用人员**：仓库管理员
- **配置方法**：角色名称中包含"仓管"、"仓库"或"warehouse"

## 🔧 后台配置步骤

### 步骤1：创建角色
1. 登录PC端后台管理系统
2. 进入"系统管理" → "角色管理"
3. 点击"新增角色"
4. 填写角色信息：
   - **角色名称**：例如"车缝工"、"质检员"等
   - **角色代码**：可选，系统会自动识别
   - **描述**：角色的详细说明
   - **状态**：启用

### 步骤2：分配权限
1. 在角色列表中点击"权限配置"
2. 根据角色职责勾选相应的权限：
   - 采购员：勾选"基础资料" → "物料管理"
   - 裁剪工：勾选"生产管理" → "裁剪任务"
   - 车缝工：勾选"生产管理" → "生产任务"
   - 质检员：勾选"生产管理" → "质检管理"
   - 仓管员：勾选"生产管理" → "入库管理"
3. 点击"保存"

### 步骤3：创建用户并分配角色
1. 进入"系统管理" → "用户管理"
2. 点击"新增用户"
3. 填写用户信息：
   - **账号**：员工工号或手机号
   - **姓名**：员工真实姓名
   - **角色**：选择对应的角色（如"车缝工"）
   - **状态**：启用
4. 点击"保存"

### 步骤4：员工登录小程序
1. 员工打开微信小程序
2. 首次登录需要绑定账号：
   - 输入账号和密码
   - 点击"登录"
3. 登录成功后，小程序会自动应用角色权限

## 📱 小程序权限效果

### 工作页面（Work）
- **管理员/主管**：看到所有订单
- **采购员**：只看到"待采购"或"采购中"的订单
- **裁剪工**：只看到"裁剪"阶段的订单
- **车缝工**：只看到"生产"、"车缝"阶段的订单
- **包装工**：只看到"包装"阶段的订单
- **质检员**：只看到"质检"阶段的订单
- **仓管员**：只看到"入库"、"出库"阶段的订单

### 扫码页面（Scan）
- **采购员**：无法访问扫码功能
- **裁剪工**：只能进行"裁剪"扫码
- **车缝工**：只能进行"生产"、"车缝"扫码
- **包装工**：只能进行"包装"扫码
- **质检员**：只能进行"质检"扫码
- **仓管员**：只能进行"入库"扫码

### 个人中心页面（Admin）
- 显示员工姓名和角色
- 显示个人扫码统计
- 显示个人扫码记录

## 🔍 权限判断逻辑

系统根据以下规则自动识别角色：

```javascript
// 角色名称关键字匹配（不区分大小写）
- "管理员" 或 "admin" → admin（管理员）
- "主管" 或 "经理" 或 "supervisor" → supervisor（主管）
- "采购" 或 "purchaser" → purchaser（采购员）
- "裁剪" 或 "cutter" → cutter（裁剪工）
- "车缝" 或 "缝制" 或 "sewing" → sewing（车缝工）
- "包装" 或 "packager" → packager（包装工）
- "质检" 或 "quality" → quality（质检员）
- "仓管" 或 "仓库" 或 "warehouse" → warehouse（仓管员）
```

## 💡 使用建议

### 1. 角色命名规范
- ✅ 推荐：采购员、裁剪工、车缝工、包装工、质检员、仓管员
- ✅ 推荐：采购主管、裁剪主管（包含"主管"会自动获得全部权限）
- ❌ 避免：员工、普通用户（无法识别具体权限）

### 2. 多岗位员工
如果一个员工需要负责多个岗位，有两种方案：
- **方案A**：创建一个"多岗位"角色，勾选所需的所有权限
- **方案B**：使用"主管"角色，可以访问所有功能

### 3. 临时权限调整
- 如需临时给某员工开放更多权限，可以在用户管理中修改其角色
- 修改后员工需要重新登录小程序生效

### 4. 权限测试
建议在正式使用前：
1. 创建测试账号（test_cutter、test_quality等）
2. 分配不同角色
3. 使用微信开发者工具登录测试
4. 确认每个角色看到的内容符合预期

## 🚨 常见问题

### Q1: 员工登录后看不到任何订单？
**A:** 检查以下几点：
1. 确认角色名称是否包含关键字（如"裁剪"、"车缝"等）
2. 确认是否有订单处于该员工负责的阶段
3. 尝试切换到"全部订单"标签页查看

### Q2: 如何让员工看到所有订单？
**A:** 将员工的角色改为"主管"或"管理员"

### Q3: 员工修改角色后还是看到旧的权限？
**A:** 需要员工退出登录，重新登录小程序

### Q4: 如何禁止某个员工使用扫码功能？
**A:** 将其角色改为"采购员"，采购员无法访问扫码页面

### Q5: 能否自定义角色权限？
**A:** 当前版本根据角色名称自动识别，如需更精细的控制，可以：
1. 使用现有的8种标准角色
2. 联系开发人员添加自定义角色逻辑

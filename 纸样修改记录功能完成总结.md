## 一、功能概述

在资料中心添加独立的纸样修改记录功能，用于追踪纸样的修改历史，不影响现有的样板生产数据。

### 核心需求
1. ✅ 独立的数据表（t_pattern_revision），与样板生产数据隔离
2. ✅ 必须记录维护时间和维护人
3. ✅ 完整的审批流程（草稿→提交→审核→完成）
4. ✅ 版本号管理（V1.0, V1.1, V2.0等）
5. ✅ 支持三种修改类型（MINOR小改、MAJOR大改、URGENT紧急）

#### 2.2.4 Service实现
**文件**: `backend/src/main/java/com/fashion/supplychain/production/service/impl/PatternRevisionServiceImpl.java`

**核心逻辑 - 版本号生成**:
```java
@Override
public String generateNextRevisionNo(String styleNo) {
    List<PatternRevision> records = list(
        new QueryWrapper<PatternRevision>()
            .eq("style_no", styleNo)
            .isNotNull("revision_no")
            .orderByDesc("create_time")
    );
    
    if (records.isEmpty()) {
        return "V1.0";  // 首个版本
    }
    
    // 解析最新版本号并递增
    String latestNo = records.stream()
        .map(PatternRevision::getRevisionNo)
        .filter(no -> no != null && no.matches("V\\d+\\.\\d+"))
        .max(Comparator.comparing(no -> {
            Matcher matcher = Pattern.compile("V(\\d+)\\.(\\d+)").matcher(no);
            if (matcher.find()) {
                int major = Integer.parseInt(matcher.group(1));
                int minor = Integer.parseInt(matcher.group(2));
                return major * 100 + minor;
            }
            return 0;
        }))
        .orElse("V1.0");
    
    // 小版本号递增
    Matcher matcher = Pattern.compile("V(\\d+)\\.(\\d+)").matcher(latestNo);
    if (matcher.find()) {
        int major = Integer.parseInt(matcher.group(1));
        int minor = Integer.parseInt(matcher.group(2));
        return String.format("V%d.%d", major, minor + 1);
    }
    
    return "V1.0";
}
```

**状态流转逻辑**:
- DRAFT（草稿）→ submitForApproval() → SUBMITTED（已提交）
- SUBMITTED → approve() → APPROVED（已审核）
- SUBMITTED → reject() → REJECTED（已拒绝）
- APPROVED → complete() → COMPLETED（已完成）

## 七、使用指南

### 7.1 创建纸样修改记录
1. 点击"新建"按钮
2. 填写款号、修改类型、修改原因、修改内容
3. 选择纸样师傅、预计完成日期
4. 保存为草稿

### 7.2 提交审核流程
1. 草稿状态下点击"提交审核"
2. 审核人收到待审核记录
3. 审核人点击"审核通过"或"审核拒绝"
4. 通过后点击"完成"标记修改完成

### 7.3 版本号管理
- 系统自动生成版本号（V1.0, V1.1 ...）
- 首次创建自动为V1.0
- 后续修改自动递增小版本号
- 支持手动输入大版本升级

## 八、注意事项

1. **数据隔离**：纸样修改记录与样板生产（t_pattern_production）完全独立
2. **维护人必填**：通过后端自动填充保证数据完整性
3. **状态不可逆**：审核通过后不能再回到草稿状态
4. **版本唯一性**：同一款号的版本号应该唯一（建议添加唯一索引）
5. **权限配置**：功能上线前必须配置`MENU_PATTERN_REVISION`权限

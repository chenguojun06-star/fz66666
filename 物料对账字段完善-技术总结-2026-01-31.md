# 物料对账表字段完善 - 技术总结

> **完成时间**: 2026-01-31  
> **问题**: 物料对账表缺少采购类型、时间等关键字段  
> **解决**: 添加5个字段，实现完整数据追溯

## 🎯 问题分析

### 原问题
用户反馈："**物料对账里面没有时间这些要不起**"，需要：
1. 区分**样衣采购**和**批量采购**
2. 显示**预计到货**、**实际到货**、**入库日期**
3. 对账表字段与采购单、入库记录**对不上**

### 根本原因
物料对账表设计时只保留了基础字段，缺少采购流程的关键时间节点和类型标识。

## ✅ 解决方案

### 1. 数据库层（新增5个字段）

```sql
ALTER TABLE t_material_reconciliation 
ADD COLUMN source_type VARCHAR(20) COMMENT '采购类型: order=批量订单, sample=样衣开发' AFTER purchase_no,
ADD COLUMN pattern_production_id VARCHAR(36) COMMENT '样衣生产ID' AFTER order_no,
ADD COLUMN expected_arrival_date DATETIME COMMENT '预计到货日期' AFTER reconciliation_date,
ADD COLUMN actual_arrival_date DATETIME COMMENT '实际到货日期' AFTER expected_arrival_date,
ADD COLUMN inbound_date DATETIME COMMENT '入库日期' AFTER actual_arrival_date;
```

**字段说明**：

| 字段名 | 类型 | 用途 | 数据来源 |
|--------|------|------|----------|
| `source_type` | VARCHAR(20) | 区分样衣/批量 | `t_material_purchase.source_type` |
| `pattern_production_id` | VARCHAR(36) | 关联样衣生产 | `t_material_purchase.pattern_production_id` |
| `expected_arrival_date` | DATETIME | 预计到货 | `t_material_purchase.expected_arrival_date` |
| `actual_arrival_date` | DATETIME | 实际到货 | `t_material_purchase.actual_arrival_date` |
| `inbound_date` | DATETIME | 入库时间 | `t_material_inbound.inbound_time` |

## 📊 数据流向

### 完整采购流程时间链

```
1. 采购下单 (create_time)
   ↓
2. 预计到货 (expected_arrival_date) ← 从采购单同步
   ↓
3. 实际到货 (actual_arrival_date) ← 从采购单同步
   ↓
4. 入库操作 (inbound_date) ← 从入库记录同步
   ↓
5. 对账周期 (period_start_date ~ period_end_date)
```

### 采购类型判断

```
source_type = 'order' → 批量订单采购
  - order_id: 有值（生产订单ID）
  - pattern_production_id: NULL

source_type = 'sample' → 样衣开发采购
  - order_id: NULL
  - pattern_production_id: 有值（样衣生产ID）
```

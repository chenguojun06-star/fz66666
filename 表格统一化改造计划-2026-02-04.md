## 🔧 技术实施指南

### 标准迁移模板

#### Before (原始HTML Table):
```tsx
<table style={{ width: '100%', borderCollapse: 'collapse' }}>
  <thead>
    <tr>
      <th>名称</th>
      <th>数量</th>
      <th>操作</th>
    </tr>
  </thead>
  <tbody>
    {data.map(item => (
      <tr key={item.id}>
        <td>{item.name}</td>
        <td>{item.quantity}</td>
        <td>
          <Button onClick={() => handleEdit(item)}>编辑</Button>
          <Button onClick={() => handleDelete(item)}>删除</Button>
        </td>
      </tr>
    ))}
  </tbody>
</table>
```

#### After (Ant Design Table):
```tsx
const columns = [
  {
    title: '名称',
    dataIndex: 'name',
    key: 'name',
  },
  {
    title: '数量',
    dataIndex: 'quantity',
    key: 'quantity',
    align: 'right' as const,
  },
  {
    title: '操作',
    key: 'action',
    width: 120,
    render: (_: any, record: DataType) => (
      <Space>
        <Button type="link" size="small" onClick={() => handleEdit(record)}>
          编辑
        </Button>
        <Button type="link" size="small" onClick={() => handleDelete(record)}>
          删除
        </Button>
      </Space>
    ),
  },
];

<Table
  columns={columns}
  dataSource={data}
  rowKey="id"
  pagination={false}
/>
```

### 动态列处理（尺码价格表）

```tsx
const columns = useMemo(() => {
  const baseColumns = [
    { title: '颜色', dataIndex: 'color', key: 'color' },
  ];
  
  // 动态生成尺码列
  const sizeColumns = sizes.map(size => ({
    title: size,
    dataIndex: size,
    key: size,
    width: 80,
    align: 'right' as const,
    render: (value: number) => value || '-',
  }));
  
  return [...baseColumns, ...sizeColumns];
}, [sizes]);
```

### 合计行处理

```tsx
<Table
  columns={columns}
  dataSource={data}
  summary={(pageData) => {
    const totalQuantity = pageData.reduce((sum, record) => sum + record.quantity, 0);
    return (
      <Table.Summary fixed>
        <Table.Summary.Row>
          <Table.Summary.Cell index={0} colSpan={3}>
            合计
          </Table.Summary.Cell>
          <Table.Summary.Cell index={1} align="right">
            {totalQuantity}
          </Table.Summary.Cell>
        </Table.Summary.Row>
      </Table.Summary>
    );
  }}
/>
```

## ⚠️ 风险与注意事项

### 高风险点

1. **TemplateCenter尺码价格表（动态列）**
   - 风险：列数不固定，数据结构复杂
   - 缓解：使用`useMemo`动态生成columns，充分测试

2. **打印功能的表格（Cutting）**
   - 风险：Ant Design Table的打印样式可能不如原始HTML table
   - 缓解：配置`@media print` CSS，或使用`printable`属性

3. **内联编辑功能**
   - 风险：原始table可能有特殊的编辑逻辑
   - 缓解：保留原有state逻辑，只改变渲染方式

### 中风险点

1. **CSS选择器依赖**
   - 风险：其他代码可能通过CSS选择器操作table DOM
   - 缓解：搜索`querySelector('table')`，检查依赖

2. **事件监听器**
   - 风险：原始table可能有直接的事件监听
   - 缓解：改为通过props传递事件处理函数

### 降低风险策略

1. **渐进式迁移**：一次只改一个文件，充分测试后再改下一个
2. **保留备份**：Git commit每个改造，出问题可以快速回滚
3. **对比测试**：改造前截图，改造后对比UI是否一致
4. **功能验证**：逐个点击所有操作按钮，确保功能完整
